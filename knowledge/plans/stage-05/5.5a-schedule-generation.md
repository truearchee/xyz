## Scope confirmation
Delivers schedule-driven section generation (Sub-scope A): migration 0020 (nullable schedule config on
`course_modules`), a pure `generate_sections` + ORM adapter, required `schedule` on the create-module
contract with a 422 for absence, and the call-site audit. Does NOT deliver: metadata edit + resolver
(5.5b), asset_kind/attachments (5.5c), reseed (5.5d), or any UI/browser-gate/full-E2E (5.5e).

## Approach
- Schedule columns are NULLABLE (validation/422 in the service layer, not the DB) so ORM-direct test
  helpers that bypass the API keep working — minimizes the call-site blast radius (Plan-agent Task 3).
- Reuse `starts_on`/`ends_on` for course dates; only `week_start_day`/`session_pattern`/`quiz_day` are
  net-new. Weekdays as lowercase day-names (avoids the int off-by-one the spec warns about).
- Generation is a pure function (D8) over (start, end, week_start_day, session_pattern); the adapter
  builds ModuleSection rows inside the existing creation transaction (D14 — already atomic, commit at
  `admin.py:110`).
- `schedule` is a required nested pydantic model → absence is an automatic 422 (no fallback branch).

- `frontend/src/lib/api/*` — regenerated client; `CreateModuleForm.tsx` + `shared.ts` minimal compat.

## Order of operations
1. Model + migration 0020. 2. Pure generator. 3. Schema + service. 4. Call-site audit (backend tests
first, then E2E payloads). 5. Unit tests. 6. Verify in an isolated Docker stack. 7. Regen client + tsc.
8. Knowledge artifacts.

## Test strategy
Pure unit tests carry the oracle (28 / 21 / 7 / max-week-7 / 0-Friday) + boundary + partial-week +
determinism + schema validation. Endpoint tests prove 28-section generation end-to-end + the 422.
Migration round-trip via alembic up→base→up. tsc proves the regenerated client is consumed correctly.

## Risks & mitigations
- 422 is suite-breaking → audit + update all API-path call sites; nullable columns keep ORM-direct
  helpers working.
- E2E section-selection breaks under schedule-driven titles → deferred to 5.5e (full suite after the
  5.5d reseed), payloads made contract-valid now, documented.
- Migration seam under parallel branches → develop on 0013, rebase the seam at merge (single head).
