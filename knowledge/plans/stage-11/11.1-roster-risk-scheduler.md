## Scope confirmation
Deliver scheduler infrastructure, `AgentRun`, deterministic risk snapshots, lecturer roster risk, and student
gentle risk. Do not build recommendations, AI phrasing, assessment analysis, workload planning, `.ics`, grade
advice, assistant changes, gamification changes, or new activity event types.

## Approach
Create a small `analytics` domain for Stage 11-owned writes and risk read DTOs. Risk computation reads shipped
inputs through `platform/query`, produces live current risk for UI reads, and persists the same deterministic
contract during `AgentRun` jobs. The scheduler is a Python service using the repo's existing Redis/RQ and a
Postgres advisory lock, avoiding a new package dependency. UI changes are thin and use the existing monochrome
component primitives.

- `backend/alembic/versions/0056_agent_runs_risk_snapshots.py` — create `agent_runs` and

## Order of operations
1. Add knowledge files and ADR.
2. Add migration/models/config/scheduler queue plumbing.
3. Add risk engine, read models, services, and endpoints.
4. Add backend unit/API tests and fix behavior until green.
5. Regenerate OpenAPI client, then wire frontend surfaces.
6. Add Playwright browser gate.
7. Run verification, write report from output, update status/log/roadmap.

## Test strategy
- Unit-test risk reasons, tier precedence, exact metric-key support, input hash stability, and cutoff behavior.
- Unit-test scheduler lock/idempotency and run failure handling without waiting for wall-clock time.
- API-test admin/lecturer/student authz and no Stage 10/AI side effects.
- Browser-test manual trigger, run polling, lecturer full view, student gentle view, live-on-read recompute, and
  idempotent retrigger.

## Risks & mitigations
- Scheduler parent process DB connections can break forked workers → scheduler uses its own process and isolated
  DB sessions, following the 4.6 recovery note.
- Risk thresholds can drift from product intent → thresholds are config defaults and changing them requires an
  `algorithmVersion` bump.
- Lecturer/student copy bleed → separate schemas and frontend components; student endpoint omits tier labels and
  peer metrics entirely.
- Parallel Stage 10 merge conflict → no Stage 10 tables or event types are read, and migration uses assigned
  `0056`.
