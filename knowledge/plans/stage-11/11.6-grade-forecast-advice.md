## Plan-review findings folded in (must-fix items resolved in this plan)

**Eng review (`/plan-eng-review`)**
- **[BLOCKER] Number-formatting contract.** Raw `Decimal required_remaining_average` (e.g.
  `87.7777…`, or `9E+1` scientific notation) is NOT what a template prints, so an allowed-set built
  from raw Decimals makes the **template fail its own numeric validator** — no safe fallback. **Fix:**
  quantize `reqAvg` HALF_UP to **1 dp** once; render the template with that exact string; build
  `allowedNumbers` from the **rendered template tokens** (the 11.2 invariant,
  `recommendations.build_deterministic_payload:426-429`) plus the quantized metric in bare **and**
  `%`-suffixed normalized forms. (Resolution §4.)
- **[BLOCKER] Status-label collision.** `FORECAST_LABELS` maps `at_risk → "At risk"`
  (`progress/schemas.py:90`), and the reused safety guard bans the substring `"at risk"`
  (`recommendations.py:81`). **Fix:** never put `FORECAST_LABELS` strings in student copy or
  `allowedFactPhrases`; use student-safe phrasings; the contradiction check operates on the **state
  enum**, not the human label. (Resolution §4.)
- **[MAJOR] `AI_REQUEST_LOG_FEATURES` model tuple** (`ai_request_log.py:27-35`) must add
  `'grade_forecast_advice'` in the same change as the migration — `tests/test_shared_check_union.py`
  asserts DB CHECK == model tuple. (Resolution §2.)
- **[MAJOR] Read-time upsert/enqueue race.** Use PG `insert().on_conflict_do_update(index_elements=
  [student_id, module_id])` and **commit before enqueue** so the row id is stable and the
  `forecast-advice-{id}` job id dedupes; worker re-claims under `FOR UPDATE`. (Resolution §5.)
- **[MAJOR] Deterministic-provider coupling.** Pin the `section_type` literal, a **named** split-marker
  constant (no inline magic string), and the exact payload key `templateAdvice`. (Resolution §3.)
- **[MAJOR] Type-union touch-points (4):** `CompletionResult.parsed`, `LLMGateway.complete` param,
  `OutputValidator.validate` param union, and its **return** union — all four. (Resolution §3.)
- **[MAJOR] `grade_forecast_advice` stays OUT of `FEATURES_REQUIRING_INGESTION_JOB`** and passes
  `ingestion_job_id=None`. (Resolution §3.)
- **[MINOR] Commit to extraction** for the single forecast path (Design call A → decided: extract).
- **[MINOR] Hash the quantized reqAvg**, not the raw Decimal (reproducibility + anti-flap). (Resolution
  §Reproducibility.)
- **[MINOR] Pin advice length bound** (~700 chars) + empty-after-strip → `too_short`.

**Adversarial validators (`/codex`)**
- **[CRITICAL] No contradiction check exists in reused code** — false hope on `impossible` ("you can
  still reach A with hard work") cites no numbers/banned words and passes both guards. **Fix:** build a
  **state-aware contradiction guard** (§4.3).
- **[CRITICAL] Impossible-case shaming vocabulary uncovered** ("fallen too far", "no way", "give up",
  "blew it", "hopeless", "too late", "out of reach", "doomed", …). **Fix:** an 11.6-specific banned set
  **layered on top of** the 11.2 guard (extend, don't mutate the shared tuple). Keep `"isn't possible"`
  / `"impossible"` **allowed** (the honest template needs them). (§4.2.)
- **[CRITICAL] `"X percent"` vs bare-number asymmetry** is a negative-control killer — handled by the
  bare+`%` allowed-set fix and a prompt instruction to use `%`. (§4.1.)
- **[HIGH] Peer/diagnosis paraphrases + hyphenated number-words** ("twenty-five") slip the substring
  guards. **Fix:** extend peer/diagnosis sets; split number-word tokens on `-`/`'`. (§4.2/§4.4.)
- **[MEDIUM] `allowedFactPhrases` is decorative** (never enforced). **Decision:** drop the unenforced
  whitelist claim and instead add an **unsupported-fact lexicon** for grade advice (resit, retake,
  extra credit, deadline, drop/withdraw, office hours, …). (§4.2.)
- For `impossible`, **drop `reqAvg` (which can exceed 100) from the allowed set** so a ">100% needed"
  quote is impossible; the impossible template cites `best`, not `reqAvg`. (§4.1.)

**Design review (`/plan-design-review`)**
- **[MAJOR] Impossible copy** must lead with the **reachable goal**, not the loss, and must not restate
  what `ForecastBody` already shows (`ProgressDashboard.tsx:326-339` already renders best-reachable +

  badge already carries the one allowed status color in `ForecastBody`). Card = the *next move*,

- **[MINOR] Card vocabulary:** match the sibling analytics cards (StudentRiskCard:

#### 4.1 Number formatting + allowed-set contract (resolves the BLOCKER)
- Quantize `required_remaining_average` HALF_UP to **1 dp** (`reqAvgDisplay`) **once**; the template and
  the prompt both use `reqAvgDisplay`. (Display as e.g. `87.3%`.)
- `allowedNumbers = _numeric_tokens(<rendered template advice text>)` ∪ `{bare, "%"-suffixed}` normalized
  forms of each **quantized** metric the state may legitimately cite (`reqAvgDisplay`, and the integer
  `target_points` if cited). Built from the **rendered** string (11.2 invariant) so the template's own
  numbers are tautologically allowed.
- For `state == "impossible"`, **exclude `reqAvg` from `allowedNumbers`** (it can exceed 100; the
  template cites `best`, not `reqAvg`) — a ">100% needed" quote is then un-allowable.
- `allowedFactPhrases` is **not** used as a positive whitelist (it was decorative in 11.2). Grade letters
  (`target_letter_grade`, `best_reachable_letter_grade`, `current_letter_grade`) are allowed facts by
  construction; fabricated facts are blocked by the unsupported-fact lexicon (§4.2).

#### 4.2 `ForecastAdviceNumericConsistencyValidator` + extended copy guards

  `_normalize_text`) and the peer/diagnosis scans from `recommendations.py` **by import**.
- **Number-word hyphenation fix:** when tokenizing, also split each word token on `-`/`'` and test the
  sub-parts against `NUMBER_WORDS` (catches "twenty-five", "fifty-fifty").
- **Extended banned/peer/diagnosis/unsupported sets — 11.6-specific, layered (do not mutate the shared
  11.2 tuples):**
  - Defeatist/shaming (impossible-case core): `too far`, `fallen behind`, `falling behind`, `no way`,
    `give up`, `gave up`, `giving up`, `blew it`, `blown it`, `hopeless`, `too late`, `out of reach`,
    `no longer achievable`, `no longer reachable`, `can't get`, `cannot get`, `won't reach`,
    `will never`, `doomed`, `lost cause`, `write off`, `written off`, `throw in the towel`, `no point`,
    `not worth`, `in trouble`, `in danger`, `thin ice`, `underperforming`, `struggling`, `slacking`,
    `not trying`, `poor effort`, `bad grades`, `weak performance`. (Keep `isn't possible` /
    `not possible` / `impossible` ALLOWED — the honest fact.)
  - Peer: `average student`, `others in the`, `fellow students`, `where the class`, `than the class`,

- `validate_forecast_advice(parsed, *, context)` runs: schema → **numeric/invention** (no token outside
  `allowedNumbers`) → **peer/diagnosis/unsupported** → **extended student-safety** (the reused

  the whole result.

#### 4.3 State-aware contradiction guard (resolves the CRITICAL gap)
- Context carries `forecastState`. Phrase families (substring, post-`_normalize_text`):
  - `REACHABLE_CLAIMS = {still reach, can reach, still possible, within reach, still achievable,
    still reachable, can still get, on track for, is reachable}`
  - `UNREACHABLE_CLAIMS = {not possible, isn't possible, is not possible, no longer reachable,
    not reachable, out of reach, cannot reach, can't reach, won't reach, more than the remaining}`
- Per AI-eligible state:
  - `impossible`: reject any `REACHABLE_CLAIMS` (false hope); **require** an `UNREACHABLE_CLAIMS` phrase
    **and** the presence of `best_reachable_letter_grade` (honest + constructive). `target` letter via
    no-contradiction (must not be claimed reachable).
  - `at_risk` / `requires_high_score`: reject `UNREACHABLE_CLAIMS` (target IS reachable) and overclaiming
    (`guaranteed`, `certain`, `definitely will`); **must include** `target_letter_grade` + `reqAvgDisplay`.
- Because the deterministic template is honest **by construction**, this guard can be strict without
  risk to the AI layer (fallback is always valid). Negative controls (§Test plan) prove valid AI output
  still passes.

## Deterministic template advice (per state) + must-include
`[target]`,`[best]`,`[current]` are letter grades; `[reqAvgDisplay]` is reqAvg HALF_UP to 1 dp;
`[module]` is the module title. Each template passes **both** validators **by construction** (it is the
fallback) and is the immediate render. AI rephrases only the AI-eligible states. The advice is the
*next move* — it does **not** restate `ForecastBody`'s fact lines verbatim.

| State | AI? | Template advice (deterministic, fallback-safe) | Must-include |
|---|---|---|---|
| `at_risk` | yes | "**[target]** is within reach — around **[reqAvgDisplay]%** on the work that's left gets you there. A focused review of **[module]** is a strong next step." | `[target]`, `[reqAvgDisplay]` |
| `requires_high_score` | yes | "**[target]** is still on the table — it'll take about **[reqAvgDisplay]%** on what's left, so a focused plan for **[module]** is your best next step." | `[target]`, `[reqAvgDisplay]` |
| `impossible` | yes | "Aiming for **[best]** is your strongest goal from here — putting your focus there is well worth it. Reaching **[target]** would now take more than the remaining work can add, so it may help to revisit your target when you're ready." | `[best]` (+ an unreachable-framing phrase; `[target]` no-contradiction) |
| `on_track` | no | "You're on track for **[target]** — keeping your current pace on the remaining work should get you there." | n/a (deterministic only) |
| `achieved` | no | "You've already done enough to reach **[target]** — nice work. Keeping steady will hold it." | n/a (deterministic only) |
| `final_no_remaining` | no | "Your final result for **[module]** is **[final_letter_grade]**." (`final_letter_grade` is always set when `remaining_weight==0`, never `None`) | n/a (deterministic only) |

- AI-eligible states = `{at_risk, requires_high_score, impossible}` (rule-15 frugality: the states where
  warmth + direction help most). `on_track`/`achieved`/`final_no_remaining` are deterministic-only — no
  AI call. (Owner may expand to `on_track` later; see Design call D.)
- Impossible template leads with `[best]`, frames unreachability as plain math ("more than the remaining
  work can add"), offers a revisit — honest, no shaming, no false hope, no banned substring, tone-neutral.

## Reproducibility
- Advice row: `algorithmVersion="forecast-advice-v1"`, `inputHash`, `sourceCutoffAt`, `forecastState`.
- `inputHash = sha256(canonical_json({algorithmVersion, forecastState, targetLetterGrade, targetPoints,
  earnedSoFar(quantized), remainingWeight(quantized), reqAvgDisplay, currentLetterGrade,
  bestReachableLetterGrade}))` — **quantized** values, not raw 26-dp Decimals, so the hash is
  reproducible across environments and does **not** flap near threshold boundaries (rule-15).
- AI cache: `aiProvenance { modelId, promptVersion, inputHash, generatedAt }`; reused only while
  `aiInputHash==inputHash` and `aiPromptVersion=="v1"`. `sourceCutoffAt` pins the forecast instant.

- **Student-safety incl. impossible shaming (MUST FAIL):** "you're failing"; "you're at risk";
  "you've fallen too far"; "there's no way"; "give up on the A"; "it's too late"; "hopeless".
- **Contradiction (MUST FAIL):** state=impossible + "you can still reach [target]"; state=at_risk +
  "[target] is out of reach"; state=impossible advice missing the unreachable framing or `[best]`;
  state=at_risk overclaiming "you'll definitely get [target]".
- **Negative controls (MUST PASS + render AI text):** at_risk "[target] is still reachable — aim for
  about [reqAvgDisplay]% on what's left; a focused review of [module] helps"; impossible "[best] is your
  strongest goal from here; [target] is no longer reachable, and you can revisit your target". Plus the
  `%`-spelled regression control ("…around [reqAvgDisplay]% …" must pass; the prompt forbids the word
  "percent", and both forms are in the allowed set). A standalone positive+negative control test like
  11.2's `test_standalone_output_validators_cover_positive_and_negative_controls`.
- Endpoint authz: non-student → **403**; not-enrolled → **404**; two co-enrolled students each get
  their **own** advice (distinct numbers; no other student's data in the payload).

- AI job: success stores `ai_text` + provenance + `ai_request_log_id`; forced invalid → retry →

## Risks / mitigations
- **Template fails its own validator (was BLOCKER)** → quantized 1-dp reqAvg, allowed-set from rendered
  template + bare/`%` forms, `%`-not-"percent" prompt rule; template-self-validation tests incl.
  fractional + scientific-notation reqAvg.
- **Status-label "At risk" banned-term collision (was BLOCKER)** → never surface `FORECAST_LABELS` in
  student copy; contradiction on state enum.
- **False hope / shaming on impossible (was CRITICAL)** → state-aware contradiction guard + layered
  impossible-case banned lexicon; gate asserts unreachable framing + best + no banned phrase.
- **Over-strict validator kills AI** → negative controls (backend + gate) must PASS and render; template
  always valid by construction.
- **Read-time concurrency** → on-conflict upsert + commit-before-enqueue + stable job id; concurrency test.
- **Drift between My Progress and advice** → single extracted forecast path + equality test.
- **Redundancy with `ForecastBody`** → advice is the *next move*, visually subordinate, tone-neutral.
- **Migration discipline** → single `0059` below `0060`; round-trip + CHECK-parity tests.
