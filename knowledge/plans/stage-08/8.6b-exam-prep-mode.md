# Stage 8.6b — Implementation plan (HOW)

Spec: [[specs/stage-08/8.6b-exam-prep-mode]]. Builds on the 8.6a coordinator seam (only adds a strategy +
read model + a prompt + the create branch + UI — coordinator core untouched).

## Read-first findings (confirmed)
- Student scope list EXISTS: `GET /student/modules/{id}/exam-prep-scopes` (`listStudentExamPrepScopes`) →
  `ExamPrepScopeSummary {id, name, coveredWeeks, available, reasonCode}`; frontend `api.quiz.listExamPrepScopes`.
  Reuse — no new list endpoint.
- `resolve_exam_prep_scope(scope, student_id)` (`domains/quiz/scope_service.py` — FORBIDDEN to import) →
  `ScopeResolution(available, reason_code, ready_section_ids, processing_section_ids, scope_key)`. Its body
  only calls platform/query primitives `resolve_sections_by_weeks` + `resolve_section_eligibility`
  (`platform/query/section_eligibility_read.py`, `EligibilityResult{ready_ids, processing_ids}`) + reads the
  `AssessmentScope` model → replicate that mapping in a platform/query read (no quiz import).
- Quiz-pointer reason codes: `"processing"` (summaries generating) / `"no_eligible_sections"` (nothing
  quiz-bearing). available=True ⇒ ready set exists.
- Quiz CTA target: the exam-prep quiz is taken in `StudentQuizModesPanel` → mounted at
  `/student/modules/{moduleId}` (StudentModuleDetail). CTA = a LINK there; `api.quiz.startExamPrep` is the
  student's action, never the assistant's.
- Stage 9 weak areas: `progress_read.list_topic_mastery(student_id, module_id)` →
  `[(StudentTopicMasterySnapshot{module_section_id, mastery_percentage, status_label}, ModuleSection)]`.
- The 8.6a list/visibility reads are already section-OR-module aware → exam_prep (carries `attached_module_id`
  = scope.module_id) flows through them unchanged.

   `EXPECTED_ALEMBIC_VERSION` 0042→0043.

3. **Coordinator** `mode_strategies.py`: `_exam_prep_turn` builder + `_MODE_TURN_BUILDERS['exam_prep']`.
   Resolve scope (access_denied if not visible) → ready sections → blob = covered sections' approved

   history + the marked question. `prompt_key=("exam_prep","v1")`, `AssistantGroundedAnswer`, grounding via
