## Scope confirmation
Delivers: lecture-attached conversations + messages that persist/reload, a real gateway-backed answer
(non-streaming, ungrounded), the conversation-list-ready data shape, and a thin inline-style chat panel
on the student section page. Does NOT deliver: grounding/retrieval (8.2), streaming (8.3), the 8.4
sidebar/standalone window/floating widget, modes/workspaces (8.4–8.6), save-to-glossary (8.5), Tailwind/
component-library/monochrome repaint (Stage 4.9), or lecturer/admin assistant.

## Approach
New `assistant` domain mirroring `quiz`. **Create-then-poll** execution: `POST …/messages` saves the
user message + a `pending` assistant message, COMMITS, enqueues an `ai`-queue job AFTER commit; the job
runs `LLMGateway.complete(priority="interactive", feature="assistant")`, persists the answer +
provenance, flips status; the frontend polls with the 4.5d backoff until terminal. This reuses the quiz
pattern verbatim, exercises the interactive limiter headroom (rule 15), and makes the 8.3 SSE swap a
transport-only change.

## Test strategy
- pytest: open-or-create idempotency + race re-read; send idempotency (no dup user msg / no dup AI call);
  retry never dupes user msg; ownership 404-not-403; access re-check 404 after unpublish/unassign;
  readiness states; gateway path writes AIRequestLog `feature='assistant'` at interactive priority.
- E2E (deterministic provider at the boundary, full gateway path): the spec's 8.1 browser-gate block.
- Real-provider smoke: one live call, assert model-ID echo, record numbers.

## Risks & mitigations
- **Head-of-line blocking** of interactive turns behind background jobs → `at_front=True` + limiter
  interactive headroom; verify worker concurrency in `docker-compose`.
- **Long synchronous AI call** → avoided by create-then-poll (no long-held request).
- **Multi-head migrations** with parallel Stage 9 → rebase `down_revision` before PR; assert single head.
- **History-as-transcript** packing is a contract stretch → acceptable (transcript_text is generic
  "source text"); extending `registry.render` is a separate platform decision, deferred.
