# Stage 8.6a — Implementation plan (HOW)

Approved in plan mode (decisions D1=B, D2=A, D3=A). This is the HOW; the spec
([[specs/stage-08/8.6a-mode-coordinator-homework]]) is the WHAT/WHY/DONE. Scope = **8.6a only** (mode
foundation + Homework), then pause.

## Read-first findings (confirmed against code)
- `conversation_kind` is a **CHECK constraint** (not a DB/Python enum) with FOUR existing values
  (`lecture_default, manual, floating_widget, workspace`) — all map to the SAME existing general-chat path.
  Widen by DROP+CREATE the named constraint `ck_assistant_conversations_kind`. (`assistant_conversation.py:40-44`,
  migration `0032`.)
- Coordinator seam = `generate_assistant_answer_async` (`generation_service.py:218-313`): `_claim_message`
  → `_resolve_and_retrieve` → short-circuits → ONE `gateway.complete(priority="interactive",
  feature="assistant")` → `decide_grounding` → `_persist_grounded_answer` + `_build_snapshot`. `_claim_message`
  loads `conv` but does NOT yet read `conversation_kind`. `_TurnContext.section_id` is already nullable.
- `context_snapshot` = JSONB on `assistant_messages` (migration 0033); adding fields needs no migration.
- Idempotency template = `open_or_create_conversation` + `uq_assistant_conversations_one_lecture_default`
  partial-unique index (`service.py:188-247`, `assistant_conversation.py:52-58`).
- **Migrations 0042–0047 are free** (head 0041; gap 0034–0037 = frozen 8.3; 0038–0039 = Stage 9).
- Deterministic provider dispatches by `rendered.prompt_key.name` and **raises `ValueError` for unknown
  names** (`provider.py:436`) — a new prompt name MUST get a fixture branch or every CI/E2E homework turn
  crashes.
- Registry requires `{{transcript}}` in `content` and only substitutes `{{transcript}}`/`{{section_type}}`
  (`registry.py:81,140`). `CHECKSUMS.json` is a CI drift guard.
- The conversation **list** + **detail** read models are section-centric (inner join on
  `attached_section_id`): `get_visible_student_conversation_list` (`student_summary_read.py:387`),
  `get_conversation_detail` 404s when section is null (`service.py:278`), `_resolve_owned_conversation`
  only re-checks visibility for section-bound convos (`service.py:164`). A module-bound homework
  conversation must still be listable/openable → these get a module-aware branch.
- Existing `assistant/v2` prompt is **Cerebras** (general chat = plain writing). Homework was specced as a
  reasoning task → Nvidia/Think, but the rule-11 smoke proved Think returns `not_json` for an interactive
  turn (F-6e ramble) → **homework also routes Cerebras/V2/32k** (`backend: cerebras`), the chat route.
  Off-topic markers (`provider.py:60`): movie/weather/restaurant/…

## Steps

### 1. Migration 0042 — `backend/alembic/versions/0042_assistant_modes_homework.py` (`down_revision="0041"`)
- DROP+CREATE `ck_assistant_conversations_kind` → adds `'homework_help'` only (5-value set). Downgrade
  restores the 4-value set.
- `add_column` `attached_module_id UUID NULL` FK → `course_modules.id` ON DELETE CASCADE.
- Two partial-unique indexes (resume-or-create over the nullable section):
  `uq_assistant_conversations_one_homework_section` on `(student_id, attached_module_id, attached_section_id)`
  WHERE `kind='homework_help' AND attached_section_id IS NOT NULL AND deleted_at IS NULL`;
  `uq_assistant_conversations_one_homework_module` on `(student_id, attached_module_id)`
  WHERE `kind='homework_help' AND attached_section_id IS NULL AND deleted_at IS NULL`.

### 2. Model — `assistant_conversation.py`
Add `attached_module_id`; update the kind CHECK string + add the two partial `Index(...)` in lockstep.

### 3. Read models — `student_summary_read.py` + `assistant_retrieval_read.py`
- `get_visible_student_module(db, *, student_id, module_id) -> VisibleStudentModule | None` (active member of
  an active module).
- `get_visible_student_conversation_list`: make it section-OR-module aware — LEFT JOIN section, JOIN module on
  `COALESCE(section.course_module_id, conv.attached_module_id)`, membership on the effective module; visibility
  predicate `(section-bound AND section published+active) OR (section NULL AND attached_module_id NOT NULL)`.
  `StudentConversationListRow.attached_section_id/section_title/section_type` → Optional; `module_id/title`
  always present. **Section-bound behavior unchanged** (8.4 must not regress).
- `retrieve_module_chunks(db, *, student_id, module_id, query_vector, top_k)` — ONE exact pgvector scan over
  the module's published+assigned sections' ACTIVE embedded transcripts (same 4.7 joins as
  `retrieve_section_chunks`, drop the single-section/transcript pins, add `Transcript.lifecycle_state='active'`),
  order by distance, limit top_k. NO ANN index.

### 4. Schemas + service + router
- `schemas.py`: `CreateConversationRequest {kind, module_id?, section_id?, assessment_scope_id?}`; add
  `attached_module_id` + `conversation_kind` to `ConversationRead`; make `ConversationListItem` section fields
  Optional + add `conversation_kind`; derive `grounding_chip` per kind.
- `service.py`: `create_conversation` (kind→binding matrix; 8.6a allows only `homework_help` + `module_id`
  required, `section_id` optional, others 422; module/section visibility; resume-or-create against the natural
  key with `IntegrityError` re-read); `_resolve_owned_conversation` re-checks MODULE visibility for module-bound
  convos; `get_conversation_detail` + `_to_conversation_list_item` + `_compose_answer_basis` handle the
  module-bound (sectionless) case. Immutability: no path reads `conversation_kind` from a body.
- `router`: `POST /student/assistant/conversations` (`createStudentAssistantConversation`), `no-store`.

### 5. Coordinator — `mode_strategies.py` (new) + `generation_service.py`
- `ModeStrategy` protocol → `ModeTurn`; `LectureGroundedStrategy` (today's behavior extracted verbatim);
  `HomeworkStrategy`; `MODE_STRATEGIES` (4 legacy + default → lecture; `homework_help` → homework).
- `generate_assistant_answer_async`: dispatch by kind; SHARED single gateway call + SHARED persist/snapshot.
- Extend `_TurnContext` + `_claim_message` (surface `conversation_kind`, `attached_module_id`); extend
  `_build_snapshot` (merge `snapshot_extra` + `"mode"`).
- `HomeworkStrategy.prepare`: section bound → reuse `_resolve_and_retrieve`; module-only → module visibility +
  `retrieve_module_chunks`, readiness = ≥1 ready section else `context_unavailable`. Compose the homework blob
  (untrusted fence around the marked latest question). `output_schema=AssistantGroundedAnswer`;
  `prompt_key=("homework_help","v1")`; grounding via `decide_grounding`.

### 6. Prompt + provider
- `backend/prompts/homework_help/v1.yaml` (`backend: cerebras` — rule-11-smoke-corrected, guardrail sentinel `HOMEWORK_GUARDRAIL_V1` +
  fence markers, `{{transcript}}`/`{{section_type}}`); add to `CHECKSUMS.json`.
- `provider.py`: extract the v2 grounded fixture into `_assistant_grounded_fixture`; add `homework_help` branch.

### 7. Frontend
Regen client; `api.assistant.createConversation` in `wrapper.ts`; `ensureOpenForMode` in
`AssistantStoreProvider`; `ModePicker.tsx` (homework: module + optional section); `HomeworkStarters.tsx`; wire
the workspace "New chat"; non-editable mode label via `ConversationView` `header`. Widget unchanged.

### 8. Tests
pytest `test_assistant_modes.py` (create matrix, resume-or-create idempotency, immutability, homework turn,
`feature="assistant"`, `retrieve_module_chunks` visibility, 3 guardrail layers); vitest
`ConversationView`/`ModePicker`; Playwright `8.6a-assistant-homework.spec.ts` + `db.mjs` helper; regression via
8.1/8.2 re-run; env-gated rule-11 homework smoke.
