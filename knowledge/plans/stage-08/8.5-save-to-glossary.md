# Plan — Stage 8.5

HOW for [[specs/stage-08/8.5-save-to-glossary]]. Reuse-and-wire; no Stage 7 redesign.

## Backend (8.5a)
1. **Migration 0041** (`backend/alembic/versions/0041_glossary_conversation_source.py`, down_revision
   0040; guarded/re-runnable like 0040): add `source_conversation_id`/`source_message_id` FKs (SET NULL)
   to `glossary_source_references`; drop+recreate the `source_type` CHECK to add `'conversation'`; add
   partial-unique index `uq_glossary_source_references_conversation_message (glossary_entry_id,
   source_message_id) WHERE source_type='conversation'`. Downgrade deletes `'conversation'` rows first.
   Mirror in the ORM model `glossary_source_reference.py` (2 FK cols, widened CHECK, the Index in
   `__table_args__` with the identical predicate). Bump `dev_reseed.EXPECTED_ALEMBIC_VERSION` 0040→0041.
2. **Contract** (`glossary/schemas.py`): `ConversationSaveSource{conversationId,messageId}`;
   `SaveHighlightRequest.module_section_id` optional + `conversation` optional + a `model_validator`
   requiring exactly one. Add the two FK fields to `GlossarySourceReferenceRead`. Regenerate client.
3. **Read model** (`platform/query/assistant_save_source_read.py`): `get_conversation_save_source(...)`
   → `ResolvedConversationSource{section_id, course_module_id, message_role, message_status,
   message_content}` or `None` for not-owned/missing/soft-deleted/unbound/message-not-in-conversation/
   not-visible. Reuses `get_visible_student_section`. No assistant-domain import (rule 8).
4. **Normalizer** (`glossary/snippet.py`): conservative `normalize_for_containment` (strip
   `** __ * _` + backticks, collapse whitespace, casefold) + `selected_text_in_message`.
5. **save_service.py**: `save_term` gains `definition_context`, `source_conversation_id`,
   `source_message_id`; `_resolve_definition` uses `definition_context` (falls back to `selected_text`
   when None — unchanged for summary/manual/quiz). New `_add_source_reference` helper: conversation →
   `pg_insert(...).on_conflict_do_nothing(...)` against the partial-unique index (idempotent); else
   `db.add` (unchanged). Applied on both new-entry and duplicate paths.
6. **service.py**: branch `save_from_highlight` on `payload.conversation` (summary branch untouched);
   `_save_from_conversation` → require_student → selectedText required (422) → read model (None→404) →
   role!=assistant (422) → status!=completed (409) → containment (422) → subject=course_module_id →
   `save_term(... source_type='conversation', selected_text=snippet[:500], definition_context="",
   source_conversation_id, source_message_id)`. New codes in `glossary/policy.py`.
7. **Tests** (`tests/test_glossary_conversation_save.py`): cases a–m (the spec's negative list + the
   empty-context/cache-collapse proofs).

## Frontend (8.5b)
8. **`SaveToGlossary.tsx`**: optional `source?:{conversationId,messageId}` prop; when present call
   `saveHighlight({conversation:source,...})`, else the existing `{moduleSectionId,...}`. A11y unchanged.
9. **`ConversationView.tsx`**: thread `conversationId` + `saveSectionId` → `MessageBubble` →
   `AssistantAnswerBody`; wrap the completed-assistant `SummaryMarkdown` in `<SaveToGlossary>` only when
   both are set (bound). Single mount point covering inline panel + workspace + widget.
10. **Callers**: `AssistantPanel` (sectionId), `AssistantWorkspaceConversation`
    (`detail.attachedSectionId`), `WidgetDrawer` (`detail.attachedSectionId ?? sectionId`) pass the two
    props.

## Gate (8.5b)
`tests/e2e/8.5-assistant-save-to-glossary.spec.ts` (modeled on 8.1 + 7-glossary): open lecture chat →
ask → completed reply → highlight → save → entry+source+definition → studyable + flashcard → duplicate
(idempotent) → negatives (no affordance on own message; unbound 404 via API; cross-student 404 via the
seeded `student2`). Run with the full active suite serially on a clean DB (rule 14).
