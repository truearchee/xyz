## Test strategy
Use the exact Docker test commands from the spec for backend config, health CORS, full backend suite, and frontend type-check. Use Browser automation for the manual browser gate and inspect network responses. Use `rg` for static architecture constraints.

## Risks & mitigations
- Config implementation differs from the user-provided Pydantic example because current code uses a lazy custom settings class -> keep behavior explicit and test it directly.
- Browser polling can overlap if requests hang -> wrap each generated-client call in `Promise.race` with a 5-second timeout and a 2.5-second poll.
- Existing Docker containers may use stale code -> rebuild/restart services before browser verification.
