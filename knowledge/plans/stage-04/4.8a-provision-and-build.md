## Scope confirmation
**Delivers (in-repo only — "you provision, I build"):** dual DB URL + pooler-safe engine; recovery
advisory-lock engine on the direct/session URL; `GET /health/ready` (red until 4.8b); the
fault-injection **fail-loud boot check**; the frontend `output:'standalone'` production image;
`fly.toml` process groups; an executable `scripts/check-staging-env` (incl. the stale-name trap);
`.env.staging.example`; upload body-limit confirmation; embed offline-model-check confirmation. End
state: images build, `/health` up without a DB, `check-staging-env` accepts good / rejects bad,
**`/health/ready` legitimately 503**.

**Does NOT build (copied from the spec):** release migration / extension-bootstrap execution /
first-admin+seed bootstrap / deploy script / runbook rollback (**4.8b**); `/internal/sse-probe`,
`ENABLE_INTERNAL_SSE_PROBE`, adding the staging origin to `CORS_ORIGINS`, the §8 proof harness
(**4.8c**); the hook-free smoke + staging Playwright config + fixture + token-validates assertion
(**4.8d**); `allow_credentials` removal (**4.9**); any `pgcrypto` bootstrap (dropped); any product
feature/UI.

## Approach
Keep the **single code path** that works locally and remotely: one `create_async_engine` call whose
kwargs are computed from the URL (remote pooler ⇒ `statement_cache_size=0` + unique
`prepared_statement_name_func` + `ssl=require` + small pool; local ⇒ today's behavior). Introduce a
**separate direct-engine factory** for the two things that need real session semantics — Alembic
(used in 4.8b) and the reaper's advisory lock — addressed via `DIRECT_DATABASE_URL` (defaulting to
`DATABASE_URL` when unset, so local stays single-URL). Hygiene (fault-flag refusal) is a tiny shared
function imported by both entrypoints, deliberately consistent with the existing call-time guard's
`IS_NON_PROD` predicate. Everything else (frontend standalone, fly.toml, check-staging-env,
.env.staging.example) is additive config that does not alter runtime behavior locally.

## Test strategy
- **/health/ready** — unit/integration: 503 when `alembic_version` missing/≠head or Redis down; 200
  when all green (maps to spec Verification line 1).
- **Fail-loud boot** — parametrized: (flag set, `ENVIRONMENT=staging`) ⇒ `SystemExit`; (same flags,
  `ENVIRONMENT=development`) ⇒ no raise (proves reconciliation #5 / O1); message contains the flag
  name, never a secret.
- **Engine builder** — assert `statement_cache_size=0` + a `prepared_statement_name_func` present for
  a pooler-style URL; local URL unchanged.
- **check-staging-env** — a tiny bats-style/sh harness feeding good + each bad env (localhost API,
  hook set, fault flag, dev Supabase, `DATABASE_URL==DIRECT`, non-pooler `DATABASE_URL`, missing
  `LLM_API_KEY`, missing CORS origin, **stale `K2THINK_API_KEY`/`APP_ENV`/etc. present**) → assert
  exit codes.
- **Frontend** — `tsc --noEmit` clean; `npm run build` emits `.next/standalone`.
- **Images** — backend + frontend build; frontend runtime stage starts `node server.js`.
- **Liveness without DB** — start the backend container with no/wrong DB → `/health` 200,
  `/health/ready` 503.

## Risks & mitigations
- **Connection budget numbers depend on the provisioned Supabase plan `max_connections`.** → Spec
  small bounded pools now; finalize the exact integers against the plan in 4.8b and record the budget
  table in the runbook. `check-staging-env` cannot read `max_connections`; document it instead.
- **Admin-triggered reaper may still use the pooled engine** → explicit audit step (3); if it does,
  give it the direct engine. Note in the report which call sites were changed.
- **`prepared_statement_name_func` uniqueness across forked RQ children** → use a per-connection
  unique scheme (e.g. uuid-suffixed) so forked workers don't collide on the pooler.
- **`output:'standalone'` needs static assets copied** → follow Next's documented standalone layout
  (copy `.next/static` + `public`); verify the built image actually serves pages.
- **Pinned Node digest must match Fly's arch (amd64/arm64)** → pin a multi-arch digest or the arch
  Fly builds for; verify the image runs.
- **Engine `None`-when-unset contract** → keep `engine`/`async_session` `None` when `DATABASE_URL`
  unset so existing import-time behavior and tests are preserved.
