## Scope confirmation
Delivers the full `platform/llm` stack exercised against `DeterministicTestProvider`, the AIRequestLog
+ GeneratedLectureSummary data model (migration 0008), the `ai` queue/worker, the after-embed summary
pipeline, and the projection extension. **Does NOT** add any real K2Think/IFM call, brief/detailed
generation through a real provider, lecturer UI, student visibility, product retry, the stuck-row
sweeper, map-reduce, or streaming (`stream()`/`stream_raw()` raise `NotImplementedError`). Gate 2.A is
GREEN ([[4.5a]]).

## Approach
Mirror existing infra patterns: Protocol-based provider (like `storage/base.py`); typed `Settings`
properties; SQLAlchemy models matching `IngestionJob`/`TranscriptChunk`; idempotent Alembic migration
matching `0007`; RQ queue/worker matching the embedding worker; after-embed handoff matching
chunk→embed in `chunk_service`/`embedding_service`. The gateway is the single orchestrator; the
provider is transport-only; `AIRequestLog.open()` fires before `ContextBuilder.fit()` so
`invalid_input` is loggable (Patch A). Summaries are success-only artifacts with full provenance
(Patch B). Deterministic provider keeps CI fully exercising render→context→limiter→log→validate.

## Test strategy
- Gateway full-path (deterministic) → log written + output validated → maps to "deterministic full-path" done-means.
- AIRequestLog opens before ContextBuilder; invalid_input closes with no providerRequestId → "invalid_input loggable".
- Limiter TTL lease acquire/release + reclaim on simulated death; headroom; TPM-before-RPM.
- Validators: brief bounds; detailed required sections; labNotes for lab; missing section → invalid_output.
- Fault injection: invalid_output retry→terminal; invalid_input non-retryable no summary row; provider_transient 5xx.
- PromptRegistry startup validation + hashing; drift guard fails on unversioned content change.
- Migration 0008 upgrade/downgrade; projection summary steps + summarized + per-step failure.
- Summary handlers: success stores artifact w/ provenance + ai_request_log_id; failure no artifact + failure_category; idempotency.

## Risks & mitigations
- Redis required in tests → limiter tests use fakeredis or a real redis fixture; skip-with-reason if unavailable.
- `summarizing` overallState beyond literal spec word → flagged for confirmation in report/open-questions.
- Migration partial-unique index for two job types → single `WHERE jobType IN (...)` index keyed on
  (transcript_id, job_type) gives per-type uniqueness; tested.
