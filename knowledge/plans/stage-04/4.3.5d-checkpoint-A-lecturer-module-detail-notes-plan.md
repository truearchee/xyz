## Approach
First map the existing frontend route, shell, wrapper, generated DTO, and content component conventions. Then add a thin route under the lecturer app tree and put behavior in feature components under `frontend/src/features/content/lecturer/`.

The page will load section detail from the backend for each section returned by the lecturer section list. The list DTO is known to be thin from Checkpoint 0; using per-section detail avoids backend projection changes and keeps Checkpoint A frontend-only.

Notes save will call a wrapper method that delegates to the generated content client, then the module detail component will re-fetch from the backend before showing the saved notes as persisted.

## Changes, file by file
- `frontend/src/app/(app)/lecturer/modules/[moduleId]/page.tsx` - thin route that renders `LecturerModuleDetail`.
- `frontend/src/features/content/lecturer/LecturerModuleDetail.tsx` - client component that loads real backend sections/details, renders loading/empty/error/unauthorized states, and coordinates re-fetch after notes save.
