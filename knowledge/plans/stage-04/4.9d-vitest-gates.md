## Approach
- **Logic tests** mock at the module boundary: §6.1 mocks `./index` (a real ApiError class so `instanceof`
  holds) + supabase + spies `location.assign`; §6.2 mocks `wrapper.api.transcripts` (via `vi.hoisted`) and
  renders the badge; §6.3 mocks `wrapper.api` + supabase (capturing `onAuthStateChange`) + testHooks. Native
  vitest `expect` (no jest-dom matcher-augmentation risk). The 403-no-redirect case is asserted as a NEGATIVE.
- **Gate scripts** are deterministic node greps (the §4.4 authoritative inline-style/token gates), excluding
  the §5-left dead code (F-4.9-4). `lint` = `next lint` (eslint-config-next); `typecheck`/`build` reuse tsc/next.
- **CI**: GitHub Actions `ci.yml` (frontend required gates + backend pytest service-container job; e2e nightly).
- **Pre-commit**: Husky at the repo root, hook `cd frontend && fast gates`; PROVE it blocks (inject a violation).
- **Branch protection**: handed off (gh absent + interactive auth) with exact `gh api` commands + a failing-merge demo.

## Risks & mitigations
- vi.mock hoisting (factory referencing top-level consts) → `vi.hoisted` (fixed §6.2/§6.3).
- `next lint` surfacing pre-existing errors → fix the one (`module` rename); the inline-style rule is the grep (authoritative).
- gh not installed / remote behind → local enforcement proven; remote enforcement is the developer handoff (not faked).
