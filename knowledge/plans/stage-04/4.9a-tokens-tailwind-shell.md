## Scope confirmation
**Delivers:** Tailwind v4 (via `@tailwindcss/postcss`) + the two-layer token system as the styling
source of truth; self-hosted Space Grotesk + Inter via next/font; the shell infrastructure (portal
mount points, role-aware token-styled chrome, landmarks + skip-link, reserved assistant anchor,
token-styled error/404/403 surfaces); ADRs 044/045/046.
**Does NOT build** (from the spec): any public component or the Toast/Modal *component* logic (only
their portal roots); any feature-surface restyle or `style={}` removal (4.9c); Vitest/gates/CI (4.9d);
hygiene batch (4.9e); dark mode/second theme; monospace font; the assistant widget (anchor only); any
backend change; `upload.ts` and the `next.config.ts` webpack stub are untouched.
**No ambiguity blocking start** — design direction + validated tokens are locked in design-plan.md.

## Approach
- **v4 over v3 (ADR-044, REPO FIT):** no existing CSS/PostCSS to fight; v4's CSS-first `@theme` makes
  "CSS variable == Tailwind token" one object and lets us reset *only* the color namespace. Use
  `@tailwindcss/postcss` (not the Next/Turbopack plugin) so dev/build/standalone behave identically and
  the existing `next.config.ts` webpack hook is left alone. If v4 cannot build cleanly here, that is a
  **finding** → fall back to v3 (don't force it).
- **Two layers (ADR-045):** raw `--palette-*` live in `:root` (never emit utilities); semantic
  `--color-*` live in `@theme` referencing the raw layer; `--color-*: initial` makes "components use
  semantic tokens only" a *compile-impossible-to-violate* rule. Spacing/font/radius/shadow namespaces
  are NOT reset (so their scales survive).
- **Fonts:** `next/font/google` self-hosts Space Grotesk + Inter (browser makes no external request).
  Build-time/dev-compile fetches from Google once and caches — see Risk R-font (local-font fallback).
- **Shell split:** 4.9a ships the portal *mount points* + chrome; 4.9b builds the Toast/Modal/EmptyState
  *components* that render into them and refactors the error surfaces onto EmptyState.

## Risks & mitigations
- **R-font (next/font dev-image fetch):** `next dev` fetches Google fonts on first compile inside the
  frontend container; if that egress is blocked the dev image errors. → Mitigation: prefer
  `next/font/google` (the container already reaches Supabase/API); if it proves flaky, switch to
  `next/font/local` with committed `.woff2` (fully offline) — record the choice in ADR-044.
- **R1 purge footgun:** dynamic class strings vanish in `next build`. → 4.9a keeps classes literal; the
  `build` gate guards it (the literal-class discipline is enforced hard in 4.9b's `variants.ts`).
- **R3 React-19 peer deps:** verify `react-aria-components`/`lucide-react` install clean on 19.1.0; if a
  peer range fights, pin a compatible version (don't `--force`) and note it.
- **R8 image rebuild:** rebuild the baked frontend image before the E2E smoke or it tests stale code.
- **Shell E2E regression:** AppShell is on the `4.3.5b`/`4.3.5c` path. → preserve selectors byte-for-byte;
  re-run those specs in 4.9a (don't wait for 4.9c); a broken selector is a rule-10 stop, not a loosen.
- **Empty-State sequencing:** error/404/403 ship token-styled now, refactor onto EmptyState in 4.9b —
  tracked, not a silent gap.
