## Approach (the load-bearing decisions)
- **className conversion, DOM/attributes byte-preserved.** On E2E-dense surfaces, convert `style={}` →
  `className` with semantic tokens and keep the EXACT element/`data-testid`/`role`/`aria-live`/label.
  Swap in 4.9b components (Button/Input/Card/Badge/EmptyState/StepProgress) ONLY where they don't touch
  a hook (e.g. a fresh Card wrapper, an empty state with no testid). This is the lowest-risk path and
  honors the rollback stance.
- **Hex → semantic token mapping** (the recon found ~44 hexes; consolidated):
  ```
  #111827 text-text · #4b5563/#374151 text-text-muted
  #d7dde8/#cbd5e1/#e5e7eb/#e2e8f0 border-border · (input boundary) border-border-strong
  #ecfdf5/#a7f3d0/#047857 success(-surface/-/-text) · #14532d/#9ad4aa success too
  #fef2f2/#fecaca/#7f1d1d/#f0b4b4 danger(-surface/-/-text) · #b42318 danger-text
  #eff6ff/#bfdbfe/#1d4ed8 info · #eef2ff/#c7d2fe/#3730a3 info(draft) → info-surface/-/-text
  #fed7aa/#7c2d12 warning(-surface/-text) · #f3f4f6/#d1d5db neutral (surface-muted/border)
  #174a63/#ffffff primary/on-primary · #6b7280/#94a3b8 text-muted · #2563eb primary (highlight)
  #f8fafc surface-muted · #334155 text · #f7faf9/#cfe1da/#1f2937 notes → surface-muted/border/text
  ```
  (Lecturer/student "active vs inactive" badges → Badge `success`/`neutral` tones.)
- **Slices, each full-suite-gated + its own commit** (per-session-commit revertibility; rollback = revert
  the surface, never loosen the spec):
  1. login + `TranscriptStatusBadge` + `TranscriptSummaryPanel` (named 4.5d/4.6 panels). **DONE — 11/11.**
  2. lecturer: `LecturerModuleDetail` + `SectionTranscriptControl`/`SectionUploadControl`/`SectionAssetList`/`SectionAssetRow`/`SectionNotesEditor`/`SectionPublishControl`.
  3. student: `StudentModuleDetail`/`StudentSectionDetail`/`StudentSectionView`/`StudentAssetRow`/`SummaryMarkdown`.
  4. admin: `shared.ts` (panelStyles → Card/Input/Table) + `users/*` + `modules/*`; + `ModuleListView`. Final 4.9c checkpoint here.

## Test strategy (per slice)
- `tsc` + `next build` clean; inline-style grep + token-purity grep clean over the slice's files.
- Full active suite (3-part orchestration, `--workers=1`) green — the gate.
- No-loosen proof: `git diff tests/` empty (no spec touched) unless a hook genuinely moved (then update its
  spec in the same commit + call it out per-surface in the report).

## Risks & mitigations
- **Specificity trap / half-migration:** removing inline styles fully per surface; inline-style grep is the proof.
- **Hook drift:** preserve byte-for-byte; the full suite + the `git diff tests/` empty check catch it; revert-on-red.
- **login uncertainty (4.3.5b assertions I can't fully see):** verified by the suite — slice 1 came back 11/11, so the Card/Input/Button restyle preserved getByLabel + button names.
- **No-behavior creep:** a design-plan state the thin v1 skipped = finding, logged, not built.
