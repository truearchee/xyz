## Scope confirmation
Implement only the student module detail page, published-only rendering from the backend student read path, and signed URL open behavior. Do not add lecturer feature work except smoke-data preparation through existing UI, student mutation controls, backend changes, raw storage URL construction, final Stage 3 gate work, or Stage 3 FULLY VERIFIED status.

## Implementation approach
1. Run the required pre-checks and generated-client freshness check.
2. Map the student route conventions and existing student components.
3. Confirm backend `GET /modules/{moduleId}/sections` filters student responses server-side to published sections.
4. Confirm generated/wrapper methods for section list/detail and signed URL.
5. Make `/student` assigned-module cards link to `/student/modules/{moduleId}`.
6. Add `/student/modules/[moduleId]/page.tsx`.
7. Add student detail components:
   - `StudentModuleDetail.tsx`
   - `StudentSectionView.tsx`
   - `StudentAssetRow.tsx`
8. The student detail component loads module metadata, section list, and per-section student detail from wrapper methods. It does not fetch all lecturer sections or frontend-filter forbidden sections.
9. Asset open buttons call `api.content.getAssetDownloadUrl` and open the returned URL. They do not construct storage URLs or expose storage keys.
10. Render loading, empty, forbidden, and `role="alert"` error states.
11. Rebuild/restart the frontend container before Docker-backed checks.
12. Run verification commands and browser smoke with server-response proof.
13. Update report/findings/status/architecture/log and commit.

## Verification plan
- `docker compose exec frontend npx tsc --noEmit`
- `docker compose exec frontend npx next build`
- Direct-fetch scan must remain empty outside generated core and `upload.ts`.
- JWT/session-role scan must remain empty.
- Backend diff guard must remain empty.
- Generated-client freshness must have no generated core/model/service/index drift.
- Browser smoke must prove:
  - published section present in student server response and UI
  - draft/unpublished section absent from student server response and UI
  - lecturer notes visible
  - asset filename visible
  - Open file requests backend signed URL and opens/downloads PDF
  - no student mutation controls

## Risk controls
- Treat backend response as the authority for student visibility; do not frontend-filter full lecturer data.
- Do not reuse lecturer components that contain mutation controls.
- Use wrapper/generated client for signed URL. Do not build storage URLs manually.
- Preserve 403 session behavior through wrapper errors.
