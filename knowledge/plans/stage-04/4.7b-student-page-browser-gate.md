## Scope confirmation
Delivers the **thin student UI** + the **browser gate G1–G9**. Student lecture/lab detail renders
title/notes/materials immediately and brief + detailed summaries **independently**, each in one of four
per-slot states (LOADING/GENERATING/UNAVAILABLE/NOT_APPLICABLE) + READ_ERROR, with bounded polling
(reused 4.5d backoff, generating-only, unmount cleanup, wall-clock cap). Summary markdown is rendered
**sanitized (raw HTML disabled)**. Consumes the generated client from 4.7a.

**Requires P1 (Stage 3 restore) green** before the browser gate runs. **Do NOT build:** anything in
spec §3.3 exclusions; no lecturer controls (Replace/Retry/upload) on the student page.

## Approach
Reuse the lecturer summary display patterns (`TranscriptSummaryPanel` rendering, `DetailedSummaryView`,
backoff polling from `TranscriptStatusBadge`) but driven by the new student endpoints and stripped of all
lecturer-only affordances. Add `react-markdown` (raw HTML off by default; deliberately NOT adding
`rehype-raw`) for S1. The page never blocks on summary readiness.

## Changes, file by file
- `frontend/src/features/content/student/StudentSectionView.tsx` (or a new sibling) — add brief +
  detailed summary blocks with per-slot state rendering + sanitized markdown.
- `frontend/src/features/content/student/StudentModuleDetail.tsx` — surface the coarse `summaries_state`
  per section (no fan-out).
- `frontend/src/features/content/student/useStudentSummaryPolling.ts` (new) — bounded backoff poll of the
  summaries sub-resource, generating-only, unmount-safe, capped.
- `frontend/src/features/content/student/SummaryMarkdown.tsx` (new) — `react-markdown` wrapper, raw HTML
  disabled, unsafe protocols blocked.
- `frontend/package.json` — add `react-markdown`.
- Consume regenerated `frontend/src/lib/api/*` via `wrapper.ts` (401/403/404 handling reused).
- `tests/e2e/4.7-student-summaries.spec.ts` (new) — G1–G7 gate.
- `tests/e2e/4.7-stage3-content-visibility.spec.ts` — the P1 restore (authored in pre-flight; G8).
- `tests/e2e/fixtures/{seed,db,teardown}.mjs` — §14 seeded fixtures (courses A/B, A1–A5, sentinel,
  inactive-membership variant) + teardown manifest entries.

## Order of operations
1. Confirm P1 green (else HALT 4.7b per circuit breaker; do not run/claim the browser gate).
2. Add react-markdown; SummaryMarkdown sanitized render.
3. Student page summary blocks + polling hook; `tsc --noEmit` green.
4. Seed fixtures for §14; author 4.7-student-summaries.spec.ts (G1–G7).
5. Run 4.7 gate; then full active suite (rule 14) incl. restored Stage 3 (G8) and fault specs (--grep).

## Test strategy
- G1 read brief+detailed (row1); G2 brief-first seeded (row2); G3 raw transcript unreachable (UI +
  sentinel + API/no-signed-URL); G4 unpublished→404; G5 unenrolled→404 byte-identical, inactive same;
  G6 non-student→403; G7 UNAVAILABLE seeded (row4); G8 restored Stage 3; G9 full suite green.
- Security gates G3/G4/G5/G6 are honest-or-red — never mocked/weakened/skipped.

## Risks & mitigations
- **Sanitization regression** → react-markdown default (no rehype-raw); a G3-adjacent assertion that a
  seeded `<script>`/`javascript:` payload does not execute/render as markup.
- **Polling leaks** → unmount clears timers; assert no infinite poll on terminal states.
- **Timing flakiness** → all G2/G7/supersession states SEEDED, not raced (§14).
