## Scope confirmation
Delivers the active-summary preview endpoint, the lecturer replace/retry/per-step UI on the 4.5d surface,
the 4.6d browser-gate spec, and the cross-stage e2e fixes (4.3.5e 409→pending, db.mjs lifecycle_state).

## Approach
The contract was already regenerated in 4.6a/b (`retrySectionTranscriptProcessing`, `failureCategory`,
`retryable`, `lifecycleState`), so the only backend gap is the lecturer **preview endpoint over
`ActiveTranscriptSummaryResolver`** (deferred from 4.6a). The UI is additive on the existing 4.5d
components (no modal — inline confirm, house style). The browser gate drives the real lecturer UI; the
retry flow clears the forced fault by **recreating the embedding_worker** mid-run (global env can't mix).
Fencing stays a deterministic pytest (a backend race, not a UI click).

## Order of operations (executed)
1. Preview endpoint (schema + service + route) + backend tests → regen client.
2. Wrapper methods + lecturer UI (Replace + inline confirm + double-replacement warning + pending badge +
   Retry + per-step strip + sanitized reason).
3. Fencing pytest (stale worker aborts after reaper+retry).
4. e2e spec (4.6d-replace-retry) + the 4.3.5e/fixtures fixes.
5. Verify: backend pytest, frontend tsc, client regen, `npx playwright test --list`.

## Test strategy
Backend: 3 preview-endpoint tests (eligible flags, hasPendingReplacement, authz) + 1 fencing test.
Frontend: `tsc --noEmit`. E2E: the 4.6d gate (retry + replacement continuity) + edited 4.3.5e compile via
`--list`; the LIVE run is the developer's acceptance step.

## Risks & mitigations
- **Rule-14 cross-stage breaks** (4.6a 409→pending; 4.6b summaries parallel to embed): fix 4.3.5e + the
  `is_active` fixtures now; the live run may surface more timing-sensitive assertions to fix.
- **Fault clear** via `execSync docker compose` from the Playwright runner; fall back to a two-invocation
  driver if docker access is unavailable.
- **Deterministic provider = identical v1/v2 text** → assert on `activeTranscriptId` + DB lineage + eligibility.
