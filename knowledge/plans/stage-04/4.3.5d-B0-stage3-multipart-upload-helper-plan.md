## Scope confirmation
Implement only the centralized browser multipart helper at `frontend/src/lib/api/upload.ts`. Do not add upload UI, replace UI, publish/unpublish controls, student UI, backend changes, generated client edits, auth architecture changes, or Stage 3 FULLY VERIFIED status.

## Approach
Run the baseline and generated-client freshness checks first. Then map the backend content router, content service, generated `ContentService`, generated DTOs, API core error class, and existing wrapper auth behavior.

Implement `uploadSectionAsset` and `replaceSectionAsset` with `FormData`, the backend-confirmed field name, `Authorization: Bearer <current Supabase access token>`, configured API base URL, typed `SectionAssetResponse` return values, and wrapper-aligned 401/403 behavior.

To avoid broad auth refactors, keep the helper self-contained and import existing `AuthRequiredError` and `ForbiddenError` from `frontend/src/lib/api/wrapper.ts`. The helper may duplicate the small 401 sign-out/redirect action if the wrapper does not expose it.

## Verification strategy
- `docker compose exec frontend npx tsc --noEmit` verifies TypeScript.
- `docker compose exec frontend npx next build` catches Next route/build issues.
- Direct-fetch scan must only find generated API core and `upload.ts`.
- JWT-role scan must remain empty.
- Backend diff guard must stay empty.
- Generated client freshness must show no generated model/service/core/index drift.
- Final `git status --short` must be clean after commit.

## Risks & mitigations
- Generated client already has multipart methods but may set up fetch differently than browser uploads need. Mitigation: use explicit `FormData` and do not set `Content-Type`.
- 401/403 behavior could drift from wrapper. Mitigation: reuse exported wrapper error classes and match sign-out/redirect behavior for 401 while preserving session for 403.
- Next/frontend container may run stale copied source. Mitigation: rebuild/restart frontend before containerized verification if needed.
- Accidentally adding product UI. Mitigation: restrict source edits to `frontend/src/lib/api/upload.ts` unless contract mapping proves a tiny shared API utility is necessary.
