## Approach
The smoke is the inverse of the local suite: **no `__xyzE2E`, real Supabase logins, real visibility
rules**. The design centers on the §1 state machine — the negative controls are written FIRST so the
spec cannot pass on weak auth: a non-member 404, a member-but-draft 404 byte-identical to it, then a
member-and-published 200. Auth is a real Supabase password sign-in per role in its own browser context;
the student's own token drives both the UI assertions and the direct-API negative controls (no shared
context, no privileged token). The fixture is a synthetic lecture rich enough that the REAL detailed
summary passes the 6-section validator. The existing `4.7-student-summaries.spec.ts` is the reference
for WHAT to assert; this re-expresses it hook-free against staging.

## Order of operations
1. `playwright.staging.config.ts` + the hook-free `login.ts` helper.
2. The fixture VTT (rich enough for the validator + the sentinel).
3. The spec — **negative controls first** (S0→NC1→S1→S2→NC2+equality), then S3/S4/NC3/NC4, then §2/§3.
4. Local: type-check the staging config; parse the fixture; **run the full active suite 11/11** (rule 14).
5. Hand off to the developer: run `playwright.staging.config.ts` against staging; return NC results,

6. On a green hosted run: close-out (roadmap/STATUS/log; FULLY VERIFIED).

## Test strategy
- **§1 is the gate** — the NCs are the assertions that make it real: NC1 (non-member→404), NC2
  (member+draft→404) **with `expect(nc1.body).toEqual(nc2.body)`** (existence-leak proof), NC3
  (member+published→200, both summaries rendered), NC4 (sentinel phrase absent from every student
  response + no transcript affordance). Separate `browser.newContext()` per role.
- **§2** — assert pipeline terminal via the status endpoint; the detailed summary's structure; both
  model IDs from the AI provenance; `/health` + `/health/ready`; `window.__xyzE2E === undefined`;
  `/internal/sse-probe` chunks arrive over time (timestamps increasing), not one flush.
- **§3** — bounded backoff (≤10 s, ≤15 min); post-smoke `overallState` terminal + both summary rows.
- **Local regression** — full active suite 11/11 (the deferred-to-here guard from 4.8a–c).

## Risks & mitigations
- **Hook-free auth** (the core mechanism) → `supabase-js signInWithPassword`; if a UI-only login is
  required, drive the real `/login` form. Either way no `__xyzE2E` (O1).
- **Fixture too thin → detailed validator fails** (failure #15) → author the VTT against the six
  section headers; dry-run the validator locally against a deterministic detailed summary before the
  hosted run.
- **Pipeline slower than 15 min on cold workers** → keep workers + PG warm for the window (§7.D); the
  budget failure is a real finding, not a flake to ignore.
- **NC timing** (asserting draft-404 before publish) → drive the state machine strictly in order;
  the student fetches are explicit API calls at each step, not racing the pipeline.
- **Real-provider cost / rule 11** → exactly one real run; both routes' model IDs captured on the
  first pass; no second call.
