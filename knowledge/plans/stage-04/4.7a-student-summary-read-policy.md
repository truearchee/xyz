**Conclusions:** (1) No endpoint serves raw transcript/segment/chunk **text** ‚Äî segment/chunk text is
DB-only, surfaced solely as counts. (2) No signed-URL path mints a **raw transcript object** ‚Äî only
uploaded section assets, gated to published sections. (3) Every transcript endpoint already returns
**403 for students**. G3(c)/S4(b)(c) are satisfied today; 4.7's new student endpoints must add neither a
text-bearing field (S4a, ¬ß8.3 schema hygiene) nor a transcript signed-URL path.

## Approach
A new isolated domain `domains/student_summaries/` owns the **policy** (the security decision) and the
**precedence** pure function; `platform/query/student_summary_read.py` owns the **scoped reads** (WHERE-clause
enforced visibility, no policy logic ‚Äî rule 8). A new `api/routers/student_summaries.py` exposes the two
Option-B endpoints + a student module-list surface, with a student-only role dependency and ¬ß8.3
response models that are structurally incapable of leaking transcript text/provenance/job internals.
The detailed structured summary JSON is flattened to a **markdown string server-side** so the frontend
only sanitize-renders. Reuse: `is_summary_eligible`, `get_latest_transcript_summaries`,
`get_transcript_processing_status_read`, the lifecycle active-transcript resolution.
