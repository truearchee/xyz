## Scope confirmation
Repaint the shipped violet frontend to the v2 **monochrome Apple×Linear** identity — a **token + shape** change,
not a component-API change. **Does NOT:** add/rename props or variants (beyond the pill shape), add components or
screens, add dark mode, change backend / DB / OpenAPI, or reach into `--palette-*` from feature code. The
pipeline failed state stays danger-red + explicit "Failed". **Scope extended** (spec Amendment 2026-06-15) to
sweep `frontend/src/features/**` for the same token/class treatment so the app is monochrome end-to-end;
`features/modules/ModuleDetailView.tsx` stays untouched.

## Approach
The two-layer token model means most of the repaint is **data**: rewrite Layer 1 raws + Layer 2 semantic
mappings in `globals.css`, and every component/surface that already consumes semantic tokens recolours for free.
The non-automatic parts are **shape** (button pill + press scale, Modal radius, resting-shadow removal),
**typography** (single system family + 17/19px scale), and the **feature-surface ad-hoc class strings** (pills /
shadow / weight / reading size). The contrast gate is hardcoded to the old palette keys, so it is rewritten to
the new semantic pairs (same math + thresholds). Exact hexes from design-plan §1.2 (consistent with
design-system §2).

## Changes, file by file
- `frontend/src/app/globals.css` — Layer 1 warm-gray ramp + green/amber/red status raws (replace
  violet/zinc/rose/indigo); Layer 2 remap (graphite `primary`/`focus-ring`/`info`, parchment `surface-muted`,
  add `text-subtle`=gray-400, `danger-hover`=red-700, gray-800 `overlay` scrim); retune `--text-*`
  (body 17 / reading 19 + per-size line-height + tracking); radius 8/12/16 + add `--radius-xl: 20`; delete
  `--shadow-sm`. Keep `--z-*`, `--motion-*`, reduced-motion guard.
- `frontend/src/app/layout.tsx` — remove `next/font/local` + both font consts + `<html>` variable classes;
  `<body>` → `bg-surface-muted` (parchment page).
- `frontend/src/components/ui/variants.ts` — `buttonBase` `rounded-md` → `rounded-full` + `active:scale-[0.96]`
  + transition includes transform.
- `frontend/src/components/ui/Card.tsx` — drop `shadow-sm` from `cardBase`.
- `frontend/src/components/ui/Modal.tsx` — `rounded-lg` → `rounded-xl`.
- `frontend/scripts/check-contrast.mjs` — rewrite `pairs` to the monochrome semantic pairs + gray-800 scrim.
- `frontend/src/features/**` — pattern sweep: ad-hoc buttons `rounded-md font-bold` → `rounded-full font-medium`;
  remove resting `shadow-sm`; heading `font-bold` → `font-semibold`; student summary body → `text-lg leading-[1.65]`.
- delete `frontend/src/fonts/inter-latin-wght-normal.woff2` + `space-grotesk-latin-wght-normal.woff2`.
- Close-out docs: `knowledge/steps/stage-04/4.9f-monochrome-restyle.md`, `design-system.md` (banner removed +
  reconciled), `design-plan.md` (supersession → shipped), `STATUS.md`, `log.md`.

## Order of operations
1. Token layer (`globals.css`) + font swap (`layout.tsx`, delete woff2) + parchment body.
2. Rewrite `check-contrast.mjs`; run it → fix tokens until all PASS.
3. Components/ui shape (pill + press, Card shadow, Modal radius).
4. Feature-surface sweep (pills / shadow / weight / reading).
5. Verification battery (gates, typecheck, vitest, E2E, mobile, visual).
6. Reconcile docs (design-system banner removal + value reconcile; design-plan supersession) + report + STATUS + log.

## Test strategy
- AA: `node scripts/check-contrast.mjs` all PASS (incl. the gray-500/gray-100 ≈4.6:1 tight pair + composited scrim).
- No off-token colour: `npm run check:design-tokens`; no stray inline styles: `npm run check:inline-styles`.
- Type safety: `npm run typecheck`. Component/a11y contracts: `npm run test:unit` + `npm run test:a11y`.
- Behaviour unchanged: full active Playwright suite via the docker e2e overlay (`-f docker-compose.e2e.yml`,
  `--workers=1`, baked-image rebuild + route warming).
- Visual: re-capture the design-match screenshot set @ desktop + 375px mobile; compare to design-plan §2.0–§2.3.

## Risks & mitigations
- Contrast tightness (gray-500 on gray-100) → the gate is the arbiter; fix the token, never the threshold.
- Type-scale ripple (text-base→17, text-lg→19) → headings set explicit `leading-*`; confirm in the visual pass.
- E2E stack friction (baked images, e2e overlay, cold next-dev compile) → rebuild images + curl-warm routes.
- Scope creep → token/class only; no API change; ModuleDetailView untouched.
