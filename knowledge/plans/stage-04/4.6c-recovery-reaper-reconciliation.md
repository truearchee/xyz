## Scope confirmation
Delivers the stuck-row reaper, loss-safe storage reconciliation, the MaintenanceRun observability table,
a singleton advisory lock, storage `list_objects`, admin trigger endpoints, and a worker-startup hook.
Does NOT deliver lecturer UI / preview endpoint / browser gate (4.6d), student surface (4.7), cron
(11.1), heartbeat columns, or a MaintenanceRun UI.

Decisions locked: liveness = RQ-registry + age (no heartbeat); reaper at startup, reconciliation admin-only.

## Approach
Two idempotent units under a Postgres advisory lock held on a DEDICATED connection (so it survives the
work's commits and is always explicitly released). The reaper scans for never-enqueued parse / stuck
queued / stuck running rows using per-step age thresholds, confirms liveness via stable RQ job_ids for
embed/summary (age-only for parse/chunk), re-enqueues or marks `crashed` (fenced re-read FOR UPDATE).
Reconciliation lists the bucket (recursive, scoped to `…/transcripts/…`, capped), diffs against
`SELECT storage_key FROM transcripts` (all lifecycle states retained), reports orphans/missing,
deletes only in cleanup mode + flag (capped). Every run writes a MaintenanceRun. Liveness, time, storage,
and session_factory/engine are injected so everything is testable without Redis/Supabase.

## Test strategy

liveness injected; FakeStorageProvider gains `list_objects` + per-key created_at.

## Risks & mitigations
- **Advisory lock vs pool**: hold the lock on a dedicated `engine.connect()` and always `pg_advisory_unlock`
  before it returns to the pool (else it leaks).
- **parse/chunk liveness is age-only** (no stable RQ id) → re-enqueue is idempotent (claim-dedup); crash only
  on confident-not-live.
- **Supabase list is non-recursive** → recurse + paginate, capped; missing-ref detection skipped when capped.
- **Loss-safe** → report-only default; cleanup double-gated (flag + mode) + capped; missing never auto-fixed;
  superseded retained → referenced.
