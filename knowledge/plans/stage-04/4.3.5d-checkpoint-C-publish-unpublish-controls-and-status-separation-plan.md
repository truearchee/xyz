## Scope confirmation
Implement only lecturer-side publish/unpublish controls and status separation on the existing lecturer module detail page. Do not add student UI, signed URL opening, backend changes, section management controls, full Stage 3 browser gate work, transcript/AI work, or Stage 3 FULLY VERIFIED status.

## Implementation approach
1. Run the required pre-checks and generated-client freshness check.
2. Confirm generated `ContentService` publish/unpublish methods and existing wrapper exposure.
3. If `api.content.unpublishSection` is missing, add it through the wrapper.
4. Add `SectionPublishControl.tsx` for section-level control, pending state, visible status text, and mutation error display.
5. Wire `SectionPublishControl` into `LecturerModuleDetail.tsx`.
6. Publish/unpublish handlers call wrapper methods and then call the existing `loadModule()` re-fetch path.
7. Keep asset `processingStatus` rendering in `SectionAssetRow.tsx` untouched and separate from section `publishStatus`.
8. Rebuild/restart the frontend container before Docker-backed checks.
9. Run verification commands and browser smoke.
10. Update report/findings/status/architecture/log and commit.

## Verification plan
- `docker compose exec frontend npx tsc --noEmit`
- `docker compose exec frontend npx next build`
- Direct-fetch scan must remain empty outside generated core and `upload.ts`.
- JWT/session-role scan must remain empty.
- Backend diff guard must remain empty.
- Generated-client freshness must have no generated core/model/service/index drift. If `wrapper.ts` changes, treat it as intentional handwritten wrapper work.
- Browser smoke must prove draft + completed visible together, published + completed after publish, and unpublished/draft + completed after unpublish.

## Risk controls
- Do not mutate local status and stop; always re-fetch backend data after publish/unpublish.
- Keep status ownership clear: section `publishStatus` in the section control/header, asset `processingStatus` in each asset row.
- Do not introduce publish/unpublish into the upload helper.
- Do not add signed URL or student work.

## Open questions
None. Use the backend-returned unpublish status label instead of assuming whether the backend returns `draft` or `unpublished`.
