## Goal
Ship Stage 11.2: student detail recommendations for lecturers and passive student nudges, with a deterministic
recommendation contract first and lazy cached AI phrasing as the final layer.

## Why now
11.1 established `AgentRun`, risk snapshots, and canonical `RiskReason` output. 11.2 turns those deterministic
risk reasons into reviewable support workflows without recalculating risk or letting AI decide eligibility,
severity, grade risk, peer comparisons, or facts.

## Dependency
- 11.1 is FULLY VERIFIED and is the source of truth for deterministic risk inputs.
- 11.2 must reuse the latest 11.1 `student_risk_snapshots.risk_reasons` / `RiskReason` output as the trigger
  for recommendations.
- 11.2 must not recompute risk a second way and must not read Stage 10 data.

## Build
- Add a `Recommendation` persistence contract with:
  - separate `lecturerState` and `studentState`;
  - one active recommendation per `(student, reasonCode, target)`, where `target` is a canonical scoped key;
  - deterministic payload, source snapshot/run references, `algorithmVersion`, `inputHash`, `sourceCutoffAt`;
  - AI text/provenance fields for lecturer and student copy.
- Add a deterministic recommendation sync that consumes existing risk snapshots and `RiskReason` objects only.
- Render no-AI template copy first; this path must be independently useful and testable.
- Add lazy cached AI phrasing through `platform/llm`, one BACKGROUND-priority call per recommendation, with
  prompt provenance and template fallback.
- Add two backend output validators with adversarial tests:
  - numeric consistency: AI text invents no numbers, peer comparisons, diagnoses, new risk reasons, or facts
    absent from `deterministicPayload`;
  - student-copy safety: student-facing copy only, with banned ideas regenerating once and then falling back to
    the deterministic template.
- Add lecturer student-detail recommendation UI:
  - reasons, cited deterministic metrics, draft text, student nudge preview;
  - actions are exactly Copy draft, Mark acted, and Dismiss;
  - no Send button ever.
- Add student nudge UI:
  - primary placement in My Progress `Where you stand`;
  - secondary dashboard banner with at most one active nudge;
  - dismissible `Not now`;
  - never modal, push, email, or toast.

## Recommendation contract
- `reasonCode` and `target` come from 11.1 `RiskReason` output.
- `target` must be canonical and scoped, for example `module:<moduleId>` or
  `topic:<moduleId>:<normalizedTopicTitle>`.
- `deterministicPayload` contains only snapshot-derived reasons, displayed metrics, canonical `allowedNumbers`,
  canonical `allowedFactPhrases`, allowed reason codes, allowed metric keys, target labels, and template text.
- `lecturerState` and `studentState` are independent. Lecturer dismiss or mark-acted never hides the student
  nudge; student `Not now` never hides the lecturer draft.
- A recommendation is visible only while its triggering deterministic reason still holds against current data.
  Persisted rows keep state, AI text cache, and provenance; visibility revalidates live-on-read so a cleared
  problem hides immediately, then the next deterministic sync can close the row.
- Once a recommendation has been dismissed for an audience, that audience must not see the same
  `(student, reasonCode, target)` again.

## AI contract
- AI is lazy, cached, asynchronous, and last-layer only.
- AI runs through `platform/llm`, PromptRegistry, Redis limiter, and `AIRequestLog`.
- The job uses BACKGROUND priority and makes one call per recommendation to produce both lecturer and student copy.
- Pages render deterministic/template text immediately; AI never blocks the initial page render.
- If the AI result is missing, pending, invalid, unsafe, or unavailable, the UI ships the deterministic template.
- If either lecturer or student AI text fails validation, reject the whole AI result; never persist or serve partial
  AI copy.
- AI provenance is `{ modelId, promptVersion, inputHash, generatedAt }`; regenerate only when `inputHash` or
  prompt version changes.
- AI never calculates risk or grades; it only phrases deterministic output.

## API changes
- Lecturer endpoints for owned modules/students to read recommendation detail, ensure lazy AI copy, mark acted,
  and dismiss.
- Student endpoints to read their own active module nudge, read their dashboard banner candidate, and dismiss
  with `Not now`.
- All endpoints must enforce lecturer-owned-student and student-self boundaries.
