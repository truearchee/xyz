## Goal
Ship a student-facing `.ics` download that exports the student's active workload plan as a valid snapshot calendar
file with stable event UIDs and timezone-safe dates/times.

## Why now
11.4 now persists active `WorkloadPlan` and `WorkloadPlanItem` rows with stored estimates, reasons, and scheduled
windows. 11.5 consumes that persisted plan without recalculating it, giving students a portable calendar snapshot
before the final 11.6 AI advice session.

## Build
- Add a student-facing `.ics` download for the student's active workload plan.
- Export scheduled `WorkloadPlanItem` rows as `VEVENT` entries:
  - stable `UID` per `WorkloadPlanItem`;
  - `DTSTAMP` equals export generation time;
  - `DTSTART` and `DTEND` equal the stored plan-item scheduled window;
  - `SUMMARY` is `Study: [topic]`;
  - `DESCRIPTION` includes reason and stored estimate;
  - `PRODID` is `XYZ LMS`.
- Export module deadlines from active/published module sections with `due_at` as timed deadline marker events.
  This session chooses timed deadlines, not all-day deadlines, so due times are preserved.
- Handle timezone explicitly by normalizing exported datetimes to UTC `Z` values while preserving the module or
  institution timezone as calendar metadata for verification. The exported instants must convert back to the
  expected institution-local time with no one-hour shift.
- Expose the export from the student workload planner UI as a file download.
- Make clear in the UI/download naming that the export is a snapshot and does not auto-update.
- Keep all export logic deterministic except `DTSTAMP`, filename timestamp if used, and HTTP response generation
  time.

## API changes
- Add a student-only calendar download endpoint for an active workload plan, expected shape:
  `GET /student/workload/plans/{plan_id}/calendar.ics`.
- The endpoint returns `text/calendar; charset=utf-8` with `Content-Disposition: attachment`.
- The endpoint exports only active plans owned by the caller.
- The UI obtains `plan.id` from the existing active plan read and downloads that specific plan.

## Worker / job changes
None.

## Authz rules
- Student role only.
- The plan must belong to `current_user.user_id`.
- The plan must be active.
- Another student's plan export returns 403.
- Lecturer/admin/unassigned users return 403.
- Missing or inactive plans do not create a new plan and do not export another student's data.

## Verification
- Backend unit tests:
  - generated text is valid iCalendar shape with `BEGIN:VCALENDAR`, `VERSION:2.0`, `PRODID:XYZ LMS`, `VEVENT`,

  - plan item events use stored windows, stored estimate, and stored reason;
  - unscheduled tight residual items are skipped or represented only if a valid event time exists, with behavior
    documented in tests;
  - deadline events are timed and preserve `due_at`;
  - line escaping/folding handles commas, semicolons, backslashes, and newlines;
  - UTC-normalized export converts back to the expected module/institution local time across a timezone edge case;
  - re-exporting the same plan keeps the same UIDs for plan items and deadlines while allowing `DTSTAMP` to change.
- Backend API tests:
  - owner student can download their active plan;
  - another same-module student gets 403 for the first student's plan id;
  - lecturer/admin/unassigned users get 403;
