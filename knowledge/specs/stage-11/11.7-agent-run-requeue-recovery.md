## Goal
Fix the Stage 11.1 scheduler/manual trigger handoff so committed `queued` or `failed` `AgentRun`s are safely re-enqueueable when the original RQ enqueue failed, without duplicating jobs already present in RQ.

## Why now
Two reviews found the same critical bug: an `AgentRun` can commit before RQ enqueue, and retry paths only enqueue when the row was newly created. That strands the run forever in `queued`.
