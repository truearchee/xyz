## Goal
Ship deterministic, lecturer-only, aggregate-only assessment analysis for quiz answers: per-question rates,
most-missed questions, distractor breakdown, and per-topic class mastery when question metadata supports it.

## Why now
11.1 and 11.2 established the Stage 11 analytics surface and lecturer roster workflow. 11.3 adds the next
lecturer insight layer without adding AI, new risk logic, or persisted aggregate tables.

## Build
- Verify question topic metadata before implementing topic mastery:
  - `QuizQuestion` has `source_section_id` and `source_summary_id`.
  - `QuizQuestion` does not have `topic_id` or `source_week_number`.
  - Per-topic class mastery may use reliable section provenance only; never infer a topic from text.
- Add compute-on-read assessment analytics in `platform/query` over completed quiz attempts and
  `StudentAnswer`.
- Add per-question aggregate stats:
  - stable question key;
  - question text;
  - answer count;
  - correct count;
  - incorrect count;
  - correct rate when the small-cohort rule allows it.
- Add most-missed question output derived from the same aggregate rows.
- Add distractor breakdown for wrong answer options chosen, grouped by deterministic option identity/text.
- Add per-topic class mastery when answered questions can be mapped to a source section through
  `QuizQuestion.source_section_id` or, if needed, `QuizQuestion.source_summary_id ->
  GeneratedLectureSummary.module_section_id`.
- Return a topic-mastery unavailable state when no usable provenance exists for the cohort.
- Enforce the small-cohort rule: if an aggregate's answer count is less than 3, return
  "Not enough submissions for an aggregate insight" instead of percentages or misleading ranked insight.
- Add a lecturer-owned-module API endpoint for assessment insights.
- Add lecturer UI on the module page showing question stats, most-missed questions, distractor breakdown,
  topic mastery or unavailable state, and small-cohort messages.
- Reference latest `AgentRun` history in the response for context, without persisting or duplicating stats.

## Do not build
- No AI, prompt, provider, `AIRequestLog`, validator, rule-11 smoke, or model-routing work.
- No recommendations changes except where unavoidable to share existing analytics routing/components.
- No risk recomputation, grade advice, workload planner, `.ics`, assistant, or gamification work.
- No Stage 10 reads.
- No new persisted heavy aggregate tables.
- No per-student identity exposure in lecturer analytics.
- No guessed topic mapping from question text, answer text, summary text, or week labels.

## Data model changes
- No migration planned.
- Current Alembic head is `0057`.
- 11.3 should not create a migration unless implementation discovers a narrow missing index or constraint.
  If a migration becomes necessary, stop before using `0060+`; `0058` and `0059` are the only remaining
  Stage 11 buffer numbers under the current owner instruction.

## API changes
- Add `GET /lecturer/modules/{module_id}/analytics/assessment-insights`.
- Response shape should include:
  - `moduleId`, `moduleTitle`;
  - latest relevant `AgentRun` metadata, if available;
  - aggregate question stats;
  - most-missed questions;
  - distractor breakdowns;
  - topic mastery availability and rows;
  - small-cohort flags/messages.

## Worker / job changes
None. 11.3 computes on read and does not schedule or enqueue work.

## Authz rules
- Lecturer role only.
- Lecturer must have an active membership on the requested module.
- Cross-course lecturer access returns 403.
- Student and admin access to lecturer assessment insights returns 403.
- Response must not include student names, emails, IDs, or per-student answer rows.

## Verification
- Backend tests for:
  - exact per-question correct rates from a known answer distribution;
  - most-missed ordering;
  - distractor counts/rates;
  - topic mastery present when section provenance exists;
  - topic mastery unavailable when provenance is absent;
  - small-cohort suppression below 3 answers;
  - aggregate-only response shape;
  - lecturer own-course allowed and cross-course/student/admin forbidden.
- Frontend type-check and component tests for:
  - question stats;
  - distractor breakdown;
  - topic mastery present/unavailable states;
  - small-cohort message with no percentages.
- Browser gate:
  - seed a quiz with a known answer distribution;
  - lecturer opens analysis;
  - per-question rates, most-missed questions, and distractor breakdown match the seed exactly;
  - per-topic mastery appears when provenance exists, or "unavailable" appears when it does not;
  - a less-than-3 submission cohort shows the small-cohort message, not percentages;
  - no student identity appears.
- Full active Playwright suite per rule 14.
