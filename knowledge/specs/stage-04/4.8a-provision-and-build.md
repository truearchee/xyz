## Goal
Make every in-repo change required to **build and provision** the staging topology ‚ÄĒ dual DB URLs,
strict readiness, the fault-injection boot backstop, the frontend production image, `fly.toml`,
`check-staging-env`, `.env.staging.example` ‚ÄĒ such that images build, `/health` (liveness) is
reachable **without a migrated DB**, and `scripts/check-staging-env` correctly accepts a good
staging env and rejects every bad one. **No release migration, no first-admin bootstrap, no deploy
script (all 4.8b); readiness is legitimately RED.**

## Why now
4.8a is the umbrella spec's first sub-session (¬ß10): stand the topology up to "INFRA PROVISIONED,
readiness still red." It front-loads the constraint-discovery items that are pure code/config
(pooler hygiene, connection budget, the advisory-lock-over-pooler fix, embed sizing, env hygiene)
so the live deploy in 4.8b/4.8d only exercises wiring we have already locally proven.

## Reconciliations (evidence in context ‚ÄĒ per developer review of the umbrella Filing note)

These five facts were verified against the tree during 4.8a scoping; each governs a build decision
below. **Show, not tell:**

1. **Upload path = FastAPI multipart passthrough** (NOT signed-URL PUT). `frontend/src/lib/api/upload.ts:201-216`
   (`uploadTranscript`) ‚Üí `uploadMultipart` (`:112-164`) ‚Üí `fetch(POST, body: FormData)` to
   `/modules/{id}/sections/{id}/transcript`; explicit `413: 'Payload Too Large'` handler at `:151`.
   ‚áí **Failure-catalog #1 fork #1 ACTIVATES** (API body-size); the storage-CORS fork is dormant.
2. **ECC P-256 already supported** ‚ÄĒ `app/platform/auth/jwt.py` decodes with `algorithms=["RS256","ES256"]`.
   ‚áí The "confirm asymmetric signing" **config task is dropped**; the *runtime assertion* that a
   staging-minted token VALIDATES is **kept and carried to 4.8d** (smoke), not done here.
3. **Reaper lock is cross-process via `pg_try_advisory_lock`** (`app/domains/recovery/locks.py:30-42`)
   ‚ÄĒ failure-catalog #9 closes. **Consequence (the thing to verify):** a session-level advisory lock
   is per-connection and is released at the `conn.commit()` (`locks.py:36`) when that connection
   returns to a *transaction pooler*. ‚áí The recovery engine MUST use `DIRECT_DATABASE_URL`
   (session endpoint), per adr-041. Today worker startup builds it from `DATABASE_URL`
   (`app/workers/worker.py:56`); 4.8a re-points it to the direct URL. Both lock call sites
   (worker startup + admin-trigger) must use a session/direct engine.
4. **pgcrypto has no consumer** ‚ÄĒ `rg` finds only `CREATE EXTENSION IF NOT EXISTS vector`
   (`alembic/versions/0006_transcript_chunks.py:30`, `0007_embedding_schema_preflight.py:115`); no
   `pgcrypto`, no `gen_random_uuid`/`uuid_generate`/`uuid-ossp` (UUIDs are app-side `uuid7()`).
   ‚áí **vector-only bootstrap; pgcrypto dropped from the migration scope AND from `check-staging-env`.**
   Note: the extension `CREATE` already lives in migrations 0006/0007 ‚ÄĒ F006's residual gap is only
   the migration-role **privilege** (verified in 4.8b), not a missing migration.
5. **Fault flags run locally under `ENVIRONMENT=development`** (`.env:42`, `.env.example:81`;
   `docker-compose.fault.yml` sets the fault flags but NO `ENVIRONMENT`). The retained step-guard
   uses `IS_NON_PROD = ENVIRONMENT ‚ąČ {production, staging}` (`config.py:144-146`,
   `pipeline_faults.py:35-39`). ‚áí The new **boot check uses the SAME predicate** (refuse iff a fault
   flag is set AND `not IS_NON_PROD`), so it is consistent with the step-guard and does NOT break the
   local fault harness. See **Open question O1** ‚ÄĒ a literal `{test,e2e}`-only allow-set would
   refuse `development` and break local fault testing; flagged for your call, not silently adopted.

## Build
- **B1 ‚ÄĒ Dual DB URL + engine hygiene.** Add `DIRECT_DATABASE_URL` setting. Parametrize the async
  engine in `session.py`: `connect_args` for asyncpg over the transaction pooler
  (`statement_cache_size=0`, unique `prepared_statement_name_func`), `ssl=require` when the URL is
  remote, small bounded pool. Provide a **direct-engine factory** (session endpoint, `NullPool`) for
  migrations and the reaper lock. Local default (`postgresql+asyncpg://‚Ä¶@db`) must keep working with
  the same code path.
- **B2 ‚ÄĒ Recovery engine ‚Üí direct URL (advisory-lock-over-pooler fix).** Re-point the worker startup
  recovery engine (`worker.py:_startup_recovery_async`) from `DATABASE_URL` to `DIRECT_DATABASE_URL`
  (falling back to `DATABASE_URL` when direct is unset, e.g. local single-URL). Audit the
  admin-triggered reaper/reconciliation path so its advisory-lock engine is also session/direct.
- **B3 ‚ÄĒ `/health/ready` (built, red until 4.8b).** Add `GET /health/ready`: DB reachable + Redis
  reachable + `alembic_version == head`. Returns 503 (red) when any check fails ‚ÄĒ which **includes
  the legitimate 4.8a state of an un-migrated DB**. `/health` stays liveness-only (no deps, unauth).
- **B4 ‚ÄĒ Fault-injection fail-loud boot check.** A shared helper (co-located with
  `pipeline_faults.py`) that refuses to boot when `PIPELINE_FAULT_INJECTION_ENABLED`,
  `PIPELINE_FAULT_INJECTION`, or `LLM_FAULT_INJECTION` is set while `not IS_NON_PROD`. Called at the
  top of `create_app()` (`main.py`) and worker `main()` (`worker.py`). Exits via `SystemExit` with a
  message naming the offending flag(s) + `ENVIRONMENT` (NO secret values). The step-level
  `_assert_safe_to_inject()` raise **stays unchanged**.
- **B5 ‚ÄĒ Frontend production image.** `output: 'standalone'` in `next.config.ts`; multi-stage
  `frontend/Dockerfile` (`npm ci` ‚Üí `npm run build` ‚Üí runtime stage running `node server.js`), Node
  base pinned **by digest**. Staging build bakes staging `NEXT_PUBLIC_API_BASE_URL` +
  `NEXT_PUBLIC_SUPABASE_URL` + `NEXT_PUBLIC_SUPABASE_ANON_KEY`; `NEXT_PUBLIC_E2E_TEST_HOOKS`
  **absent** at build.
- **B6 ‚ÄĒ `fly.toml`.** Process groups mapping 1:1 to the compose `command:` split: `api` (uvicorn),
  `worker` (default/ingestion), `embedding_worker`, `ai_worker`, `frontend`. Embed group sized for
  torch+MiniLM (memory > the others); `release_command` slot present but **left inert/commented for
  4.8b**. Pin images by digest/immutable tag.
- **B7 ‚ÄĒ `scripts/check-staging-env` (executable).** The ¬ß7.H matrix on the **real** names. Fails
  (non-zero) if any hold: `NEXT_PUBLIC_API_BASE_URL` contains localhost; `NEXT_PUBLIC_E2E_TEST_HOOKS`
  set; any fault flag set (`PIPELINE_FAULT_INJECTION_ENABLED`/`PIPELINE_FAULT_INJECTION`/`LLM_FAULT_INJECTION`);
  `SUPABASE_URL`/`SUPABASE_JWKS_URL` point at the dev project; `DATABASE_URL == DIRECT_DATABASE_URL`;
  `DATABASE_URL` is not the pooler endpoint; `LLM_API_KEY` missing while `LLM_PROVIDER=k2think`;
  `CORS_ORIGINS` missing the staging frontend origin. **PLUS ‚ÄĒ stale-name trap:** fail if any
  **renamed left-hand name** is present in the env (`K2THINK_API_KEY`, `K2THINK_BASE_URL`, `APP_ENV`,
  `SUPABASE_SERVICE_ROLE_KEY`, `EMBEDDING_MODEL_SHA`, `NEXT_PUBLIC_API_URL`) ‚ÄĒ a lingering old name is
  exactly the silent misconfig the script exists to catch.
- **B8 ‚ÄĒ `.env.staging.example`.** Full env matrix with real names, secrets redacted; the A1
  pooler/direct split documented inline.
- **B9 ‚ÄĒ Upload body-size (fork #1).** Confirm the platform ingress permits bodies ‚Č• the app caps
  (`MAX_TRANSCRIPT_UPLOAD_BYTES`=10 MB, `MAX_SECTION_ASSET_UPLOAD_BYTES`=25 MB) so the **app** cap is
  the binding 413, not a surprise proxy 413. Record the platform limit + any raise in the runbook
  (no app code change expected; the caps already exist).
- **B10 ‚ÄĒ Embed offline model check (confirm only).** `validate_model_snapshot()`
  (`embedding_encoder.py`) already runs at embed-worker boot (`worker.py:93-99`) reading the baked
  `MODEL_REVISION` offline. Confirm it does not reach HuggingFace; no new code expected.

- 2026-06-12 ‚ÄĒ Developer review of the 4.8a plan: approved to implement with refinements baked into
  code (not just the plan):
  - **O1 RULED:** ship the `IS_NON_PROD` predicate (refuse boot in production/staging; one predicate
    shared with the step-guard). NOT tightened to `{test,e2e}` (would break local fault testing under
    `ENVIRONMENT=development` for zero security gain ‚ÄĒ development isn't hosted). **Gap-closer:**
    `check-staging-env` must assert `ENVIRONMENT=staging` *explicitly* (not merely "set"), because the
    boot check permits `development`; layered (boot guards the flag, script guards the env value).
  - **MF1 (reaper):** the fix is connection-lifetime + lock-variant, not a URL swap. Hold the lock on
    a single pinned `direct_engine.connect()` (NullPool, never handed back mid-reap); keep the
    **session-level** `pg_try_advisory_lock`/`pg_advisory_lock` (the `_xact_` variant releases at the
    first commit ‚ÄĒ `locks.py:36`). Critical section is **multi-commit** (the reaper opens several
    `factory()` sessions), so session-level on a dedicated connection is required. **Add a
    concurrency test:** two reaper invocations race ‚Üí exactly one proceeds. (Unit test proves
    singleton-under-concurrency on a direct connection; the pooler-handback failure mode is provable
    only against a real pooler ‚ÄĒ asserted by code + the 4.8d staging smoke. Stated, not overclaimed.)
  - **MF2 (`/health/ready`):** return **503 not 500** when `alembic_version` is absent (catch
    `UndefinedTable`/`ProgrammingError`) ‚ÄĒ that absent table IS the 4.8a end state. Assert a **single**
    head (`get_current_head()` raises on branched heads ‚Üí 503).
  - **MF3 (pooler discriminator):** drive `statement_cache_size=0` from an **explicit setting**
    (`DATABASE_POOLER`), NOT a port/host sniff (Supabase direct + pooler can both be :5432).
  - **MF4 (Fly topology):** **two apps** ‚ÄĒ `frontend`, and `backend`+3 workers as process groups ‚ÄĒ
    because that wires `CORS_ORIGINS` (frontend host) and `NEXT_PUBLIC_API_BASE_URL` (backend host).
    Emit `backend/fly.toml` + `frontend/fly.toml` (B6 amended from a single `fly.toml`).
  - **Should-verify (report, don't necessarily change):** asyncpg pool + RQ fork safety (connections
    established post-fork in the child, not inherited); **full local active suite green at 4.8a
    close** (session.py is highest-regression-risk); `/health` is already DB-free (confirm); engine
