## Purpose
Fix F-4.3.5d-005.

4.3.5d-E2 proved that after a lecturer unpublishes a section, the authenticated student section-list response excludes the unpublished section. However, a fresh signed URL request for the same section asset returned:

```text
404 {"detail":"SECTION_NOT_FOUND"}
```

The required contract is:

```text
403
```

The student is authenticated and assigned to the module. The asset exists. The denial is an authorization denial caused by unpublished visibility, not a missing section or unauthenticated request.

## Outcome
PASS on 2026-06-08 16:28 +04.

The backend now returns `403 CONTENT_FORBIDDEN` for an authenticated student requesting a fresh signed URL for an existing unpublished section asset. The E2 proof passed and Stage 3 returned to `FULLY VERIFIED`.
