# Session 4.5.1a — Partition + Map + Reduce (BACKEND VERIFIED target)

> Slice 1 of [[specs/stage-04/4.5.1-map-reduce-summarization]]. The map-reduce ENGINE: partition → map →
> reduce, fully exercised on the deterministic adapter (no real provider). Brief-from-detailed + downstream
> gating + backfill + fencing are 4.5.1b; real-provider smoke + 4.7 re-gate are 4.5.1c. **No source edits
> until this spec + plan are approved.**

## Goal
Replace the single-call detailed-summary generation with a deterministic **partition → map → reduce** engine
that covers the WHOLE transcript and never 408s (every unit + the reduce input is kept under the budget),
with full resumability + provenance + a provider-free coverage proof.

## Build
**Config (lands HERE — partition can't run without it):**
- `LLM_SUMMARY_MAP_UNIT_CHAR_BUDGET` (~9000) + `LLM_SUMMARY_MAP_UNIT_TOKEN_BUDGET` (~2200 est) — BOTH caps;
  headroom under the 115–145s ceiling. `LLM_SUMMARY_REDUCE_INPUT_*_BUDGET` (the §3.3 guard threshold).
  `LLM_SUMMARY_MAX_MAP_UNITS` (cost-guard, ~20). `maxMapConcurrency=1` (cfg). All config, not hardcoded.

**Partition** (`summary_service` / a new `map_reduce.py`):
- Over `TranscriptSegment` (ordered), AFTER the existing `normalize_segment_text`. Group CONSECUTIVE segments
  into the fewest units each fitting BOTH caps (reserve prompt-overhead + completion budget); never split a
  segment. Deterministic for (content, budget, chunkerVersion). Compute `partitionConfigHash` (budget +
  chunkerVersion + transcript checksum). If units > `MAX_MAP_UNITS` → fail loud (cost guard). NO touch to
  `TranscriptChunk` (Stage-8 retrieval, different sizing).

**Map** (one call/unit):
- `gateway.complete(prompt_key=detailed_summary_map/v1, output_schema=DetailedSummaryPartial, priority="background",
  feature="detailed_summary_map", ...)` per unit. New prompt `prompts/detailed_summary_map/v1.yaml` (DETAILED
  route Think/Nvidia; bounded output; emits the detailed sections SCOPED to the unit + optional
  `carriedForwardContext`). **Map validation** (distinct, lenient): valid JSON, unitIndex + segment bounds
  match the request, required sections MAY be empty but empty is explicitly marked. Persist each partial to
  `MapUnitSummary` (status, partialContent, partitionConfigHash, sourceTranscriptChecksum, aiRequestLogId).

**Reduce** (combine + dedup + coherence, with the C1 guard):
- BEFORE the call, check concat(partials) vs the reduce budget. WITHIN → one reduce call. OVER → **tiered
  reduce**: ordered groups (each under budget) → group-summaries → reduce those; repeat until one fits.
- New prompt `prompts/detailed_summary_reduce/v1.yaml` (DETAILED route): dedupe + synthesize + resolve
  cross-unit references (via carriedForwardContext) into one coherent `DetailedSummary`. **Reduce validation**
  (strict): all required sections present/non-empty (labNotes lab-only) + a `coverageManifest` listing every
  consumed unitIndex — assert NONE omitted.
- Persist the final `GeneratedLectureSummary` with `generationStrategy="map_reduce"`, `truncated=false`,
  `generationMetadata` (prompt versions, mapUnitCount, sourceMapUnitSummaryIds, coverageManifest). (The
  brief is 4.5.1b; 4.5.1a wires the DETAILED job to the engine.)

**Schema (migration, additive):** `MapUnitSummary` table (partition-bound identity, C3); `GeneratedLectureSummary`
+ `generationStrategy` / `generationMetadata` (truncated already exists from 0014); `AIRequestLog.feature`
CheckConstraint + `SummaryFeature` type extended (`detailed_summary_map`, `detailed_summary_reduce`).

**Deterministic adapter + fixtures:** canned `detailed_summary_map` (per unit) + `detailed_summary_reduce`
(incl. the tiered/group path) output. Fixtures: a multi-unit (≥2) transcript + a **late-topic sentinel**
(final unit = `LateLectureSentinelTopic`; reduce output + coverageManifest MUST include it + the final unit
index). Optional dense fixture to trip the §3.3 guard.

## Verification (BACKEND VERIFIED, deterministic adapter)
`pytest`: partition is deterministic + fewest-units + respects both caps + never splits a segment +
`partitionConfigHash` stable + MAX_MAP_UNITS cost-guard fires; map writes valid `MapUnitSummary` partials
(empty sections allowed-marked); reduce GUARD picks single-vs-tiered correctly (dense fixture trips tiered);
reduce validation rejects an omitted unit; **coverageManifest covers all units (late-topic-sentinel asserts
the FINAL unit + its topic are present)**; resume skips completed+valid units and re-maps on
partitionConfigHash/checksum mismatch (C3); the orchestrating job persists `generationStrategy=map_reduce`,
`truncated=false`. Migration fresh-DB round-trip. `tsc`/frontend untouched (no UI in 4.5.1a). No real provider.
