## Goal
Adopt Tailwind v4 with the two-layer token system as the **single styling source of truth**, wire
self-hosted fonts, and stand up the **shell infrastructure** (token-styled app chrome + portal mount
points + landmarks/skip-link + reserved assistant anchor + route-level error/404/403 surfaces) — so
that 4.9b's components and Stage 5+'s toasts/modals/failures have a home. **No public components, no
feature-surface restyle, no tests** this session.

## Why now
First sub-session of Stage 4.9 (umbrella §9). The token contract + shell are the foundation every
later sub-session consumes; the **HARD GATE** (umbrella §9) is that design direction is locked before
this lands — it is (design-plan.md, Phase 0). Build-order: tokens → components (4.9b) → restyle (4.9c).

## Build
1. **Add deps** (devDeps unless noted; update + commit `frontend/package-lock.json`; each reason in the report per §4.4):
   - `tailwindcss@^4` + `@tailwindcss/postcss@^4` (+ `postcss@^8` peer) — the v4 engine via PostCSS (ADR-044).
   - `react-aria-components@^1` (runtime) — headless behavior primitive for 4.9b Modal/Toast, styled through our tokens (ADR-046).
   - `lucide-react` (runtime) — single tree-shakeable icon source, per-icon imports only (ADR-046, §4.4).
2. **Tailwind v4 via PostCSS** (NOT the Next plugin — parity across `next dev`/`build`/standalone; `next.config.ts` webpack stays untouched): `postcss.config.mjs` (`@tailwindcss/postcss`); `globals.css` = `@import "tailwindcss";` + `@theme` + `@source "../components"; @source "../features"; @source "../app";`; import `globals.css` in `layout.tsx`.
3. **Two-layer token system** in `globals.css` (ADR-045), values from design-plan Part 1.2/1.2b:
   - Layer 1 raw palette as `:root` `--palette-*` vars (violet ramp, zinc ramp, status raws) — private, no utilities.
   - Layer 2 semantic roles inside `@theme` referencing Layer 1 (`--color-surface/-muted/-raised`, `--color-border`, `--color-border-strong`, `--color-text/-muted`, `--color-primary`/`--color-on-primary`, success/warning/danger/info + `-on-` pairs + tonal `-surface`/`-text`, `--color-focus-ring`).
   - **Disable default color namespace** with `--color-*: initial` (build-enforced: only semantic color utilities resolve; spacing/font/radius/shadow scales untouched).
   - Type scale + weights, spacing, radius, elevation (shadow), motion (duration/easing + `prefers-reduced-motion` convention), and **z-index layering** as `@layer utilities` named classes `.z-base/.z-dropdown/.z-modal/.z-toast/.z-assistant`.
4. **Re-confirm token-level WCAG AA** against the *shipped* CSS: the semantic→raw mapping matches design-plan Part 1.2 and every validated pair holds (solid success/warning/info = large/UI only; `--color-border-strong` = zinc-500 for control boundaries). A drift is a finding (rule 13).
5. **next/font** in `layout.tsx`: Space Grotesk → `--font-display`, Inter → `--font-sans`; map both into the `@theme` type tokens; apply the variable classNames to `<html>`; body defaults to Inter. **No external font CSS / `<link>` / `@import`. No monospace family** (dropped per design-plan 1.3).
6. **Shell infrastructure:**
   - Root `layout.tsx` renders the portal **mount points**: `<div id="modal-root" class="z-modal">`, `<div id="toast-root" class="z-toast">` (the *components* that render into them are 4.9b).
   - `AppShell.tsx` → token-styled, **role-aware** chrome (student/lecturer/admin nav), semantic `<nav>`/`<main>`, a **skip-to-content link**, sane heading order; reserve `<div id="assistant-anchor" class="fixed … z-assistant" aria-hidden>` (Stage 8 slot — anchor + layer only, no widget). **Preserve every existing E2E-relevant selector** (the "Log out" control, role nav, user display) — `4.3.5b`/`4.3.5c` run inside this shell.
   - Route-level `error.tsx`, `not-found.tsx`, and the 403/unauthorized surface → token-styled (heading + message + one clear action/link). These ship as token-styled markup now; **4.9b refactors them onto the Empty State component** (noted, not a gap). The 403 surface is the *visual* form only — the keep-session/render-unauthorized **logic is unchanged** (asserted later in §6.1).
7. **ADRs** adr-044 (Tailwind v4 + App Router setup; REPO FIT — fall back to v3 is a *finding* if v4 fights tooling), adr-045 (two-layer token architecture; namespace disabled; theming-ready, dark mode out of scope), adr-046 (primitive strategy = React Aria headless + lucide icon source; deps recorded).
