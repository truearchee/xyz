## 1. Goal & obligations
Build the **complete AI provenance and capacity infrastructure before the first K2Think call**,
then generate a brief and a detailed study summary per transcript through that infrastructure.

**UI proof obligation:** the lecturer triggers the pipeline and *reads a real AI-generated summary
in the browser*, produced through the logged AI stack.

This stage is infrastructure-first. Slices 3/5/6/8 all consume `platform/llm`. The boundary shape
is fixed now even where the MVP only exercises part of it.

## 4. Architecture (adr-028)
```
feature job handler (summary)
  → LLMGateway.complete(prompt_key, output_schema, context_refs, priority)
        │
        ├─ PromptRegistry.render(prompt_key) → RenderedPrompt
        ├─ AIRequestLog.open()               → row status=running  ← BEFORE any check
        ├─ ContextBuilder.fit(rendered)      → token count + route selection
        │     if over-context with no route  → AIRequestLog.close(invalid_input); raise
        ├─ RateLimiter.acquire(backend, est_tokens, priority)  TTL lease
        │     if blocked                     → AIRequestLog.close(rate_limited); raise/backoff
        ├─ LLMProvider.send(rendered, backend) → RawCompletion
        │     on 429                         → in-call backoff under limiter
        │     on 5xx                         → AIRequestLog.close(provider_transient); raise
        ├─ OutputValidator.validate(raw, output_schema)
        │     on failure                     → AIRequestLog.close(invalid_output); raise
        └─ AIRequestLog.close(succeeded, usage)
           return CompletionResult
```
- **`LLMGateway`** owns all cross-cutting concerns: `PromptKey`, rendering, budget, limiter,
  logging, validation, retry classification. Real and deterministic providers are behaviorally
  identical behind it.
- **`LLMProvider`** = thin transport adapter only: `send()` and `stream_raw()`.
- **`AIRequestLog.open()` is called before any other check** so `invalid_input` (detected by
  `ContextBuilder.fit`, before transport) is loggable. **(Patch A — structural fix.)**
- **Rule 6 reconciliation (adr-028):** roadmap rule 6 names `LLMProvider` as the interface with
  `complete()`/`stream()`. That public contract now lives at **`LLMGateway`**; the intent
  (everything passes through the chain; stream signature fixed day one so Stage 8 can't bypass the
  gateway) is preserved and strengthened. Rule 6 gets a one-line annotation in the same commit as
  adr-028.
- **Hard ordering (rule 6):** `AIRequestLog` table + write path exist and are exercised in CI
  before any real K2Think call exists in the codebase. 4.5a lands everything against the
  deterministic provider; 4.5b introduces the first real call.

`platform/llm` is infrastructure consumed by domains. `backend/app/ai/` is migrated or deleted in 4.5a.

## 6. Component specs

### 6.1 LLMGateway public interface
```python
@dataclass(frozen=True)
class PromptKey:
    name: str
    version: str

class LLMGateway:
    def complete(self, *, prompt_key, output_schema, context_refs, priority) -> CompletionResult: ...
    def stream(self, *, prompt_key, output_schema, context_refs, priority): ...  # NotImplemented until 8.3

class CompletionResult(TypedDict):
    parsed: BriefSummary | DetailedSummary
    model_id_echoed: str
    usage: Usage
    backend_used: Backend          # "cerebras" | "nvidia"
    reasoning_level: str | None    # D4
    ai_request_log_id: UUID        # FK to log row for provenance chain
```
`output_schema` on the call site prevents "valid JSON but wrong shape for this job." `stream()`
raises `NotImplementedError` until 8.3 — signature fixed now so Stage 8 can't bypass the gateway.

### 6.2 LLMProvider (transport adapter only)
```python
class LLMProvider(Protocol):
    def send(self, *, rendered: RenderedPrompt, backend: Backend) -> RawCompletion: ...
    def stream_raw(self, *, rendered: RenderedPrompt, backend: Backend) -> Iterator[str]: ...
```
HTTP + auth only. No rendering, logging, limiting, or validation.

### 6.3 DeterministicTestProvider
- Returns fixed, valid, schema-conformant output so CI runs the **full gateway path** (render,
  ContextBuilder, limiter, AIRequestLog, OutputValidator) at the provider boundary only (rule 11).
- E2E-only fault-injection switch forces: `invalid_output`, `invalid_input` (over-context, before
  transport), `provider_transient` (5xx).
- Fault flags impossible to enable outside test/E2E (consistent with 4.6/4.8 hygiene rules).

### 6.4 PromptRegistry (flat files — adr-026)
Version-controlled `/prompts`; loaded **and validated at startup** (malformed/missing = boot
failure). Each file: `name`, `version`, `content`, `max_tokens`, `model`, `backend`,
`reasoning_level?`. Lookup key = `PromptKey(name, version)`. **Drift guard:** content hash per file;
CI (`prompt_drift_guard.py`) fails if content changed without a version bump. Hash stored as
`promptContentHash` in `AIRequestLog` and `GeneratedLectureSummary` per call.

### 6.5 RateLimiter (rule 15 + TTL leases + priority headroom)
| dimension | Cerebras | Nvidia |
|---|---|---|
| requests/min | 20 | 10 |
| tokens/min | 100,000 | 105,000 |
| concurrency | 10 | 10 |

- Reservation = estimated prompt tokens (D2) + prompt `max_tokens`; reconciled with response usage.
- **Priority headroom:** `LLM_INTERACTIVE_HEADROOM_PERCENT=20` → background jobs consume ≤80% of
  RPM/TPM/concurrency per backend.
- **429 backoff inside the call path under the limiter.** RQ retries reserved for `provider_transient`
  and bounded `invalid_output` only.
- **Concurrency slots are TTL leases:** acquired with TTL; long calls heartbeat-extend; released in
  `finally`; expired leases reclaimable so a crashed worker cannot leak a slot. Blocked/rate-limited
  decisions logged in `AIRequestLog`.
- TPM binds before RPM for summaries (~13–18k prompt tokens ⇒ ~5 detailed calls/min on Nvidia).

### 6.6 AIRequestLog — one row per LLMGateway completion attempt (**Patch A**)
```
id, ingestionJobId(FK), attemptNumber, feature("summary_brief"|"summary_detailed"),
modelId, promptVersion, promptContentHash, renderedPromptHash,
backendUsed(nullable), reasoningLevel(nullable), inputContentHash, estimatedPromptTokens,
promptTokens(nullable), completionTokens(nullable), totalTokens(nullable),
requestStartedAt, requestCompletedAt(nullable), latencyMs(nullable),
status(running|succeeded|rate_limited|provider_transient|invalid_output|invalid_input|failed),
errorClass(nullable), errorCode(nullable), providerRequestId(nullable),
debugTextTruncated(non-prod only; never transcript/prompt text), createdAt,
index(feature, createdAt)
```
**Key semantic (Patch A):** `AIRequestLog.open()` is called **before** `ContextBuilder.fit()`.
Provider fields (`providerRequestId`, latency, token counts, `backendUsed`, `requestCompletedAt`)
are **nullable** because some gateway attempts fail before transport (e.g. `invalid_input`).
**`failed` (generic):** unexpected gateway failure not classifiable as a typed status; must be
justified over a typed status — not a catch-all. **One row per gateway attempt;** retries open a
new row with `attemptNumber` incremented. **Never stores raw transcript text or full prompts.**

### 6.7 OutputValidator
- **Brief:** parse to `BriefSummary`; non-empty `text` within length bounds; reject refusals/chatter.
- **Detailed:** parse to `DetailedSummary` (D1) via passed `output_schema`; required non-empty:
  `overview`, `keyConcepts`, `importantDefinitions`, `mainExplanations`, `examples`,
  `examRelevantPoints`; `labNotes` required only when section type is `lab`.
- Validation failure → `invalid_output` → bounded RQ retry; persistent → terminal `invalid_output`.

### 6.8 `ai` RQ queue + worker
Dedicated `ai` queue + worker container (mirrors embedding-worker isolation). Worker startup
verifies provider config; in the real-provider path asserts echoed model ID matches config (4.5b).

## 7. Data model

### 7.1 GeneratedLectureSummary — success artifacts only (**Patch B**)
```
id, transcriptId(FK), moduleSectionId(FK), summaryType(brief|detailed_study),
contentJson(jsonb NOT NULL), contentSchemaVersion(NOT NULL), modelId, promptVersion,
promptContentHash,            ← Patch B
backendUsed, reasoningLevel(nullable), sourceTranscriptChecksum, inputHash,
aiRequestLogId(FK NOT NULL),  ← Patch B
generatedAt, createdAt, updatedAt
unique (transcriptId, summaryType, sourceTranscriptChecksum, promptVersion,
        promptContentHash, inputHash)   ← Patch B: prompt-aware uniqueness; regen never overwrites history
```
**No `status` column.** Row exists ⇒ success. **`aiRequestLogId FK NOT NULL`:** every artifact
traces to the gateway attempt that produced it. If an "active/latest summary" concept is needed
later, defer to an `isActive` flag or read projection — do not overwrite/lose history.

`contentJson`: brief `{"text": "..."}`; detailed `{overview, keyConcepts, importantDefinitions,
mainExplanations, examples, examRelevantPoints, labNotes?}` (`labNotes` required only for lab).

**Deviation from Slice 2** (`status: generated|failed`): success-only table; failure state belongs
to `IngestionJob`+`AIRequestLog`. Rationale in adr-027.

### 7.2 IngestionJob (extend)
`jobType` already enumerates `generate_brief_summary`, `generate_detailed_summary` (migration 0005).
Add `failureCategory` (§7.5). Carries `attempts`, `status(queued|running|completed|failed)`,
`errorMessage`, FK to transcript, linkage to its `1..n` `AIRequestLog` rows.

### 7.3 Idempotency + one-active index
- **Idempotency key** = `hash(transcriptId, jobType, sourceTranscriptChecksum)`.
- **One-active partial-unique index** on `IngestionJob (transcriptId, jobType) WHERE jobType IN
  ('generate_brief_summary','generate_detailed_summary') AND status IN ('queued','running')`
  — extends migration 0007's pattern.

### 7.4 Status projection (prereq 2.C)
`overallState` gains `summarized` after `embedded`; `steps` gains `summary_brief`,
`summary_detailed` (`pending|running|completed|failed`). The projection is the **sole doneness
authority**. Per-step failure must be representable
(e.g. `{embed: completed, summary_brief: completed, summary_detailed: failed}`).

### 7.5 Failure taxonomy
```
provider_transient   retryable              5xx, timeout        → RQ retry
rate_limited         retryable / delayed    429                 → in-call backoff
invalid_output       retryable, bounded     failed validation   → bounded RQ retry, then failed
invalid_input        NON-retryable          over-context        → failed; retry only after change
failed               non-typed gateway failure (§6.6)
```
`IngestionJob.failureCategory` records the terminal category; 4.6 UI copy differs by category.

## 8. Pipeline flow (after-commit enqueue)
```
embed step completes
  DB transaction: projection.overallState='embedded';
                  insert IngestionJob(generate_brief_summary, queued);
                  insert IngestionJob(generate_detailed_summary, queued)
  After commit:   enqueue RQ jobs (ai queue)  brief→Cerebras budget, detail→Nvidia budget
brief job (Cerebras): load full normalized transcript → gateway.complete(BriefSummary) → store
detailed job (Nvidia): load full normalized transcript → gateway.complete(DetailedSummary) → store
brief lands first → steps.summary_brief=completed; both complete → overallState='summarized'
```
**After-commit enqueue, not a transactional outbox.** **Known 4.5 resilience gap:** if commit
succeeds but enqueue fails, summary IngestionJob rows stay `queued` with no RQ job; 4.5 UI shows
generating without claiming failure; the **Stage 4.6 sweeper** re-enqueues. **Do NOT add an ad-hoc
fix inside 4.5.** One call per summary over the FULL normalized transcript (rule 15). No map-reduce.

## 9. Routing (adr-025)
| summary | model | backend | rationale |
|---|---|---|---|
| brief | `MBZUAI-IFM/K2-V2-Instruct` | Cerebras (default) | plain writing; 32k fits; separate budget |
| brief (fallback) | `MBZUAI-IFM/K2-Think-v0` | Nvidia | only if prompt exceeds Cerebras window |
| detailed | `MBZUAI-IFM/K2-Think-v0` | Nvidia | reasoning; 128k; structured output |

Over Nvidia window → `invalid_input` (D3). Fallback consumes Nvidia budget (accepted at MVP scale;
alternative — letting brief fail while detailed succeeds — rejected as confusing UX).

## 10. Prompts (semantic filtering lives here)
Normalization is structural-only by design. Prompts explicitly instruct the model to ignore
greetings, mic checks, camera/background talk, admin announcements, small talk, closing remarks.
`brief_summary/v1`: short bounded paragraph. `detailed_summary/v1`: emit `DetailedSummary` JSON (§7.1).

## 11. Failure contract (consumed by 4.6)
Terminally failed summary job leaves: `IngestionJob.status='failed'` + `failureCategory` +
`errorMessage` + `attempts`; `1..n` `AIRequestLog` rows with terminal statuses; **no
`GeneratedLectureSummary` row**. Exact contract Stage 4.6's lecturer retry reads.
