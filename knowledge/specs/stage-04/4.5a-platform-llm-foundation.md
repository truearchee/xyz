## Goal
The complete `platform/llm` AI-infrastructure stack — gateway, provider protocol, prompt registry,
Redis limiter, AIRequestLog write path, output validator, summary Pydantic models, `ai` queue/worker,
and the after-embed summary-job pipeline — exists and is fully exercised in CI **against a
`DeterministicTestProvider`**, with **no real K2Think call anywhere in the codebase**.

## Why now
Roadmap rule 6: the AIRequestLog table + write path must exist before the first K2Think call;
4.5a lands the whole boundary so 4.5b only swaps in the real transport. Gate 2.A is GREEN
([[4.5a]]).

## Build
- `platform/llm`: LLMGateway orchestration chain; LLMProvider protocol; DeterministicTestProvider
  (with E2E-only fault injection: invalid_output/invalid_input/provider_transient); K2ThinkProvider
  stub (no real call); PromptRegistry (flat files, startup validation, content hashing); RedisRateLimiter
  (RPM/TPM/concurrency, TTL leases + reclaim, priority headroom); ContextBuilder (D2 estimator,
  route selection); OutputValidator (+ Brief/Detailed validators); AIRequestLog open/close helpers
  (gateway-attempt semantics, hashes only); errors module.
- DB: AIRequestLog + GeneratedLectureSummary models; IngestionJob.failure_category; migration 0008
  (+ one-active partial-unique index for both summary job types).
- Prompts: brief_summary/v1.yaml, detailed_summary/v1.yaml with semantic-filtering instructions.
- CI: prompt_drift_guard.py.
- Pipeline: summary_service handlers; jobs.py wrappers; `ai` queue + enqueue helpers; after-embed
  enqueue of both summary IngestionJob rows (queued) + after-commit enqueue; ai_worker compose service.
- Projection: summary_brief/summary_detailed steps + `summarizing`/`summarized` overallState.
- Dev: reenqueue_summaries.py. Delete backend/app/ai/.

## Do not build
- Any real K2Think/IFM HTTP call (K2ThinkProvider.send raises NotImplementedError in 4.5a).
- Brief/detailed generation through a REAL provider (4.5b/4.5c).
- Lecturer summary UI / status-badge rework (4.5d); student visibility (4.7).
- Product retry flow or the queued-row repair sweeper (4.6).
- Map-reduce summarization; streaming transport (8.3 — stream() raises NotImplemented);
  prompt DB table; ANN index; quiz/glossary/assistant/analytics.

## Data model changes
- New: `ai_request_logs`, `generated_lecture_summaries`. Alter: `ingestion_jobs.failure_category`
  (+ partial-unique one-active index for summary job types). Migration `0008` (down_revision 0007).

## API changes
- No new endpoints. The projection response (`/processing-status`) gains two step fields and new
  overallState values → regenerate the TS client (rule 3); no new UI consumption in 4.5a.

## Worker / job changes
- New `ai` RQ queue + `ai_worker` container. New job types wired: `generate_brief_summary`,
  `generate_detailed_summary`. After-embed enqueues both (queued rows + after-commit enqueue).
