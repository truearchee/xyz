## Goal
Accept a VTT/TXT transcript from an assigned lecturer for a lecture/lab section, store the raw file unchanged in object storage, persist the full `transcripts` row at `status = uploaded`, and enforce the one-active-transcript-per-section invariant — without enqueuing or processing anything.

## Why now
Stage 4 starts the transcript and AI pipeline by landing the raw-upload boundary and database status model only. Later Stage 4 sessions can parse, chunk, embed, summarize, and expose student-facing summaries from this stable raw transcript layer.

## Build

### Service flow (authoritative ordering)
```
1. Resolve CurrentUserContext; gate role: student/admin -> reject.        [before body parse]
2. Resolve section via platform/query scoped read; None -> 404.           [before body parse]
3. section.type not in {lecture, lab} -> 422.                             [before body parse]
4. Fail-fast duplicate check: active transcript exists for section -> 409. [before body parse]
     (cheap DB read on the path param; don't make a lecturer stream 10 MB to learn this)
5. Parse multipart manually (post-auth). Exactly one part named `file`;
     zero or multiple file parts -> 422.
     Unexpected file parts -> 422 (reject; minimal surface).
     Non-file form fields -> ignored (no business meaning in 4.1).
6. Bounded read into ValidatedTranscriptUpload:
     - enforce max size -> 413 (authority is the app-level bounded read, not Content-Length)
     - compute sha256 incrementally (lowercase hex)
     - capture file_size; size 0 -> 422
7. Validate content (decoding is validation-only; raw bytes are what gets stored):
     - .vtt: must begin with WEBVTT after optional BOM/leading whitespace
     - .txt: UTF-8-decodable (BOM allowed/stripped for the check only), non-empty
8. Derive effective mime_type: .vtt -> text/vtt, .txt -> text/plain.
     Sanitize filename for the storage key; keep the original display name for the DB.
9. Generate transcript_id (uuid) + storage_key.
10. StorageProvider put RAW bytes/stream to the private bucket.
11. Insert Transcript(status='uploaded', is_active=true, source_type='manual_upload',
      uploaded_by_user_id=ctx.user_id, checksum, file_size, mime_type, original_file_name, storage_key).
      - IntegrityError on partial unique index (race past step 4) -> delete object, 409.
      - Any other DB failure after a successful put -> delete object, log orphan key, 5xx.
12. Return 201 + TranscriptMeta.
```
Step 4 is the courtesy fast-path; **step 11's partial unique index is the correctness guard** for the TOCTOU race.

### Filename sanitization (storage key only)
```
- strip path separators (/ \) and any ../ traversal
- strip control characters; normalize/collapse whitespace
- cap to 200 chars, preserving the extension
- if the sanitized name becomes empty -> fall back to transcript.{ext}
The DB stores the display name in original_file_name, but it is also sanitized
before persistence: strip control characters and cap to 255 chars. "Original"
means the user-facing name, not every byte the browser sent.
The storage key NEVER trusts the uploaded filename.
Frontend renders original_file_name as text only (not HTML).
```

### `ValidatedTranscriptUpload` (internal value object)
```
originalFileName, safeFileName, effectiveMimeType, sizeBytes, sha256, payload(spooled tmp file or bytes)
```
Built once in step 6; nothing downstream re-reads the stream or re-hashes.

### Frontend (minimal, presentational)
Regenerate the OpenAPI client; add a `TranscriptUpload` control + a status display beside the existing lecturer section components. Copy must read like *"Transcript uploaded. Processing is not available yet."* — never imply processing started. Keep `tsc --noEmit` green.

- Any status transition beyond `uploaded`.

## Data model changes
New table `transcripts` (snake_case DB):
```
transcripts
- id                       uuid pk
- module_section_id        uuid fk -> module_sections(id), not null
- source_type              text not null     (manual_upload | zoom_import)   # only manual_upload written in 4.1
- original_file_name       text not null     # untrusted display name
- storage_key              text not null
- mime_type                text not null     # derived (text/vtt | text/plain); not raw client MIME
- file_size                bigint not null
- checksum                 text not null     # sha256 of raw bytes, lowercase hex
- language                 text null          # unpopulated in 4.1
- status                   text not null default 'uploaded'
                           (uploaded|queued|parsing|chunking|embedding|generating|completed|failed)
- uploaded_by_user_id      uuid fk -> <table per P6>, null   # nullable so future zoom_import isn't forced to fake an uploader
- is_active                boolean not null default true
- superseded_at            timestamptz null
- created_at               timestamptz not null default now()
- updated_at               timestamptz not null default now()   # configure onupdate per repo convention, though unused in 4.1
```
Indexes / constraints:
```sql
-- one active transcript per section (the real guard; the service pre-check is only a fast-path)
CREATE UNIQUE INDEX uq_active_transcript_per_section
  ON transcripts (module_section_id) WHERE is_active = true;

-- one storage object maps to one transcript row
CREATE UNIQUE INDEX uq_transcripts_storage_key ON transcripts (storage_key);

-- plain FK index for later history queries
CREATE INDEX ix_transcripts_module_section_id ON transcripts (module_section_id);

CHECK (NOT (is_active = true AND superseded_at IS NOT NULL));          -- active row cannot be superseded
CHECK (source_type <> 'manual_upload' OR uploaded_by_user_id IS NOT NULL);
CHECK (file_size > 0);
CHECK (checksum ~ '^[a-f0-9]{64}$');                                   -- requires lowercase hex; hexdigest() emits it
-- enum guards (use native enum instead only if P3 shows the repo standardized on that):
CHECK (status IN ('uploaded','queued','parsing','chunking','embedding','generating','completed','failed'));
CHECK (source_type IN ('manual_upload','zoom_import'));
```
- `status` index is **deferred to 4.2** (when the worker polls by status). YAGNI here.
- The migration includes a working `downgrade` (drop table + indexes), exercised by the migration round-trip test.

## API changes
**Request** — `POST /sections/{section_id}/transcript`, `multipart/form-data`, field `file` (Blob, per the 3.1 codegen pattern). Match the existing router prefix/versioning convention.

**Response DTO** (camelCase) — `TranscriptMeta`. Internal fields (`storageKey`, `checksum`, `isActive`, `supersededAt`) are **not** exposed.
```
TranscriptMeta {
  id, moduleSectionId, sourceType, originalFileName,
  mimeType, fileSize, language, status,
  uploadedByUserId, createdAt, updatedAt
}
```
`POST` → `201` + `TranscriptMeta`. `GET /sections/{section_id}/transcript` → `200` + `TranscriptMeta` (active transcript) or `404` if none.

Error contract: `401` unauth · `403` student/admin · `404` unassigned lecturer / no transcript · `409` active transcript already exists · `413` too large · `422` unsupported section type / invalid file content / bad multipart shape.

## Worker / job changes
None. Explicitly nothing enqueued.

## Authz rules
Consume the resolved `CurrentUserContext` — the transcript domain must **not** issue its own membership query (P0 hard rule). `platform/query` returns *authorized section context only*; transcript-specific rules (type must be lecture/lab, duplicate → 409, file validation) live in `transcripts/service`. Do not add `can_upload_transcript` to the read layer.

`POST` (evaluate before touching the request body):
```
unauthenticated                         -> 401
role == student                         -> 403
role == admin                           -> 403
lecturer not assigned to parent module  -> 404   (fail-closed; match adr-011 convention)
section.type not in {lecture, lab}      -> 422
assigned lecturer + lecture/lab         -> proceed (read/validate/store)
```
`GET` (**lecturer-only in 4.1**; student access deferred to 4.7 via summary endpoints):
```
unauthenticated                         -> 401
role == student                         -> 403
lecturer not assigned to parent module  -> 404
assigned lecturer, no transcript        -> 404
assigned lecturer, active transcript    -> 200 + TranscriptMeta
```
- Upload/read are **independent of `publish_status`** — draft sections are valid targets.
- Per P1/3.1: do **not** declare `file: UploadFile = File(...)` if that causes FastAPI to parse the body before handler authz. Use `Request` + manual multipart parse after CurrentUserContext + section checks, or follow the exact 3.1 pattern that already solved this.

## Tests required
Reuse the fake `StorageProvider` from `test_content.py`.

**Unit — validators / sanitization / checksum**
- `.vtt` starting with `WEBVTT` (incl. with BOM/leading whitespace) → accept; `.vtt` without `WEBVTT` → 422.
- `.txt` non-empty UTF-8 → accept; empty `.txt` → 422; non-UTF-8 → 422.
- `.pdf` / `.srt` extension → 422.
- oversize → 413; size 0 → 422.
- client MIME `text/plain`/`application/octet-stream` on a valid `.vtt` → accepted (MIME advisory).
- derived mime asserted: `.vtt`→`text/vtt`, `.txt`→`text/plain`.
- known bytes → known lowercase-hex sha256.
- filename: `"../../lecture.vtt"` → safe key, `original_file_name` preserves the display name; `"lecture\n3.vtt"` → safe key; very long name → truncated, extension preserved; name sanitizing to empty → `transcript.vtt`.

**Integration**
- assigned lecturer + lecture → 201, `status='uploaded'`, object stored, **checksum present in DB** (not in response), response omits `storageKey`/`checksum`/`isActive`/`supersededAt`.
- assigned lecturer + lab → 201.
- assigned lecturer + assignment → 422; + supplementary → 422.
- assigned lecturer + **draft** lecture section → 201 (publish-independent).
- student → 403; admin → 403; unassigned lecturer → 404.
- **auth-before-parse**: unauthorized (student/admin/unassigned) oversized multipart → 403/404 with the storage mock asserted **never called** and no row created.
- zero file parts → 422; multiple file parts → 422.
- duplicate active transcript (existing active at request time) → 409, no second active row, loser object cleaned up.
- **lost-race (TOCTOU) branch**: force the DB insert to raise `IntegrityError` on `uq_active_transcript_per_section` *after* a successful storage put (e.g., seed an active row between the pre-check and the insert via a service-level seam or mock). Assert: response 409, `storage.delete` called for the loser's object, exactly one active row remains. This exercises the path the pre-check cannot protect.
- storage put fails → no DB row.
- DB insert fails after put → object deleted (compensation), 5xx, orphan key logged.
- `GET` active → `TranscriptMeta`; `GET` none → 404; `GET` as student → 403.

**DB-level / migration**
- partial unique index rejects a second `is_active=true` insert for the same section (the race guard, independent of the service pre-check).
- migration round-trip: `0004` upgrade then downgrade cleanly (table + indexes dropped).

No E2E in 4.1 (no pipeline to traverse).

## Done means

- The `GET` endpoint is lecturer-only; the public DTO exposes no `storageKey`/`checksum`/`isActive`/`supersededAt`.
- No queue, worker job, or status beyond `uploaded` exists anywhere in the change.
