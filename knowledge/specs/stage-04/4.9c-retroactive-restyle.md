## Goal
Restyle the §5-eligible surfaces (login, admin, lecturer content, student view, + the 4.5d–4.7 panels)
onto the 4.9b component library + tokens, **removing inline styles**, **preserving every E2E hook
byte-for-byte**, producing the **live migration inventory**, and proving it with the **full active
Playwright suite** green. Restyle adds NO behavior.

## Why now
The one session that MODIFIES E2E-driven surfaces + removes inline styles (the specificity trap). Kept
isolated (own commit) so a bad surface reverts without losing the component work. The full suite — not
two targeted specs — is the gate (rule 14, umbrella §9/§12).

## Build
1. Restyle each §5-eligible surface onto tokens + components (Button/Input/Card/Badge/Modal/Table/EmptyState/StepProgress). **Remove inline styles.** Map the ~44 hexes → semantic tokens.
2. **Produce the migration inventory LIVE** — one row per surface AT THE MOMENT it's touched (restyled? reason? hooks preserved? inline removed? Stage 12 backlog?).
3. **Preserve byte-for-byte** every `data-testid`, form label (`aria-label`/`getByLabel`), `getByRole` name, `role`/`aria-live`, and the `window.__xyzE2E` bridge. If a hook genuinely must move, update its spec **in the same commit** and call it out per-surface.
4. **No behavior added:** a design-plan state the thin v1 never built (an error/empty it skipped) is a **finding** (in-scope-small or deferred), not silently bundled.
5. Genuine one-off surfaces Stage 5+ never reuses → **LEFT** as-is, logged in the inventory for the Stage 12 sweep (§5 componentize-or-leave).

## Do not build
- New components (4.9b done). New tests/gates/CI (4.9d). Hygiene batch (4.9e). Backend changes. `upload.ts` untouched.
- NO behavior/feature/state additions. NO loosened E2E selector to make a spec pass (rule-10 stop-and-escalate; revert the restyle instead).
