# Stage 4.5.1 — Map-Reduce Summarization (resolves F-4.5-51)

> Completes 4.5's summary capability: removes the single-call transcript-size ceiling that made Option A
> (truncate-and-label, ADR-050) the NORMAL path for real ~60-min lectures. Developer's doc 2026-06-13 is the
> authority; this captures the locked decisions + the sequencing reality.

## Sequencing reality (CONFIRMED against the live roadmap, 2026-06-13)
- AFTER the 4.6 live gate ✅ (4.6 FULLY VERIFIED — fencing/resume available; §6.1 reuses it).
- **BEFORE 4.7 — ALREADY OVERTAKEN: 4.7 is FULLY VERIFIED (shipped).** So the spec's contingency is ACTIVE:
  **4.5.1c MUST re-run 4.7's browser gate AND backfill existing truncated summaries** (§0.1) before Stage 5
  reads any of them. This is the load-bearing consequence of 4.7 having shipped on the Option-A path.
- MVP context: real lectures are ~60 min → over the ceiling → truncation is the NORMAL path today (every
  4.7 summary currently covers ~first 15 min). This is why the ordering/backfill is strict, not best-effort.

## The rule that gates everything (§0.1) — STATE, not label
Downstream (Stage 5 quiz, 7 glossary, 8 assistant) reads DB STATE, never the human label.
`GeneratedLectureSummary.generationStrategy ∈ {single_call | map_reduce | truncated_fallback}`:
- **`map_reduce` + `truncated=false`** is the ONLY state that unblocks quiz generation.
- `truncated_fallback` = dev emergency diagnostic; MUST NOT unblock 5/7/8.
- `single_call` (legacy/short) not consumed by Stage 5 once 4.5.1 ships.
- **Backfill:** any `truncated=true` or `strategy != map_reduce` summary is STALE → dev/admin re-enqueue
  command; QuizDefinition creation REQUIRES `detailed.strategy=map_reduce AND truncated=false` (enforced in state).

## Algorithm (locked, §3)
- **Partition** (4.5.1a): normalize → group CONSECUTIVE segments into the FEWEST map-units each under BOTH a
  char cap AND an estimated-token cap; deterministic; never split a segment; persist `partitionConfigHash`
  (budget + chunkerVersion + transcript checksum); `maxMapUnitsPerTranscript` cost-guard (fail loud on absurd
  counts); optional `carriedForwardContext` per unit (cheapest boundary continuity only).
- **Map** (4.5.1a): one small call/unit → partial structured extraction; DETAILED route (Think/Nvidia);
  BACKGROUND priority (shared limiter); bounded output; **map validation allows marked-empty sections**;
  per-unit AIRequestLog (`feature: detailed_summary_map`), correlated by orchestrating job id; partials
  PERSISTED (resume).
- **Reduce** (4.5.1a): DETAILED route; **§3.3 C1 INPUT GUARD + tiered fallback** — never assume "partials are
  small"; if concat(partials) over budget, reduce in ordered groups then reduce the group-summaries. COMBINE +
  DEDUP + COHERENCE (resolve cross-unit refs via carriedForwardContext) — the hardest prompt. **Reduce
  validation: all sections present/non-empty + a `coverageManifest` asserting EVERY map-unit consumed.**
- **Brief** (4.5.1b): derived from the COMPLETED detailed in ONE small call; **BRIEF route (V2/Cerebras),
  unchanged** (small writing task). DAG change: brief forks from DETAILED not parse (ADR-052); lost
  brief-first ordering accepted.
- **Truncation flag → false** for map_reduce; Option A retained ONLY as logged fallback.

## Rule-15 (ADR-051) — confronted, not buried
Rule 15 ("one call per summary, never per chunk") is deviated, justified + bounded: single-call physically
408s on real input; summary is a ONE-TIME ingestion cost (NOT per-student — quizzes read the generated
summary, never re-summarize, so the exam-week math is unaffected); minimized (coarse partition = fewest calls,
background priority, brief-from-detailed). Budget: ~60-min lecture → ~7–9 calls once, background. Rule 15 stays
law for per-student paths; ADR-051 is the single recorded exception.

## Budget (§1.2) — HEADROOM, not the last passing value
Initial **~8–9K chars, NOT 12K** (12K≈115s is at the 115–145s ceiling = zero margin; 8K≈90s comfortable).
CONFIG value (char cap + estimated-token cap, both); lands in **4.5.1a** (partition can't run without it).

## Data model (§5, additive migration)
- `MapUnitSummary` (new, lean): id, transcriptId, unitIndex, start/endSegmentId, **partitionConfigHash**,
  **sourceTranscriptChecksum**, mapPromptVersion, aiRequestLogId, status, partialContent, timestamps. RESUME
  RULE: a partial is reusable ONLY if partitionConfigHash AND sourceTranscriptChecksum match (C3 — else a
  budget change silently reuses a stale partial covering different segments).
- `GeneratedLectureSummary`: `generationStrategy`; `truncated` (defaults false for map_reduce); compact
  `generationMetadata` { map/reduce/briefPromptVersion, mapUnitCount, sourceMapUnitSummaryIds, coverageManifest }.
- `AIRequestLog.feature` CheckConstraint + `SummaryFeature` type EXTENDED: `detailed_summary_map`,
  `detailed_summary_reduce`, `brief_from_detailed` (migration).

## Failure / resume / fencing (§6)
SEQUENTIAL (`maxMapConcurrency=1`, configurable, hard-cap 2); reduce after all units; brief after reduce.
Resume from first incomplete VALID unit (4.6 resume-from-failed-step). **Stale-write fencing (§6.1, reuses
4.6):** before each map/reduce/brief write, verify lifecycle-eligible + not-superseded + checksum/partitionHash
match → abort clean, write no stale artifact.

## Deterministic adapter / gate (§7, rule 11)
Adapter gains canned map + reduce (incl. tiered) output. TWO new fixtures: a multi-unit (≥2) + a **late-topic
sentinel** (final unit covers `LateLectureSentinelTopic`; reduce output + coverageManifest MUST include it +
the final unit index — provider-free full-coverage proof). Optional dense fixture to trip the §3.3 guard.
CI/Playwright stay deterministic (the §3A gate guard holds).

## UI proof (§9) — full-lecture COVERAGE, not "a summary appeared"
Real-provider smoke on a real ~60-min transcript: (a) detailed references LATTER-portion content (not
first-portion truncated) + eyeball dedup/coherence; (b) AIRequestLog correlated chain by job id — count
SUCCESSFUL rows (retry-robust), per-phase model echo (map/reduce=Think/Nvidia, brief=V2/Cerebras), NO 408s;
(c) capture the REDUCE call's latency + output tokens (largest output — confirm it isn't the next 408).
`truncated=false`, strategy=map_reduce, no label. Browser gate: student reads a full-lecture brief+detailed.
