## Goal
A lecturer can retry failed transcript processing: the backend resumes at the earliest failed step over
the true DAG, reuses valid upstream output, deletes-and-regenerates the rows each step owns, and fences
every destructive write so a stale/superseded worker can never corrupt a newer valid attempt — and each
step records a sanitized failure category the lecturer UI will surface.

## Why now
4.6a made the state machine + provenance safe. 4.6b is the first place the pipeline may be re-run by a
human action. The retry surface and its correctness (idempotency + fencing) are the headline 4.6
capability; the reaper/reconciliation (4.6c) and the lecturer UI (4.6d) consume it.

## Build
- **Retry endpoint** (locked): `POST /modules/{moduleId}/sections/{sectionId}/transcript/{transcriptId}/retry`
  (section-scoped namespace + transcriptId target so it can retry the active transcript OR a failed
  pending replacement; reuses `get_authorized_lecturer_section_context`). Resolves the earliest failed
  step over the DAG, resets those jobs to `queued`, enqueues after commit; returns the projection.
- **DAG decouple** (ADR-46-B): summary jobs fork from **parse** (not embed) — an embed failure no longer
  blocks summaries; parse re-enqueues chunk + brief + detailed (the matrix).
- **Step ownership / delete-and-regenerate**: parse on (re-)run deletes summaries → chunks → segments
  (FK order) then regenerates; chunk deletes chunks; embed rewrites in place; summary is success-only.
- **Fencing**: before any destructive write, re-read job + transcript `FOR UPDATE` and abort if the
  transcript is `superseded` or the job is no longer the running attempt. Add the parse one-active index.
- **Failure taxonomy** (locked: full): parse/chunk/embed set a sanitized `failure_category`
  (storage_missing / unsupported_file→parse / parse_failed / chunk_failed / embedding_failed); the
  status projection surfaces a structured `failureCategory` (one of 9) + `retryable`. Add
  `StorageObjectNotFoundError` so a missing raw file is `storage_missing`, not a generic parse failure.

## Do not build
- Stuck-row reaper / storage reconciliation / `MaintenanceRun` / heartbeat → **4.6c** (the `crashed`
  category is mapped in the projection now but only produced by the 4.6c reaper).
- Lecturer replace/retry UI, active-summary preview endpoint, full browser gate → **4.6d**.
- Targeted `?jobType=` retry → deferred (YAGNI). RQ scheduler enablement (F-4.5-47) → not 4.6.
- Student surface → 4.7. parse/chunk one-active index for **chunk** (coexistence) → stays unbuilt.

## Done means
Retry resumes from the earliest failed step over the DAG; embed failure does not block summaries; summary
retry does not touch chunks/embeddings; per-step delete-and-regenerate with NO duplicate
segments/chunks/summaries; cannot retry a running step; stale/superseded job aborts before any write;
sanitized failureCategory + retryable surfaced; authz enforced; migration round-trips; suite + tsc green.
(Browser gate is 4.6d.)
