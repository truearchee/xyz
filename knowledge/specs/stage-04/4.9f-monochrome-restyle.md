# 4.9f — Monochrome restyle (violet → Apple×Linear)

## What & why
Stage 4.9 shipped the visual system in **violet**. Design Plan v2 + Design System v2 lock a new **monochrome
Apple×Linear** identity. This session repaints the SHIPPED frontend to match `knowledge/design-system.md` —
which is the **single source of truth** for every token value; this spec does not duplicate hexes. It is a
token + shape repaint, **not** a component-API change.

## Build (scope)
1. **Layer 1 palette** (`frontend/src/app/globals.css`): replace the violet/zinc raw palette with the warm-gray
   ramp + functional status raws exactly as listed in design-system.md §2 (Layer 1).
2. **Layer 2 semantic roles**: remap every `--color-*` to the new values in design-system.md §2 (Layer 2),
   including page = `surface-muted` (parchment), graphite `--color-primary`, neutral `--color-info`,
   graphite `--color-focus-ring`. Keep the default Tailwind colour namespace disabled.
3. **Typography**: swap the two-family (Space Grotesk + Inter) setup for the **single system stack** in
   design-system.md §3; apply the new scale (body 17px, reading 19px, weights 400/500/600, tracking values).
   Remove the committed `.woff2` font files + `next/font/local` wiring no longer used.
4. **Radius**: set `--radius-*` to 8/12/16/20; make **all Button variants pill** (`rounded-full`); inputs md(12),
   cards lg(16), modals xl(20), badges pill. Icon-only buttons circular.
5. **Depth**: remove resting-card shadows; cards separate by the parchment→white tone step + hairline. Keep
   `--shadow-md`/`--shadow-lg` and apply them ONLY to overlays (Modal = lg; Toast/popover = md). No gradients.
6. **Motion**: press state `scale(.96)`; keep the reduced-motion guard.
7. **Components** (`frontend/src/components/ui/*`): update only variant→class maps / token usage to the above.
   **Do not change any component's props, names, or behaviour.** The pipeline's failed state stays danger-red
   with explicit "Failed" text.

## Do not build
- No component API changes (no new/renamed props or variants beyond the pill shape).
- No new components, no new screens, no dark mode.
- No backend changes, no DB migration, no OpenAPI change.
- Do not touch `--palette-*` from feature code; all colour stays semantic.

## Verification (run, capture real output)
- `scripts/check-contrast.mjs` — every semantic pair passes AA (4.5:1 body, 3:1 large/UI). Fix tokens until green.
- `check:design-tokens` gate green (no non-semantic colour produced CSS).
- `frontend` type-check green; `vitest` component/a11y smokes green.
- **Full active Playwright E2E suite green** (rule 14) — the restyle broke no behaviour.
- Mobile sanity at 375px: no horizontal scroll on the mobile-first surfaces.
- Visual: re-capture the 4.9 design-match screenshots against design-plan §2.0–§2.3 (developer visual sign-off,
  as 4.6/4.7 were human-stamped).

## Done means
All verification green; the app renders monochrome end-to-end (graphite actions as pills, parchment page, white
cards separated by tone-step, shadow only on overlays, 17px body); component contracts unchanged; then update
`design-system.md` — **remove the TARGET banner and reconcile every value to the shipped CSS** (code wins) — and
flip the design-plan v2 supersession note to "shipped". Close-the-loop per dev-workflow.md.
