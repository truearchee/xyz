## Goal
A lecturer assigned to a module can set/clear a section's plain-text notes and move that section between visibility states (`draft`, `published`, `unpublished`); no student-facing read path is built in this session.

## Section lifecycle & status semantics
`draft` means the section has never been published and is not visible to students.

`published` means visible to students once the Session 3.3 read path exists.

`unpublished` means the section was published at least once and is now hidden.

`draft` is initial-only. No transition leads into `draft`. The API only targets `published` or `unpublished`, so the only illegal wire transition is `draft -> unpublish`; the service must also reject internal `*/ -> draft`.

## Build
1. Update lecturer notes: set or clear `lecturer_notes`; normalize, cap, and store plain text. Notes are editable in any publish state.
2. Publish: transition a section to `published`.
3. Unpublish: transition a section to `unpublished`.
4. State-machine enforcement in the service layer.
5. Authorization pipeline consumes the resolved `CurrentUserContext` and established membership query pattern; it does not reimplement auth lookup.
6. Frontend presentational publish/unpublish control and notes editor, wired through typed content API helpers.

## API changes
Endpoint prefix must match 3.1's content router. Preflight confirmed 3.1 uses module-scoped root routes, so 3.2 uses:

```text
PATCH /modules/{module_id}/sections/{section_id}/notes
POST  /modules/{module_id}/sections/{section_id}/publish
POST  /modules/{module_id}/sections/{section_id}/unpublish
```

Because routes include both IDs, the service validates `module_id` and `section_id` coherence. Mismatches return `SECTION_NOT_FOUND`.

DTO casing matches 3.1 camelCase:

`UpdateSectionNotesRequest`
- `lecturerNotes: string | null`
- No `max_length`, regex, or business constraints on the DTO.

`SectionDetail`
- `id`
- `courseModuleId`
- `title`
- `type`
- `publishStatus`
- `lecturerNotes`
- `updatedAt`

Transition table:

| From | publish | unpublish |
|---|---|---|
| `draft` | `published` 200 | 422 `SECTION_TRANSITION_INVALID` |
| `published` | 200 no-op | `unpublished` 200 |
| `unpublished` | `published` 200 | 200 no-op |

Error codes use the existing FastAPI error envelope with the code in `detail`:

| Code | HTTP | When |
|---|---|---|
| `CONTENT_FORBIDDEN` | 403 | caller is not a lecturer |
| `SECTION_NOT_FOUND` | 404 | section missing, inaccessible, or module/section mismatch |
| `SECTION_ARCHIVED` | 409 | accessible section exists but `status = archived` |
| `SECTION_TRANSITION_INVALID` | 422 | illegal state transition |
| `SECTION_NOTES_TOO_LONG` | 422 | normalized notes exceed cap |

## Authorization & validation order
1. Authenticated and active through `get_current_user`.
2. Role is `lecturer`, else `403 CONTENT_FORBIDDEN`.
3. Resolve section scoped to modules where the lecturer has an active lecturer membership. Do not filter archived rows out here. Missing, inaccessible, or mismatched module/section returns `404 SECTION_NOT_FOUND`.
4. If resolved section `status = archived`, return `409 SECTION_ARCHIVED`.
5. For publish/unpublish, validate transition legality.
6. For notes, normalize and cap-check after authz.
7. Persist only if the stored value actually changes; no-op returns before mutating ORM fields or flushing.

## Worker / job changes
None. Synchronous request-path operations only.

## Concurrency
MVP is last-write-wins. No optimistic locking, version columns, or ETags. The response always returns canonical persisted state. `lecturer_notes` is a single shared field per section.

## Notes handling
- Accept `string | null`.
- Normalize server-side: CRLF/CR to LF; trim leading/trailing whitespace; preserve internal line breaks.
- `null` or whitespace-only clears notes to `NULL`; never store `""`.
- Enforce the 5,000 character cap after normalization in service/validator.
- Store and render plain text only. Frontend must not use `dangerouslySetInnerHTML`.
- `updatedAt` changes only when the normalized stored value changes.

## Tests required
Unit/service:
- Full transition table.
- Illegal `draft -> unpublish` and internal `*/ -> draft`.
- Same-state publish/unpublish no-op issues no DB write and does not change `updatedAt`.
- Notes normalization, null/whitespace clearing, boundary length, over-cap error.
- Same normalized notes no-op issues no write and does not change `updatedAt`.
- Archived section blocks publish/unpublish/notes.

Integration:
- Assigned lecturer round trip: `draft -> publish -> unpublish -> re-publish`.
- Assigned lecturer set notes and clear notes via null and whitespace-only.
- Student/admin publish/notes return `403 CONTENT_FORBIDDEN`.
- Inactive lecturer is blocked.
- Lecturer hitting inaccessible, missing, or mismatched section returns `404 SECTION_NOT_FOUND`.
- Assigned lecturer on archived section returns `409 SECTION_ARCHIVED`.
- Unauthorized over-cap notes returns `403` or `404`, not framework `422`.
- Authorized over-cap notes returns `422 SECTION_NOTES_TOO_LONG`.
- Empty-section publish succeeds.

## Locked decisions
1. Empty sections are publishable.
2. Use action endpoints.
3. Notes cap is 5,000 characters after normalization.
4. `draft -> unpublish` is invalid; same-state transitions are idempotent no-ops with no write.
5. No `published_at` or publish history in MVP.
6. Any active assigned lecturer may edit notes and publish/unpublish.
