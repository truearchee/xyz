## Goal
Lab sections accept download-only attachment assets, starting with `.ipynb`, while existing PDF section assets remain processable-but-inert and the backend proves attachments create no transcript or AI pipeline rows.

## Why now
5.5b made lab `due_at` editable. 5.5c completes the backend model for lab material upload: asset kind, upload-time lab deadline setting, server-side attachment validation, and safe backend-served downloads.

## Build
- Add `SectionAsset.asset_kind` with values `processable` and `attachment`, defaulting to `processable`.
- Add migration `0021` with `down_revision = "0020"` that backfills existing `section_assets` rows to `processable`.
- Keep existing PDF section assets on the signed-URL path as `processable`.
- Allow at least `.ipynb` uploads to lab sections as `attachment`.
- Validate uploads by extension and content type, with existing size caps; notebooks must never be executed or rendered.
- Set attachment `processing_status` to `completed` immediately.
- Add backend streaming download for attachment assets with `Content-Disposition: attachment` and `X-Content-Type-Options: nosniff`, reusing the existing role/resource authorization boundary.
- Preserve signed URL downloads for processable PDFs and prevent attachment assets from being served through signed URLs.
- Allow assigned lecturers to set lab `due_at` during material upload, using the same lab-only rule as metadata edit.
- Add deterministic backend/DB tests proving attachment upload creates no `transcripts`, `ingestion_jobs`, `transcript_segments`, `transcript_chunks`, or `generated_lecture_summaries` rows.

## Do not build
- No product UI for lab uploads/downloads or deadline display; 5.5e owns the browser proof.
- No notebook parsing, rendering, execution, chunking, embedding, summarization, or AI job enqueueing.
- No new processable non-PDF pipeline.
- No student submissions, grading, overdue/late computation, quiz generation, or schedule reseed.
- No section add/delete/reorder/regeneration.

## Data model changes
- `section_assets.asset_kind` text column with allowed values `processable` and `attachment`.
- Existing rows are backfilled to `processable`.

## API changes
- Section asset upload accepts both PDF processable assets and `.ipynb` lab attachments.
- Section asset responses expose `assetKind`.
- New backend attachment download endpoint returns file bytes with download-only headers.
- Existing signed download URL endpoint remains for processable PDFs and rejects attachments.
- Multipart upload may include optional lab `dueAt`/`due_at` for lab deadline setting.

## Worker / job changes
None. Attachments are structurally excluded from transcript and AI processing.

## Authz rules
- Assigned lecturers may upload assets and set `due_at` only for their assigned module sections.
- Students may download published attachment assets only through the same published-section boundary used for existing content downloads.
- Cross-module and unassigned lecturer resource access remains 404; role denial remains 403.
