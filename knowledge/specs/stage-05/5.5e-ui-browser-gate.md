## Build
- Add a dry-run module schedule preview API that calls the same `generate_sections` pure function used by persisted module creation. It must report the reference oracle: 7 weeks, 21 lectures, 7 labs, 0 Friday sections, 28 generated sections.
- Add resolver-backed section-by-week APIs for admin and assigned lecturer curation. These APIs must call `resolve_sections_by_weeks(..., include_unstamped=true)` for the by-week UI, not regroup a generic section list in the client.
- Add thin admin module creation UI for course dates, weekday pattern, quiz day, preview, and create.
- Add thin admin/lecturer by-week UI showing stored `week_number`, `session_date`, and `due_at`, with inline metadata edit.
- Add thin lecturer lab upload UI that can upload PDF and `.ipynb` files and set `due_at` at upload time.
- Add student lab material UI that shows lab deadlines and downloads PDFs through signed URLs and attachments through the backend download endpoint.
- Add a Stage 5.5e Playwright browser gate proving:
  - admin creates the reference module;
  - preview shows 7 weeks / 21 lectures / 7 labs / 0 Friday;
  - persisted result has exactly 28 sections, grouped by the resolver;
  - missing schedule module creation returns 422;
  - lecturer metadata edit recomputes week number when only `session_date` changes;
  - lecturer has no add/delete/reorder UI and backend creation/deletion/reorder is rejected or absent;
  - lecturer uploads PDF + `.ipynb` to a lab section with `due_at`;
  - student sees and downloads both files and sees the deadline;
  - `resolveSectionsByWeeks(module, [N])` returns exactly that week's lecture/lab rows.
- Run the full active Playwright suite after the 5.5d reseed and report the exact count.
- Check the Alembic migration seam. If Stage 5 `0014-0019` is not present on `origin/main`, record the `0020 -> 0021` rebase as pending and do not fake a re-parent. If it is present, rebase `0020.down_revision` to the real current head, assert one `alembic heads`, and rerun the fresh-DB round-trip.
- Stop sibling stacks, including `kyiv`, before the browser/full-suite run so `stage-55` owns `:8000` and `:3000`.
