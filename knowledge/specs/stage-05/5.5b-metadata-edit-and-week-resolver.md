## Goal
Admins and assigned lecturers can edit stored section metadata (`week_number`, `session_date`, `due_at`)
without changing section structure, and Stage 6 has a read-only `platform/query` resolver that returns
lecture/lab section IDs by stored week metadata.

## Why now
5.5a made generated sections schedule-stamped. 5.5b makes that metadata curatable and exposes the
read model Stage 6 will consume for recap/exam-prep quiz scope.

## Build
- Add a metadata-edit endpoint for existing module sections.
- Enforce an exact patch whitelist: only `week_number`, `session_date`, and `due_at`.
- Enforce D13: if `session_date` is patched without `week_number`, recompute `week_number` from the
  module's stored course anchor; if both fields are supplied, explicit `week_number` wins.
- Allow edits for admins and assigned lecturers only.
- Preserve the codebase's authorization boundary: role denial returns 403; cross-module or unassigned
  lecturer access returns 404 to avoid existence leaks.
- Restrict `due_at` edits to lab sections.
- Add a read-only `platform/query` resolver over stored `week_number`/`session_date`, including
  `includeUnstamped` modes.

## Do not build
- No section add/delete/reorder/rename.
- No schedule config edit or regeneration after creation.
- No asset_kind, attachments, lab upload due_at wiring, or download proxy (5.5c).
- No reseed (5.5d), frontend by-week UI, browser gate, or Stage 6 quiz generation.
- No overdue/late-state computation from `due_at`.

## Data model changes
None. 5.5b uses existing `module_sections.week_number`, `module_sections.session_date`, and
`module_sections.due_at`.

## API changes
- New PATCH endpoint for section metadata under the existing module/section content boundary.
- Request accepts only `weekNumber`, `sessionDate`, and `dueAt`.
- Response returns the updated section read model.

## Worker / job changes
None.

## Authz rules
- Admin: may edit metadata on any module section through a valid module/section pair.
- Assigned lecturer: may edit metadata only for assigned modules.
- Student/non-authorized role denial: 403.
- Cross-module section IDs and lecturer-not-assigned resources: 404.
