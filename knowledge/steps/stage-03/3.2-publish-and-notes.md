## Deviations from spec
- The spec required the standard sequence `build -> up -> alembic -> content tests -> full backend -> codegen -> frontend tsc -> diff check`. Because the compose setup copies frontend source into the image and does not mount it, codegen on the host made the running frontend container stale. I rebuilt/restarted only the frontend after codegen, then reran the required frontend type-check successfully.
- No schema migration was needed. Preflight confirmed `module_sections.publish_status`, `lecturer_notes`, and `status` already matched the locked shape.
