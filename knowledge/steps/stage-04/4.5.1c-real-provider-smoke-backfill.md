## The hardening journey (5 smokes — each a real finding)
- **Smoke #1 — the 408 ceiling is SOLVED.** Every map call HTTP 200 (~7700-char units, ~33s); the full 47KB
  that 408'd as a single call no longer does. NEW wall: **F-4.5.1c-1** — K2-Think-v2 (a reasoning model)
  spent its whole token budget narrating ("We need to produce…") and hit `finish_reason=length` BEFORE
  emitting JSON → `not_json` on ~33% of map calls → jobs failed. Diagnosed by reproducing one map call with
  and without JSON mode.
- **Fix (committed `1ee98a6`):** JSON mode default ON (the diagnostic CONFIRMED K2-Think-v2 honors
  `response_format=json_object` on BOTH routes — clean JSON, `finish_reason=stop`) + a hardened map prompt
  v2 + a bounded in-call invalid_output retry + the brief fork force-requeues.
- **Smoke #2/#3 — map phase SOLID** (every map call clean, partials rich across the whole lecture: one-sided
  limits, indeterminate forms, ~16 worked examples). NEW wall: the REDUCE. **F-4.5.1c-2** (tiered reduce
  non-convergence — faithful-merge doesn't compress, so tiering looped) and **F-4.5.1c-3** (the LLM-merge
  reduce compressed a 13KB rich-partial input down to a ~900-char intro-only summary that dropped the
  required `examples` key — the model's summarize prior overrode every "merge faithfully / all keys"
  instruction, in v1 AND v2).
- **Fix (committed):** the reduce was REDESIGNED to **programmatic union + LLM overview** — union+dedupe the
  map partials' structured lists in code (coverage GUARANTEED by construction; no LLM compression can drop
  map-extracted content) and call the model ONLY for the short overview it does reliably. This obsoleted the
  tiered reduce + the reduce-input budget (F-4.5.1c-2 dissolved). + an interim reduce-budget raise (16000)
  and reduce-prompt v2 are now superseded/dormant but kept in history.
