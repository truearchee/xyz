## Summary
Shipped the gated admin-only `/internal/sse-probe` (C1) with the §7.C2 constraints as acceptance;
made the §8 frontend-hook absence a **byte-clean** property of the **Dockerfile.prod** artifact (O1
exclude-from-prod); reframed the §8 backend proof to the surfaces that exist; and proved
`check-staging-env` is a **hard** deploy gate. CORS staging origin is a config value (middleware
already reads `CORS_ORIGINS`). The hosted browser checks (SSE progressive over D1; `__xyzE2E`
undefined in the staging browser; CORS preflight) are the developer's run.

**O1 validated empirically:** the runtime gate **alone** is NOT byte-clean — a prod build with
`NEXT_PUBLIC_E2E_TEST_HOOKS` unset still left `__xyzE2E` in **5** `.next/static` files (Next's minifier
did not DCE the hook). The prod-only `NormalModuleReplacementPlugin` (replacing the e2e modules with a
no-op stub) makes the deployed bundle byte-clean. Implementation gotcha recorded (F-4.8c-2): the regex
must match the **resolved `.ts` path**, not just the bare `../e2e/testHooks` request, or it only
partially fires (server/some chunks retain the hook).

**Full active Playwright suite — deferred to the 4.8d close per rule 14.** 4.8c has **no runtime
effect on existing e2e paths**: the probe is gated-off by default (no route) and the webpack exclusion
is `next build`-only (`if (!dev)`), so `next dev` (the e2e suite) is untouched by construction; the
421 backend run confirms no import/behavior regression.

## Decisions made (O-rulings)
O1 exclude e2e modules from the prod build (byte-clean); O2 `ENABLE_INTERNAL_SSE_PROBE` allowed ON in
staging (gated + admin-only; `check-staging-env` does NOT forbid it; off in a future prod); O3
admin-only probe auth.

## Key reconciliations (surfaced)
- **No backend fault-injection HTTP route exists** → the §8 backend proof is flags-absent +
  fail-loud-boot + no-HTTP-surface, NOT "404 on a fault route" (there is none).
- **The runtime hook gate is not enough for build-time absence** → O1 exclusion is genuinely required
  (the empirical 5-files-dirty result), binding the proof to the Dockerfile.prod artifact (F-4.8a-9).

## Risks / follow-ups (developer-run at deploy, into the report)
- Hosted: staging browser hits `/internal/sse-probe` over D1 → chunks arrive **progressively, not
  buffered** (the SSE de-risk); `window.__xyzE2E` undefined in the staging browser; the staging
  frontend origin passes CORS preflight (`OPTIONS` + `Authorization`/`Content-Type`/`Idempotency-Key`).
- `allow_credentials=True` removal stays **4.9**.
