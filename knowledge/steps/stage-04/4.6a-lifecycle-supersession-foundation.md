## Summary
Built the Stage 4.6 foundation. A replacement transcript now processes as `pending` alongside the
still-`active` old one and atomically swaps in only on completion; `is_active` is gone, replaced by a
3-state `lifecycle_state` with supersession lineage. Added the domain eligibility predicate + a
read-only resolver, per-row job provenance (incl. a separate embedding-creator column), and an
env-gated pipeline fault-injection harness. **No retry, reaper, reconciliation, MaintenanceRun, UI, or
browser gate** — those are 4.6b/c/d. Backend suite green (**305 passed**, +16 new); migration 0010
round-trips on a fresh DB; frontend `tsc --noEmit` clean.

## Deviations from spec
1. **parse/chunk one-active indexes DEFERRED to 4.6b** (spec/plan had "add now"). A one-active-chunk
   partial-unique index breaks the existing, tested `test_same_transcript_different_chunk_keys_serialize_replacement`
   path (chunk legitimately keeps two chunk jobs queued and serializes at runtime). Embed/summary
   one-active indexes already exist; the parse/chunk "current job" fencing pointer is designed in 4.6b
   with the retry that consumes it. Recorded as a spec Amendment.
2. **Preview HTTP endpoint not built** (deferred to 4.6d per spec §6); the resolver ships with unit coverage.

## Risks introduced
- **Two ordering/FK correctness fixes the tests surfaced** (real bugs, now covered): (a) activation
  must `flush()` the old-active demotion before promoting the pending or the one-active index trips
  (unit-of-work does not preserve assignment order); (b) the discarded pending's
  `superseded_by_transcript_id` points at the not-yet-inserted new row → demote with NULL pointer →
  insert → back-fill.
- **Dev DB drift window:** `xyz_lms` (dev/running stack) is intentionally left at migration 0009. The
  running containers still run the pre-4.6a image (expects `is_active`); applying 0010 there before a
  coordinated rebuild+deploy would break the running backend. Must be applied with the deploy.
- Superseded pendings leave orphaned storage objects until 4.6c reclaims them (by design).
