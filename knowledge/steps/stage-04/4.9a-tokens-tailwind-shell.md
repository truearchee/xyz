## Summary
Tailwind v4 adopted via `@tailwindcss/postcss` with the two-layer token system as the single styling
source of truth; self-hosted Space Grotesk + Inter via `next/font/local`; shell infrastructure stood up
(portal mount points, role-aware token-styled chrome + skip-link, reserved assistant anchor, token-styled
error/404/403 surfaces). No public components, no feature-surface restyle, no tests (those are 4.9b–d).
All verification green; the shell E2E smoke (4.3.5b + 4.3.5c) passes with every selector preserved.

## Decisions (ADRs)
- **adr-044** — Tailwind v4 via PostCSS (repo fit confirmed; no v3 fallback needed); **fonts = LOCAL**
  `next/font/local` with committed `.woff2` (verified in host build AND the rebuilt e2e dev image before
  the ADR line was written, per the developer's ordering — no build-time egress); `@source` detection;
  client/server boundary; default-namespace disable.
- **adr-045** — two-layer raw/semantic tokens; default color namespace disabled (build-enforced, proven);
  spacing keeps v4 default (4.9c won't relitigate); tonal vs solid status rule; `--color-border-strong`.
- **adr-046** — React Aria (headless) + lucide-react (icons); deps installed 4.9a, consumed 4.9b.
