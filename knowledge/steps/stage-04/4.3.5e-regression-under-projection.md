## Scope
Checkpoint C changed the 4.3.5e browser-gate terminal proof from the coarse transcript row status alone to the C-lite processing-status projection.

Expected projection state for the current Stage 4.1-4.3 pipeline:

```text
overallState === "chunked"
steps.chunk.status === "completed"
```

No schema preflight, pgvector dependency, embedding worker, vector index, summaries, K2Think, `AIRequestLog`, `LLMProvider`, `PromptRegistry`, or retry UI work was started.

## Implementation Evidence
- Added `GET /modules/{module_id}/sections/{section_id}/transcript-processing-status`.
- Projection derives from `Transcript`, `IngestionJob`, `TranscriptSegment`, and `TranscriptChunk`.
- Frontend polling uses `api.transcripts.getProcessingStatus(...)`, which wraps the generated OpenAPI client method.
- The 4.3.5e Playwright gate waits for a successful projection response with `overallState === "chunked"` and `steps.chunk.status === "completed"`.
- `safeFailureMessage` is now a bounded public message derived from failed step name, not raw `IngestionJob.error_message`.

## G.1 Supersession — Active Embedding Lane
Checkpoint G exposed that the original Checkpoint C assertion became stale after the Checkpoint E/F embedding lane was enabled.

Previous assertion:

```text
overallState === "chunked"
steps.chunk.status === "completed"
```

Observed Stage 4.4 full-stack behavior:

```text
steps.parse.status === "completed"
steps.chunk.status === "completed"
steps.embed.status === "completed"
overallState === "embedded"
chunkCount === 1
embeddedChunkCount === 1
safeFailureMessage === null
```

Disposition:

- Do not introduce a pre-embedding mode for Checkpoint G.
- 4.3.5e owns proof that parsing and chunking still work.
- 4.4 owns proof that downstream embedding reaches the latest terminal state.
- Earlier-stage regressions assert step completion; current-stage browser gates assert terminal `overallState`.

New 4.3.5e regression contract:

```text
steps.parse.status === "completed"
steps.chunk.status === "completed"
segmentCount > 0
chunkCount > 0
overallState in {"chunked", "embedding", "embedded"}
safeFailureMessage === null
```

The Playwright gate no longer asserts:

```text
overallState === "chunked"
```
