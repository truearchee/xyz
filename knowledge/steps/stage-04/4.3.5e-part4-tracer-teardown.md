## Checkpoint D behavior
- Added `tests/e2e/fixtures/run-manifest.mjs`.
- Manifest path: `tests/e2e/.runs/{runId}.json`.
- Manifest supports auth users, app users, modules, sections, memberships, assets, transcripts, transcript segments/chunks, ingestion jobs, storage keys, and creation time.
- Added helper functions for manifest creation and for recording storage keys, transcript IDs, and asset IDs.
- Updated `seed.mjs` to create an empty run manifest for each E2E run.
- Static standing seed users/modules/sections remain outside the manifest and are not teardown-owned.
- Added `tests/e2e/fixtures/teardown.mjs`.
- Teardown refuses unsafe targets, loads only one manifest, validates UUID IDs, deletes only manifest-recorded artifacts, and ignores missing rows.
- Storage cleanup uses exact backend-shaped keys only:
  - `modules/{moduleId}/sections/{sectionId}/assets/{assetId}/{file}.pdf`
  - `modules/{moduleId}/sections/{sectionId}/transcripts/{transcriptId}/{fileName}`
- Teardown deletes storage objects first, then transcript chunks, jobs, segments, transcripts, assets, sections, memberships, modules, app users, and auth users.
- Teardown is idempotent.
- Remote Supabase teardown requires `E2E_DESTRUCTIVE_TARGET_CONFIRM=dedicated-e2e-supabase`; local Supabase does not.
