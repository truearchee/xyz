## Summary
Thin student lecture/lab surface: a section detail page renders title/notes/materials immediately and
brief + detailed summaries INDEPENDENTLY, each in one of four per-slot states (LOADING / GENERATING /
UNAVAILABLE / NOT_APPLICABLE) + READ_ERROR, with bounded backoff polling (generating-only, unmount-safe,
wall-clock cap). Summary content is server-rendered markdown shown via `react-markdown` with raw HTML
disabled (S1). The module page shows a coarse per-section summary flag (one batched call, no fan-out) +
a link to each section page. Browser gate G1–G9 green against the real backend.

## Deviations from spec
- **§17 accepted-with-rationale: the full suite must run `--workers=1` (serial).** A single
  `embedding_worker` is overwhelmed when Playwright runs the pipeline specs in parallel (its default),
  producing false `embedding_failed` reds. Serial is correct for this stack; the e2e overlay
  (`-f docker-compose.e2e.yml`) is also required (see findings + the e2e memory).
- **§17 accepted-with-rationale: G2 brief-first and G7 unavailable use SEEDED states** (§14):
  `seedDetailedSummaryGenerating` (a deterministic DB transform of a fully-summarized section) for G2,
  and a no-transcript published lecture (row 5) for G7. Both avoid timing races. Row 4 (terminal summary
  failure) is additionally covered by the backend integration test `test_row4_summary_failed_is_unavailable`.
- **react-markdown used as specified (ADR-4.7-4).** Raw HTML disabled (no `rehype-raw`), img/iframe/script
  disallowed, link protocols gated. Pre-existing Next.js 15.3.3 advisories surfaced by `npm audit` are a
  Stage 4.9/12 hygiene item, not introduced here.
