## Summary
Repainted the shipped frontend from the v1 **violet** system to the v2 **monochrome Apple×Linear** identity — a
token + shape repaint, **no component-API change**. Layer 1 raw palette → warm-gray ramp + green/amber/red status
raws; Layer 2 semantic roles remapped (graphite `primary`/`focus-ring`/`info`, parchment page `surface-muted`,
added `text-subtle`, gray-800 overlay scrim); one **system font family** (dropped `next/font` + both `.woff2`);

new `--radius-xl`); **all buttons pill** (`rounded-full`) + `scale(.96)` press; **resting-card shadows removed**
(`--shadow-sm` deleted; shadow only on overlays); Modal radius → `xl`; `info` tone renders neutral. The scope was
extended (spec Amendment) to sweep `frontend/src/features/**` for the same pill/shadow/weight/reading treatment so
the app is monochrome end-to-end. The contrast gate was rewritten to the monochrome semantic pairs.

## Decisions / findings
**Findings:**
- **F-4.9f-1 — Summary-content E2E specs are RED at the 4.9-branch baseline, independent of the restyle.**
  `4.5d-summary-browser` (+ `4.7-student-summaries`, the two fault gates) assert `BRIEF_MARKER = 'core ideas of
  the topic'`, but the deterministic adapter now emits *"This brief was distilled from the session's detailed
  summary…"* — changed by **Stage 4.5.1b (commit `e5f2a78`, brief-from-detailed DAG)**, which is layered on the
  4.9 branch. Proven: the spec files are unchanged (empty diff), my backend diff is empty, and the **same spec
  fails identically on the violet baseline frontend** (rebuilt + run: same expected/received). The recorded "full
  suite 11/11" was at 4.9e (`b99e879`), BEFORE 4.5.1 was layered on. Fixing it needs backend/spec changes that
  4.9f excludes ("No backend changes"). → developer-owned: refresh the spec markers (or adapter) in a backend session.
- **F-4.9f-2 — `teardown.mjs` lacks a `map_unit_summaries` delete; my E2E runs left 13 e2e-owned orphan modules.**
  The 4.5.1 `map_unit_summaries` table RESTRICT-references `ai_request_logs`, so the `ingestion_jobs`→`ai_request_logs`
  cascade is blocked and teardown FK-errors (another pre-4.5.1-gap surfaced by running the suite on the 4.9 tip).
  A scoped purge SQL was **denied by the safety classifier** (mass DELETE on a shared DB, out of restyle scope) and
  NOT worked around. → developer-owned: purge the e2e orphans (F-4.9-1 mechanism) + add a `map_unit_summaries`
  delete step to `teardown.mjs`.
- **Type scale realized via Tailwind numeric `--text-*` tokens** (not named tokens) to avoid API churn; the
  19/1.65 long-form reading line-height is applied at the student summary surface (`SummaryMarkdown`), not by
  loosening the generic `text-lg`. **`info` → neutral** done at the token + variant-map level (`border-border` +
  neutral tonal greys) with no BadgeTone/API change. **`--font-display` aliased** to the system stack to keep ~25
  `font-display` heading usages resolving with zero feature churn. **`--color-text-subtle`** added; intentionally
  not contrast-gated (disabled/hint, WCAG-exempt).

**Decisions:** base = new `stage/4.9f` off `stage/4.9-frontend-foundation`; scope extended to feature surfaces
(Amendment). No new ADR (no durable cross-cutting decision beyond the design-direction already in design-plan v2).
