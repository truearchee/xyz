## What shipped

### §7 hygiene batch (4 items)
- **§7.1 httpx → `ASGITransport`.** The deprecated `AsyncClient(app=app, …)` shortcut lived in **three** test
  files, not one: `backend/tests/conftest.py` (the `auth_client`/`probe_client` fixtures),
  `backend/tests/test_health.py` (4 direct clients), `backend/tests/test_sse_probe.py` (the `probe_client`
  fixture on a `create_app()` app). All migrated to `AsyncClient(transport=ASGITransport(app=app), base_url=…)`.
  **Behaviour-identical** (both run the ASGI app in-process). *The first close-out gate caught that fixing
  conftest alone left 6 warnings (test_health ×4 + test_sse_probe ×2) — recorded honestly, then fixed.*
- **§7.2 CORS — dropped `allow_credentials=True`** in `backend/app/main.py` (auth is pure Bearer, rule 4 —
  the browser never sends cookies cross-origin). The `test_health.py` CORS tests (allowed-origin echo,
  preflight, unlisted-origin reject) still pass with credentials off; the full cross-origin E2E suite confirms.
- **§7.3 `gen:api` alias** — `frontend/package.json` `"gen:api": "bash ../scripts/generate-api-client.sh"` +
  README note. `upload.ts` sanctioned direct-fetch exception untouched.
- **§7.4 E2E data-hygiene PREVENTION (F-4.9-2)** — `tests/e2e/fixtures/check-orphans.mjs` (pre-run fail-loud,
  excludes the seed's standing `e2e_module` fixture) + `tests/e2e/run-active-suite.sh` (`trap cleanup EXIT` =
  unconditional/finally teardown, supports the 3-part fault orchestration — a per-invocation Playwright
  globalTeardown would have wiped the seed between invocations; that approach was built then deleted).
  **DEMONSTRATED:** clean → exit 0; inject one orphan module → exit 1 "FAILED"; delete → exit 0 again.
