## Summary
Built the thin UI slice that closes Stage 4.6. **Active-summary preview endpoint** (lecturer-only, over
`ActiveTranscriptSummaryResolver`) exposes `activeTranscriptId` + brief/detailed content + eligibility flags
+ `hasPendingReplacement` — NO student surface. **Lecturer UI** on the 4.5d surface: a **Replace** button
(file input + inline confirm, with the explicit **double-replacement warning** when a pending exists), a
**Retry** button (shown on `failed` && `retryable`, targets the active transcript), a **per-step states**
strip + sanitized reason, and a **"new version processing"** badge driven by a preview poll that also
detects the atomic swap and refreshes to the new active. **Browser gate** spec (`4.6d-replace-retry.spec.ts`)
drives RETRY FLOW (forced step failure → retry → summarized, no duplicates; embed failure does not block
summaries) and REPLACEMENT CONTINUITY (preview stays on the active while pending, flips to v2 on swap, old
superseded with lineage). Fixed the two cross-stage e2e breaks rule 14 needs. Verified by **backend pytest
349 passed (+4)**, **frontend tsc clean**, **client regen**, and **`npx playwright test --list` (9 specs
compile)**. The **live browser gate is deferred to the developer** (it deploys 4.6 + migrates the dev DB).
