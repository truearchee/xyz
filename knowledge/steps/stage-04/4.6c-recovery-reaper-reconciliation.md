## Summary
Closed the carried recovery debt. The **stuck-row reaper** re-enqueues never-enqueued / stuck-queued work
and marks genuinely-crashed jobs `failed`+`crashed` (fenced), step-aware + RQ-registry/age-aware,
singleton-locked, action-capped. **Storage reconciliation** diffs the bucket against DB transcript keys —
report-only by default, grace-windowed, prefix-scoped, deletion-capped; superseded files are referenced
(never orphans); a DB ref with no object is reported loudly and never auto-fixed. Every run writes a
**MaintenanceRun** (migration 0012). Both run at worker startup (reaper) / admin trigger (admin-only).
**No lecturer UI / browser gate (4.6d).** Backend **344 passed** (+15); migration 0012 round-trips fresh;
frontend `tsc` clean.

## Deviations from spec
1. **Liveness = RQ registry + age, no heartbeat columns** (spec §6 sanctions "or rely on RQ registry").
   embed/summary checked via stable RQ job_ids; parse/chunk by age. Recorded as a spec amendment.
2. **Reaper at startup; reconciliation admin-trigger-only** by default (`RECONCILE_AT_STARTUP` flag opts in)
   — keeps worker boot off the recursive storage-listing path. Minor deviation from "both at startup".

## Risks introduced
- **Advisory lock leak risk** handled: the lock is held on a dedicated connection and always
  `pg_advisory_unlock`-ed before the connection returns to the pool.
- **Dev DB drift** (carried): dev `xyz_lms` is at 0009; apply 0010 + 0011 + 0012 together at the next deploy.
- **Reconciliation listing cost** on Supabase (recursive) is capped; a hit cap skips missing-ref detection
  (`capped=True`) to avoid false data-loss alarms.
