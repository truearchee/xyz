## Reconciliation notes (folded into the locked v2 spec — no longer open)

Two items surfaced during the first report as caveats; both were reconciled into the **locked v2
spec** rather than changed in code:

1. **`steps.summary_detailed` literal is `not_started`** (the established no-job sentinel for *every*
   step), not a distinct `'pending'`. **Spec §5 patched** to assert `not_started` explicitly ("NOT
   'pending' or 'failed'"). A distinct `pending` value remains a 4.5d UI/contract concern (§4); not
   introduced here. The verifiable invariants hold: no detailed job row, no detailed AIRequestLog
   row, not `failed`, overallState `summarizing`.

2. **No type checker is configured in the repo** (no mypy/pyright/ruff). **Spec §15/§16 patched** to
   define the 4.5b type-check gate as `compileall` clean + full `pytest` collection, with a real
   mypy/pyright pass **deferred to 4.9 hygiene**. OpenAPI client unchanged — no API contract field
   added or modified (new columns are internal provenance; `summarizing` already existed).
