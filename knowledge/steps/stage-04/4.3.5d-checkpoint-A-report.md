## Behavior implemented
- Lecturer home assigned-module cards now link to `/lecturer/modules/{moduleId}`.
- The module detail route is thin and renders `LecturerModuleDetail`.
- `LecturerModuleDetail` loads module metadata through `api.modules.get`.
- `LecturerModuleDetail` loads section list rows through `api.content.listSections` and fetches per-section detail through `api.content.getSection`.
- Per-section detail supplies `publishStatus` and `lecturerNotes` without a backend projection change.
- `SectionNotesEditor` initializes from backend notes, shows saving state, and calls back to save notes.
- Notes save uses `api.content.updateNotes`, which wraps `ContentService.updateNotesModulesModuleIdSectionsSectionIdNotesPatch`.
- On successful notes save, the UI re-fetches module/section data before showing the persisted state.
- On failed notes save, the section editor renders an alert.
