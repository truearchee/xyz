### Key implementation decisions / deviations
- **`AIRequestLog.ingestion_job_id` is NOT NULL** in 4.5a (all callers are job-tied summaries). Stage 8
  assistant calls will relax this via migration when non-job gateway calls arrive.
- **`backend/prompts/`** (not repo-root `prompts/`) and **`backend/scripts/reenqueue_summaries.py`** —
  the backend Docker build context is `./backend`, so repo-root dirs wouldn't ship in the image
  (adr-026). `LLM_PROMPTS_DIR` overrides.
- **`docker/worker-ai.Dockerfile` → `ai_worker` compose service** reusing the backend image (the repo
  has no per-worker Dockerfiles).
- **D2 estimator = `chars/3.5`** (tiktoken not a dependency; the over-counting fallback D2 permits).
- **`summarizing` overallState** added beyond the literal spec word `summarized`, for the passive
  "generating" badge (§13). Flagged for developer confirmation (see open-questions).
- **rate_limited** failures are marked terminal `failed`+`failure_category='rate_limited'` in 4.5a
  (no in-call backoff loop yet); RQ retry is reserved for `provider_transient` + bounded `invalid_output`.
