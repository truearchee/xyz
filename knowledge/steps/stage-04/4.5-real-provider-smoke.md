## CORRECTION + new finding (2026-06-13, later) — F-4.5-NN: large-transcript single-call 408 (both routes)
The smoke above verified the real provider on the **`ensemble-methods.vtt` fixture — only ~600 chars / ~150
tokens.** Regenerating the **real `Checkpoint A` lecture transcript (46,403 chars / ~11,600 tokens)** revealed
a limit the tiny fixture hid: **BOTH routes return HTTP 408** (Cerebras brief + Nvidia detailed) — the
provider's server-side request timeout. K2-Think-v2 is a reasoning model (thinks inline before answering);
reasoning over ~11.6K input tokens exceeds the provider's per-request time budget on either route.
**Context fitting does NOT help** (`LLM_CONTEXT_FALLBACK_ENABLED` is on; 11.6K tokens *fits* the context
window — the wall is processing TIME, not window size). So **single-call summarization has a transcript-SIZE
ceiling**: it works for short transcripts, 408s on real-sized lectures.

Diagnosed during the Checkpoint A regeneration (user-requested). Side-fixes made + kept (correct regardless):
- `backend/app/workers/queues.py` — summary RQ `job_timeout` now tracks the HTTP timeout + 120s buffer
  (was RQ default 180s < 240s detailed HTTP timeout → work-horse SIGKILL'd legitimate long calls). This
  turned the detailed failure from an opaque work-horse kill into a clean 408 (real diagnosis).
- dev `.env` HTTP timeouts raised (300/600) — doesn't fix the 408 (server-side), but removes the client-side wall.
- §3A re-architected: deterministic moved off the always-on e2e overlay → gate-only `docker-compose.deterministic.yml`
  + a provider guard in `run-active-suite.sh`; dev/browse stack = k2think.

**ESCALATED (rule 10):** Option 2 (brief-from-detailed) is blocked — the detailed itself 408s. Real fix is a
NEW decision: (a) truncate the transcript input (lossy, works now), (b) map-reduce chunk→summarize→combine
(proper; EXPLICITLY excluded from Stage 4.5), or (c) accept the size ceiling (short OK; real lectures deferred
to a map-reduce stage). The corrective's "real summaries of the actual transcript" goal is **NOT met** until
this is decided. (4.6 reaper will mark the abandoned summary-less transcripts; jobs were cleaned this session.)
