## Scout subagent findings
- Mount `SectionTranscriptControl` inside each lecturer section article.
- Gate mounting to `detail.type === "lecture" || detail.type === "lab"`.
- Current section type is a plain string, not a generated enum.
- Generated status read is `TranscriptsService.getSectionTranscript(...)`.
- Upload helper should follow `upload.ts`: `FormData`, field `file`, `OpenAPI.TOKEN`, bearer header, 401 redirect, and 403 session-preserving error.
- Treat only `404 TRANSCRIPT_NOT_FOUND` as the normal no-transcript state.
- Do not reuse the old unmounted `TranscriptUpload` path because it bypassed wrapper recovery.

## Upload helper implementation
- Added `uploadTranscript(...)` to `frontend/src/lib/api/upload.ts`.
- Endpoint: `/modules/{moduleId}/sections/{sectionId}/transcript`.
- Method: `POST`.
- Multipart field: `file`.
- Return type: `TranscriptMeta`.
- Existing PDF asset upload/replace behavior remains unchanged.

## Wrapper methods added
- Added `api.transcripts.getActive(moduleId, sectionId)` in `frontend/src/lib/api/wrapper.ts`.
- The wrapper calls generated `TranscriptsService.getSectionTranscript(...)`.
- Existing auth recovery remains in force: 401 signs out and redirects to `/login`; 403 throws `ForbiddenError` without clearing the session.

## SectionTranscriptControl behavior
- Loads active transcript status when mounted.
- Renders only where `LecturerModuleDetail.tsx` passes lecture/lab sections.
- Shows `No transcript uploaded yet.` for `404 TRANSCRIPT_NOT_FOUND`.
- Shows upload input only when no active transcript exists.
- Accepts `.vtt,.txt`.
- Uses `uploadTranscript(...)` for upload.
- Shows filename, MIME type, size, and transcript status when active transcript metadata exists.
- Handles `409 TRANSCRIPT_ALREADY_EXISTS` with `This section already has an active transcript.` and reloads active metadata.
- Errors render with `role="alert"` and no stack traces.
- No replacement, retry, supersession, raw text, segment, chunk, storage key, or checksum UI was added.

## TranscriptStatusBadge behavior
- Displays status with `role="status"` and `aria-live="polite"`.
- Polls `api.transcripts.getActive(...)` every 2.5 seconds while status is non-terminal.
- Stops polling on `completed`.
- Stops polling on `failed`.
- Stops polling on timeout and renders `Processing is taking longer than expected. Refresh to check again.`
- Stops polling on unmount.
- Treats `404 TRANSCRIPT_NOT_FOUND` during polling as a neutral no-active-transcript state, not as timeout.
