## Summary
Stage 5.5 is closed from browser evidence. The UI now exposes schedule preview/create, resolver-backed
by-week views, metadata edit, lab `dueAt`, lab PDF + `.ipynb` upload, student deadline display, and
student download routing by `assetKind`.

The reference-course oracle is proven in the new browser gate: 11 May-26 Jun 2026 with Mon/Tue/Wed
lectures and Thu labs produces exactly 28 sections: 21 lectures, 7 labs, 0 Friday sections, 7 weeks.
The full active Playwright suite is green after the 5.5d reseed condition.

## What changed
- Added admin dry-run preview `POST /admin/modules/preview-sections`, implemented over the same
  `generate_sections` pure function used by module creation.
- Added resolver-backed by-week reads for admin and assigned lecturers.
- Extended student-safe DTOs with `dueAt` and `assetKind` so the UI can display lab deadlines and route
  downloads correctly.
- Reworked admin module creation UI to require preview before create and expose course dates, weekday
  pattern, week-start day, and quiz day.
- Added lecturer by-week view and inline metadata editor for `week_number`, `session_date`, and lab
  `due_at`.
- Extended lecturer lab upload UI to accept `.ipynb` and send upload-time `dueAt`.
- Extended student UI to show lab deadlines; processable PDFs use signed URLs and attachments use the
  backend `/download` endpoint.
- Added `tests/e2e/5.5e-ui-browser-gate.spec.ts`.
- Made legacy active E2E specs compatible with the 5.5 schedule UI and self-contained fault setup:
  - Stage 2 admin E2E helper now previews before create.
  - `insertSection` DB helper returns JSON through a SELECT CTE instead of parsing `INSERT 0 1`.
  - 4.5d fault specs recreate `ai_worker` with `LLM_FAULT_INJECTION` per test and restore it.
  - E2E teardown accepts exact `.ipynb` attachment keys.
