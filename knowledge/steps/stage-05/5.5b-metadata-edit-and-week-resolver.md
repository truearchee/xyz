## Summary
Built the 5.5b backend feature work. Added `PATCH /modules/{moduleId}/sections/{sectionId}/metadata`
for admins and assigned lecturers, with a strict whitelist (`weekNumber`, `sessionDate`, `dueAt`), D13
recompute behavior, backend 403/404 split, and lab-only `dueAt`. Added
`platform/query/section_week_resolver.py`, a read-only resolver that returns active lecture/lab section
metadata by stored `week_number` and defaults to excluding null metadata for Stage 6 safety. Regenerated
the TypeScript API client from the new OpenAPI contract; no product UI was built.

## Guardrails proved
- **D13 ordering:** PATCH `{sessionDate}` recomputes week after reading the full payload; PATCH
  `{sessionDate, weekNumber}` honors explicit week; PATCH `{weekNumber}` leaves the date untouched.
- **`include_unstamped` safety:** resolver default excludes null `week_number`/`session_date` rows; the
  docstring and ADR explicitly say Stage 6 must use the default to avoid pulling empty/holiday/unstamped
  sections into quiz scope.
- **403/404 split:** student role denial returns `403 CONTENT_FORBIDDEN`; unassigned lecturer and
  cross-module section access return `404 SECTION_NOT_FOUND`.
