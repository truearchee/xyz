## Summary
Built Stage 8.4 Option A (navigation/UX + conversation management) on top of 8.1/8.2 with **no new AI
surface**: a student-only ChatGPT-style **Workspace** (list / open / rename / soft-delete) and a
**floating assistant widget** on every student page, both reaching the existing 8.2 grounded lecture
conversation. The hard part — conversation-management correctness (invariants A–E) and a single source
of truth between the inline panel and the widget — is enforced server-side (one-active partial index +
soft-delete + 4.7-gated reads) and client-side (one shared store, one poll loop).

8.1 already enforced one active `lecture_default` per (student, section), so there was **no
duplicate-collapse migration** (prereq #4 resolved against the live schema). Prereqs #2/#3 verified
read-only at start: single Alembic head `0039`, all 6 sibling worktrees + main clean, only 0032/0033
touch the assistant tables, `conversation_kind` modified by no branch.

- **GET `/student/assistant/conversations/{id}` added** (amendment) — the spec's backend scope listed
  list/messages/send/rename/delete; the standalone Workspace conversation route + the context pill need
  the conversation's lecture/module titles for a deep-linkable, non-fragile view. Completes the REST
  resource (GET/PATCH/DELETE). Recorded in the spec's Amendments.
- **Workspace uses per-conversation routes** (`/student/assistant/[conversationId]`) as the plan
  specified (not a single master-detail page).
- **3 fixes the live gate caught** (see [[steps/stage-08/findings-8.4-gate-run]]): (1) a render/fetch
  loop in `AssistantWorkspaceConversation` (the churning `store` value was in a `useEffect` dep array →
  depend on the stable `useCallback` actions instead); (2) added an `assistant-conversation-title`
  heading so a rename is visible on the conversation page (was only in the list); (3) the e2e spec used
  `nav-assistant` (dashboard-only) where the lecture page exposes `shell-nav-assistant` (AppShell).
- **External review repair (post-closeout, pre-commit)**: fixed store-level send idempotency, 0040
  downgrade delete→reopen reconciliation, lecture widget context pill, stale docs, and excluded unrelated
  Stage 6 screenshots/root `package-lock.json` drift from the 8.4 diff.

  the composer is disabled until initial messages settle; added the backend pending-turn send guard;

  open/create; treated 404 load/poll failures as gone in the shared store; chose the documentation path

- **Post-rebase integration (PR #10)**: kept Stage 4.9g tokenized frontend additions from `main` while

  deep-link route to render `StudentSectionDetail` because Stage 4.7/5/7/8 browser gates and the 8.4

## Decisions made
- **ADR-053** — keyset pagination sibling envelope (`KeysetPage`), the spec's anticipated rule-10
  escalation (offset envelope can't express a cursor). Offset stays the default; keyset only for deep,
  high-churn, poll-while-appending feeds.
- **ADR-054** — conversation-management contract: soft-delete + one-active-with-`deleted_at` partial
  index + current-access-wins (404, not 403) + delete-while-pending resurrection guard.
- **/plan-design-review** (D3:B): design completeness 6→9, 18 decisions written into the plan, 0
  unresolved; built in the inline idiom (monochrome repaint stays Stage 4.9).
