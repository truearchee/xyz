# Steps — Stage 8.5 (Save-to-Glossary from the Assistant)

Written from `git status`/`git diff` + captured command output. Branch `stage-8.5-implementation`.

## What was built

**Backend (8.5a)** — extends the ONE Stage 7 write path; no Stage 7 redesign.
- `backend/alembic/versions/0041_glossary_conversation_source.py` (NEW): adds `source_conversation_id`
  → `assistant_conversations` and `source_message_id` → `assistant_messages` (nullable, SET NULL) to
  `glossary_source_references`; drop+recreate the `source_type` CHECK to add `'conversation'`; partial-
  unique index `uq_glossary_source_references_conversation_message (glossary_entry_id, source_message_id)
  WHERE source_type='conversation'`. Guarded/re-runnable; downgrade deletes `'conversation'` rows first.
- `backend/app/platform/db/models/glossary_source_reference.py` (M): the two FK columns, widened CHECK,
  the partial-unique `Index` in `__table_args__` (predicate identical to the migration).
- `backend/app/domains/glossary/schemas.py` (M): `ConversationSaveSource`; `SaveHighlightRequest`
  gains optional `conversation` + optional `module_section_id` + a `model_validator` requiring exactly
  one; `GlossarySourceReferenceRead` gains the two FK fields.
- `backend/app/platform/query/assistant_save_source_read.py` (NEW): `get_conversation_save_source(...)`
  → `ResolvedConversationSource` or `None` (pinned-404 set). Reuses `get_visible_student_section`; does
  NOT import the assistant domain (rule 8).
- `backend/app/domains/glossary/snippet.py` (NEW): conservative `normalize_for_containment` +
  `selected_text_in_message` (anti-spoofing).
- `backend/app/domains/glossary/save_service.py` (M): `save_term` gains `definition_context` +
  `source_conversation_id` + `source_message_id`; `_resolve_definition` decouples prompt context from the
  stored snippet; `_add_source_reference` helper makes the conversation source-attach idempotent
  (ON CONFLICT DO NOTHING) on both new-entry and duplicate paths; summary/manual/quiz behavior unchanged.
- `backend/app/domains/glossary/service.py` (M) + `policy.py` (M): `_save_from_conversation`
  orchestration (require_student → selectedText required 422 → read model None→404 → role!=assistant 422
  → status!=completed 409 → containment 422 → subject from section → `save_term(..., source_type=
  'conversation', definition_context="")`); five new error-code constants. Summary branch byte-for-byte.
- `backend/app/domains/admin/dev_reseed.py` (M): `EXPECTED_ALEMBIC_VERSION` 0040→0041.
- `backend/tests/test_glossary_conversation_save.py` (NEW): 16 tests (cases a–m).

**Frontend (8.5b)** — single mount point, backward-compatible.

- `frontend/src/lib/api/**` (regenerated): `ConversationSaveSource`, `SaveHighlightRequest` (+ optional
