- Added the 8.6c browser gate proving the live student path, another-student sentinel exclusion, no planner
  or calendar artifacts, no `.ics` assets, and no planner/analytics import boundary violation.

  fix is the `E2E_COMPOSE_FILES` hook in those fault specs.

  student; later runs correctly resumed it. The 8.6c gate now asserts starter chips only on an empty
  resumed-or-created conversation and asserts the newly appended reply by completed-message count.

  every full-suite run. Screenshot writing is now opt-in via `STAGE6_SCREENSHOT_DIR`.
