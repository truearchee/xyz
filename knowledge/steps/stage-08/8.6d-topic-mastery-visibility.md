## Summary
Implemented owner-approved approach A: the shared Stage 9 topic-mastery read model now filters to
published sections. This closes the hidden-section mastery leak for the Stage 9 progress API and for the
Stage 8.6b/8.6c assistant weak-topic contexts that consume the same helper.

No schema migration was needed for the visibility repair. After rebasing over Stage 10, a no-op Alembic
merge revision `0082` was added to join the Stage 8.6 migration head (`0044`) with the Stage 10 head
(`0081`).

## Files changed
- `backend/app/platform/query/progress_read.py` - `list_topic_mastery()` now requires
  `ModuleSection.publish_status == "published"`.
- `backend/tests/test_progress_api.py` - added the Stage 9 regression proving
  `/student/modules/{module_id}/progress` omits mastery rows for unpublished sections.
- `backend/tests/test_assistant_modes.py` - added exam-prep and time-management regressions proving hidden
  topic mastery does not reach prompts or assistant snapshots.
- `knowledge/steps/findings-topic-mastery-visibility.md` - added the Stage 10/11 hand-off note for direct
  reads outside the shared helper.
- `backend/alembic/versions/0082_merge_stage10_stage86_heads.py` - no-op merge revision added after rebase
  to keep Alembic at a single head.

## Behavior pinned
- Hidden section topic mastery disappears from the student progress detail API.
- Exam-prep weak-topic prompt context omits unpublished topic mastery; the generated context snapshot also
  omits the hidden section from resolved scope.
- Time-management weak-topic prompt context omits unpublished topic mastery; `weakTopicRefs` is empty for the
  hidden-only setup, and the hidden section id is absent from deadline refs.
- The shared predicate is intentionally applied in the platform query rather than only in assistant code, so
  current consumers inherit the same student-visible boundary.
