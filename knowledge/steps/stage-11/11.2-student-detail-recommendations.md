## Rule-10 findings
- **Design-plan reconciliation:** `knowledge/design-plan.md` says "draft-message modal" for Stage 11 but does not
  show a literal Send action. The Stage 11 master spec's Copy draft / Mark acted / Dismiss-only contract wins, and
  the implementation contains no Send button or send endpoint.
- **Real-provider prompt-budget finding:** the first real-provider smoke reached K2Think and echoed
  `MBZUAI-IFM/K2-Think-v2`, but the response hit `finish_reason='length'` at `max_tokens=900` before a complete
  JSON object validated. `recommendation_copy/v1` now uses `max_tokens=2000`; the prompt checksum was updated.
- **E2E provider override finding:** putting the real provider into repo-root `.env` for rule 11 made normal
  Playwright worker recreates inherit `k2think`. `docker-compose.e2e.yml` now explicitly includes `ai_worker`
  and pins E2E services to `${LLM_PROVIDER:-deterministic}`; `docker-compose.fault.yml` does the same for
  fault-injected `ai_worker` recreates.
- **E2E glossary residue finding:** Stage 8.5 uses the standing student and the deterministic term `concise`.
  Teardown now removes run-scoped glossary entries/sources/events plus that fixed E2E term for standing users.
- **CSO dependency debt:** `npm audit --audit-level=high` reports existing frontend advisories for `next`
  (critical), `undici` (high), and moderate `js-yaml`/`postcss`. Stage 11.2 did not modify frontend dependency
  manifests; record for Stage 12/security maintenance.
- **Local image drift:** base compose worker recreates can pick up a stale shared `kyiv-backend` image. Rebuild or
  retag the image from the current tree before full E2E runs that include worker-recreate specs.
