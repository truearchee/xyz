## Summary
Stage 11.4 is FULLY VERIFIED. It adds a deterministic, read-only, student workload planner:
students can set availability, generate/regenerate an active study plan, and see stored plan
items backed by remaining deadlines plus the latest 11.1 weak-topic risk gaps.

No AI, prompt, provider, `AIRequestLog`, validator, model route, Stage 10 read, assistant work,
gamification work, `.ics` export, grade advice, drag/drop, plan-item edit, accept/reject, or mark-done
behavior was added.

Legacy horizon fallback is only a safety net for modules with genuinely unknown `CourseModule.ends_on`; it uses
the later of latest known remaining `due_at` and the config-backed fallback horizon and never truncates a
resolvable course end.

## Browser gate details
The 11.4 gate seeds:
- two assigned students in the same module;
- same-day work, close deadlines with a detectable stable tiebreak, a later deadline, and a topic gap;
- availability that forces exact bounded overflow and a tight physically-impossible residual.

The gate asserts:
- nearest-deadline-first ordering: same-day, close A, close B, later deadline, then gap;
- stored estimate and reason on every item;
- deadline tasks schedule before their due dates when scheduled;
- daily total can exceed `100` only by the exact configured `25` minute allowance;
- tight work is visible, no task is dropped, and no scheduled day exceeds cap plus allowance;
- same-day `due_at` work is not silently dropped or double counted;
- plan rows expose no edit/done/accept/reject controls and no draggable rows;
- availability change plus regenerate supersedes the old plan and leaves one active plan;
- student API responses contain no other student's IDs/emails/data;
- same-module other student, lecturer, admin, and unassigned access return 403 where required.

## Rule-10 findings
- **Reserved SQL word in migration checks:** the first `0058` upgrade failed because the `window` column was
  referenced unquoted in CHECK constraints. Fixed by quoting `"window"` in the migration and ORM checks; the
  round trip then passed.
- **Generated run manifest precondition:** the new 11.4 E2E spec recorded cleanup IDs before creating its run
  manifest. Fixed by initializing `tests/e2e/.runs/{runId}.json` inside `requireRunId`, matching the Stage 9
  pattern.
- **Read-model import gap:** live generate first returned 500 because `analytics_read.py` referenced
  `StudentProgressSnapshot` without importing it. Fixed and pinned by focused backend tests plus the browser gate.
- **Stage 9 read-only selector collision:** mounting workload availability on My Progress introduced a
  `Save availability` button, conflicting with the Stage 9 no-Save assertion. The affordance is now
  `Update availability`; plan items remain read-only.
- **E2E provider setup:** an initial raw `npx playwright test` inherited `LLM_PROVIDER=k2think` from `.env` and
  failed generated-content gates against the real provider. Recreated the app services with
  `LLM_PROVIDER=deterministic` and used the documented success/fault split; both sets passed.
