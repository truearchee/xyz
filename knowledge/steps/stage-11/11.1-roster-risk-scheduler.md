## Summary
Stage 11.1 is **FULLY VERIFIED**. Implemented the scheduler and deterministic risk foundation:

- Migration `0056` adds `agent_runs` and `student_risk_snapshots` after current main head `0041`.
- `platform/scheduler` runs as a Python scheduler service with a Postgres advisory lock and enqueues recorded
  `AgentRun` jobs onto the existing default RQ worker queue.
- The new analytics domain computes `risk-v1` deterministic per-course risk from shipped inputs only:
  Stage 9 forecast data, quiz attempts, `StudentActivityEvent`, section schedule/deadlines, and topic mastery.
- UI risk reads are live-on-read; scheduled/manual runs persist snapshots and counts for history/proactive layers.
- Snapshot retention uses config-backed retention days and deletes historical snapshots only when a newer snapshot
  exists for that student/module, so the latest snapshot is preserved.
- Added admin manual trigger/polling endpoints, lecturer roster risk endpoint, and student self risk endpoint.
- Added Redis-backed rate limiting for admin manual triggers after admin auth; `AgentRun.triggered_by_user_id`
  remains the audit field for manual runs.
- Added lecturer roster UI with `Needs support: N`, tier filtering, reasons, and cited metrics.
- Added student gentle `Where you stand` view on module/progress surfaces. The student endpoint and component expose
  student-safe reason text only: no `riskTier`, severity, lecturer copy, cited metrics, or peer comparison language.
- Added ADR-056 for the scheduler/advisory-lock/idempotency/deterministic-risk contract.

No assistant, gamification, Stage 10, AI prompt, AIRequestLog, recommendation, assessment-analysis, workload
planner, `.ics`, or grade-advice work was added.
