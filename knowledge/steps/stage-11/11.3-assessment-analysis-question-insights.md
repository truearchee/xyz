## Rule-10 findings
- **Topic provenance hardening:** review found that a malformed question could point `source_section_id` at a
  section from another module. Fixed by accepting topic mappings only when the resolved section belongs to the
  requested module. Backend tests now seed a cross-module section as unresolvable provenance.
- **Local worker image drift:** the first full-suite run failed all assistant tests because the Stage 4.5 fault
  tests recreate `ai_worker` through `docker-compose.yml + docker-compose.fault.yml`, which uses the shared
  `kyiv-backend` image and bypasses the local Stage 11 bind-mounted override. The image was stale and selected
  nonexistent assistant columns. Rebuilding `kyiv-backend` from this workspace source fixed the drift; the final
  full suite passed `24/24`.
- **Cold Next-dev redirect timing:** one diagnostic full-suite run after a fresh frontend recreate failed only the
  first 11.1 login redirect at the default 10s wait; the warmed final suite passed. The new 11.3 gate uses the
  existing 30s redirect wait pattern from the shell-routing test.
- **Provider-independent backend test:** the full backend run initially exposed that a prior 11.2 recommendation AI
  test valid-output path used the `.env` provider instead of injecting `DeterministicTestProvider`. The test now
  injects the deterministic provider explicitly; product behavior is unchanged.
